## 1주차 알고리즘 입문

### 알고리즘의 다섯 가지 속성
- **정확성(Correctness)** — 모든 유효 입력에 대해 올바른 출력을 내야 한다.
- **실행 가능성(Executability)** — 각 단계가 기계적으로 실행 가능해야 한다.
- **유한성(Finiteness)** — 유한 시간 안에 반드시 종료되어야 한다.
- **효율성(Efficiency)** — 시간과 공간을 적절히 사용해야 한다.
- **명확성(Unambiguity)** — 각 단계가 모호하지 않게 명시되어야 한다.

### 유클리드 GCD
```
gcd(a, b) = gcd(b, a mod b),  gcd(a, 0) = a
```
**설명** — 공약수 집합이 $(a, b) \leftrightarrow (b, a \bmod b)$ 에서 보존되므로 최댓값도 같다. 기저 조건은 $b = 0$ 이며 이때 $a$ 가 정답이다. 복잡도는 최악 $O(\log \min(a, b))$ 이고, 연속한 피보나치 수 쌍에서 최악이 발생한다.

### $\log_2 n$ 테마 (절반 분할)
이진 탐색(Binary Search), 가짜 동전(Fake Coin), 독 포도주(Poisoned Wine) 문제는 모두 매 단계 탐색 공간을 절반으로 줄여 $\log n$ 의 성능을 얻는다.

### 이진 탐색 주의
```python
while left <= right:     # 등호 필수
    mid = (left + right) // 2
```
**설명** — 정렬된 배열에서 매 단계 중간값과 키를 비교하여 탐색 범위를 절반으로 축소한다. `<=` 에서 등호가 빠지면 `left == right` 인 마지막 한 원소를 검사하지 않아 실패한다. 복잡도는 $O(\log n)$ 이다.

### 오일러 회로 조건
- **회로 존재 조건** — 모든 정점의 차수가 짝수여야 한다.
- **경로 존재 조건** — 홀수 차수 정점이 정확히 0개 또는 2개여야 한다.
- **주의** — 다리(Bridge, 제거 시 그래프를 분리하는 간선)를 너무 일찍 건너면 반대쪽의 간선을 모두 순회할 수 없게 된다.

## 2주차 복잡도 분석

### 점근 표기법 정의
$$ O(g(n)) = \{ f : \exists c > 0, n_0 \geq 0, \forall n \geq n_0, f(n) \leq c \cdot g(n) \} $$

- **O** — 상한, "최대 이 정도로 증가한다"는 의미이다.
- **Ω** — 하한, "최소 이 정도로 증가한다"는 의미이다.
- **Θ** — 꼭 맞는 한계로, 상한과 하한이 모두 성립한다.

### 복잡도 계층
```
O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

### Big-O 증명 레시피 (다항식)
- 다항식 $f = a_k n^k + \cdots + a_0$ 에 대해 $c = |a_k| + \cdots + |a_0|$, $n_0 = 1$ 로 잡으면 대부분 성립한다.
- **예시** — $2n^2 - 8n + 3 = O(n^2)$ 은 $c = 13$, $n_0 = 1$ 로 증명된다.

### 점화식 해결 세 가지 방법
1. **반복 대입** — 점화식을 단계별로 전개하여 패턴을 찾는다.
2. **추측 후 귀납 증명** — 해의 형태를 추측하고 귀납법으로 검증하며, 필요하면 $cn \log n - d$ 로 추측을 강화한다.
3. **마스터 정리** — $T(n) = aT(n/b) + f(n)$ 형태의 점화식을 표준적으로 해결한다.

### 마스터 정리 빠른 참조
| Case | 조건 | 결과 |
|---|---|---|
| 1 | $f(n) = O(n^{\log_b a - \varepsilon})$ | $\Theta(n^{\log_b a})$ |
| 2 | $f(n) = \Theta(n^{\log_b a})$ | $\Theta(n^{\log_b a} \log n)$ |
| 3 | $f(n) = \Omega(n^{\log_b a + \varepsilon})$ + 정규성 조건 | $\Theta(f(n))$ |

### 마스터 정리 예시표
| 점화식 | $a, b$ | $n^{\log_b a}$ | $f(n)$ | Case | 결과 |
|---|---|---|---|---|---|
| $T(n) = 2T(n/2) + n$ | 2, 2 | $n$ | $n$ | 2 | $\Theta(n \log n)$ |
| $T(n) = T(n/2) + 1$ | 1, 2 | $1$ | $1$ | 2 | $\Theta(\log n)$ |
| $T(n) = 2T(n/2) + 1$ | 2, 2 | $n$ | $1$ | 1 | $\Theta(n)$ |
| $T(n) = 4T(n/2) + n$ | 4, 2 | $n^2$ | $n$ | 1 | $\Theta(n^2)$ |
| $T(n) = 4T(n/2) + n^2$ | 4, 2 | $n^2$ | $n^2$ | 2 | $\Theta(n^2 \log n)$ |
| $T(n) = 7T(n/2) + n^2$ (Strassen) | 7, 2 | $n^{\log_2 7}$ | $n^2$ | 1 | $\Theta(n^{\log_2 7})$ |

### 마스터 정리를 쓸 수 없는 경우
- $T(n) = T(n-1) + n$ 은 반복 대입으로 $\Theta(n^2)$ 이 된다.
- $T(n) = T(n-1) + T(n-2) + 1$ 은 특성 다항식으로 풀면 $\Theta(\phi^n)$ 이 된다.
- $T(n) = T(n/5) + T(7n/10) + n$ 은 귀납법과 기하급수 합으로 $\Theta(n)$ 이 된다.

## 3주차 자료구조와 기본 정렬

### 자료구조 요약
| 구조 | 접근 방식 | 주요 연산 복잡도 |
|---|---|---|
| 배열 / 리스트 | 인덱스 기반 | 접근 $O(1)$, 중간 삽입·삭제 $O(n)$ |
| 스택 (LIFO) | push / pop | 모두 $O(1)$ |
| 큐 (FIFO) | enqueue / dequeue | 모두 $O(1)$ |
| 힙 | 우선순위 기반 | insert·extract $O(\log n)$, buildHeap $O(n)$ |

### 힙의 배열 표현 (1-index)
- 노드 $i$ 의 자식은 $A[2i]$ 와 $A[2i+1]$ 이다.
- 노드 $i$ 의 부모는 $A[\lfloor i/2 \rfloor]$ 이다.
- **최대 힙(Max-Heap)** 에서는 모든 부모 키가 자식 키 이상이다.

### 정렬 완전 비교표
| 알고리즘 | 최악 | 평균 | 최선 | 공간 | 안정성 | 제자리 |
|---|---|---|---|---|---|---|
| 선택 | $\Theta(n^2)$ | $\Theta(n^2)$ | $\Theta(n^2)$ | $O(1)$ | ✗ | ✓ |
| 버블 | $\Theta(n^2)$ | $\Theta(n^2)$ | $\Theta(n)$ | $O(1)$ | ✓ | ✓ |
| 삽입 | $\Theta(n^2)$ | $\Theta(n^2)$ | $\Theta(n)$ | $O(1)$ | ✓ | ✓ |
| 병합 | $\Theta(n \log n)$ | $\Theta(n \log n)$ | $\Theta(n \log n)$ | $O(n)$ | ✓ | ✗ |
| 퀵 | $\Theta(n^2)$ | $\Theta(n \log n)$ | $\Theta(n \log n)$ | $O(\log n)$ | ✗ | ✓ |
| 힙 | $\Theta(n \log n)$ | $\Theta(n \log n)$ | $\Theta(n \log n)$ | $O(1)$ | ✗ | ✓ |
| 계수 | $\Theta(n+k)$ | $\Theta(n+k)$ | $\Theta(n+k)$ | $O(k)$ | ✓ | ✗ |
| 기수 | $\Theta(d(n+k))$ | $\Theta(d(n+k))$ | $\Theta(d(n+k))$ | $O(n+k)$ | ✓ | ✗ |

### 비교 기반 하한
- 모든 비교 기반 정렬은 최악 $\Omega(n \log n)$ 비교가 필요하다.
- 결정 트리 모델로 증명된다: $n!$ 개의 순열을 구별하려면 높이가 $\log_2(n!) = \Theta(n \log n)$ 이상이어야 한다.
- 계수 정렬과 기수 정렬은 비교를 쓰지 않으므로 이 하한을 우회할 수 있다.

### 정렬 선택 가이드
- 안정성이 필수이면 병합 정렬을 쓴다.
- 메모리 제약이 있고 최악 성능 보장이 필요하면 힙 정렬을 쓴다.
- 일반적으로 가장 빠른 정렬이 필요하면 랜덤 피벗 퀵 정렬을 쓴다.
- 작거나 거의 정렬된 입력에는 삽입 정렬이 유리하다.
- 정수 값의 범위가 제한적이면 계수 정렬 또는 기수 정렬을 쓴다.

### 기수 정렬의 LSD 순서
- 최하위(LSD) 자리부터 최상위(MSD) 자리까지 순서대로 정렬한다.
- 각 자리 정렬은 반드시 안정 정렬(보통 계수 정렬)이어야 한다.
- 이전 자리의 순서가 유지되어야 최종적으로 모든 자리가 정렬된다.

## 4주차 분할 정복

### 기본 패턴
```
DivideAndConquer(P):
    if P 가 충분히 작으면:
        직접 해결
    else:
        DIVIDE:   P 를 부분 문제 P1, ..., Pk 로 분할
        CONQUER:  각 Pi 를 재귀로 해결
        COMBINE:  부분 해들을 원래 문제의 해로 결합
```
**설명** — 큰 문제를 독립적인 작은 부분 문제로 나누어 재귀로 풀고 결합하는 패러다임이다. 부분 문제가 서로 겹치지 않으므로 각 부분 문제를 한 번씩만 풀면 된다. 점화식은 보통 $T(n) = aT(n/b) + f(n)$ 의 형태를 가지며 마스터 정리로 해결한다. 부분 문제가 겹치면 동적 계획법을 써야 한다.

### 대표 알고리즘
| 알고리즘 | 점화식 | 복잡도 |
|---|---|---|
| 병합 정렬 | $2T(n/2) + \Theta(n)$ | $\Theta(n \log n)$ |
| 이진 탐색 | $T(n/2) + O(1)$ | $\Theta(\log n)$ |
| 퀵 정렬 (평균) | $2T(n/2) + \Theta(n)$ | $\Theta(n \log n)$ |
| 퀵 정렬 (최악) | $T(n-1) + \Theta(n)$ | $\Theta(n^2)$ |
| 랜덤 선택 | $T(3n/4) + \Theta(n)$ | 평균 $\Theta(n)$ |
| 중앙값의 중앙값 | $T(n/5) + T(7n/10) + \Theta(n)$ | 최악 $\Theta(n)$ |
| 최근접 쌍 | $2T(n/2) + O(n \log n)$ | $O(n \log^2 n)$ |

### 랜덤 선택 (퀵 선택)
```
select(A, p, r, i):
    if p == r: return A[p]
    q = partition(A, p, r)
    k = q - p + 1
    if i == k:  return A[q]
    elif i < k: return select(A, p, q-1, i)
    else:       return select(A, q+1, r, i - k)
```
**설명** — 퀵 정렬의 파티션을 재사용하여 피벗의 순위 $k$ 를 계산한 뒤, 원하는 $i$ 가 어느 쪽에 있는지 판단해 한쪽 부분 배열만 재귀 호출한다. 양쪽을 모두 정렬할 필요가 없기 때문에 평균 시간은 $\Theta(n)$ 이다. 오른쪽으로 갈 때는 피벗과 왼쪽 부분을 건너뛴 만큼 $i$ 를 $i - k$ 로 보정한다.

### 중앙값의 중앙값 (Median of Medians)
- 원소를 5개씩 묶은 뒤 각 그룹의 중앙값을 구하고, 그 중앙값들의 중앙값 $M$ 을 재귀로 찾는다.
- 이 $M$ 을 피벗으로 쓰면 최소 $3n/10$ 개 원소가 $M$ 이하이고 최소 $3n/10$ 개가 $M$ 이상이므로, 재귀 크기는 최대 $7n/10$ 이 된다.
- 점화식 $T(n) = T(n/5) + T(7n/10) + \Theta(n)$ 은 $1/5 + 7/10 < 1$ 이므로 $\Theta(n)$ 으로 수렴한다.

### 최근접 쌍
1. 점들을 x 좌표 기준으로 정렬한다.
2. 중앙 x 로 좌/우를 분할하여 각각 재귀 호출한다.
3. $d = \min(d_L, d_R)$ 을 구한다.
4. 분할선에서 $\pm d$ 이내의 점들로 스트립을 구성하고 y 좌표로 정렬한다.
5. 스트립 안에서 각 점은 y 순서상 최대 7개 이웃만 검사하면 충분하다.

### 퀵 정렬의 Partition (Lomuto)
```
partition(A, p, r):
    pivot = A[r]
    i = p - 1
    for j = p to r - 1:
        if A[j] ≤ pivot:
            i += 1
            swap A[i] ↔ A[j]
    swap A[i+1] ↔ A[r]
    return i + 1
```
**설명** — 배열의 마지막 원소를 피벗으로 쓴다. 포인터 `i` 는 "피벗 이하 원소들의 끝" 을 가리키고 `j` 는 현재 검사 중인 위치이다. `A[j] ≤ pivot` 이면 `i` 를 증가시키고 스왑해 이하 영역을 확장한다. 루프가 끝난 뒤 피벗을 $i+1$ 위치로 옮겨 최종 정렬 위치를 확정하고 그 인덱스를 반환한다. 반환 인덱스 왼쪽은 이하, 오른쪽은 초과가 된다.

### 최악을 피하는 방법
- **랜덤 피벗** — 무작위 인덱스를 선택해 마지막과 스왑한 뒤 파티션한다.
- **Median-of-3** — 첫 원소, 중간 원소, 마지막 원소의 중앙값을 피벗으로 쓴다.
- **Introsort** — 재귀 깊이가 일정 수준을 넘으면 힙 정렬로 전환한다.

### 분할 정복이 실패하는 경우
- 순진한 재귀 피보나치는 $T(n) = T(n-1) + T(n-2) + 1 \to O(\phi^n)$ 로 지수 시간이다.
- 원인은 같은 부분 문제가 여러 번 반복 계산되는 중복 부분 문제(Overlapping Subproblems)에 있다.
- 이러한 문제는 DP로 해결한다.

## 5주차 그리디

### 패러다임
매 단계에서 지금 당장 가장 좋아 보이는 선택(지역 최적)을 하고, 이미 내린 선택을 되돌리지 않는다.

### 정확성을 보장하는 두 조건
1. **그리디 선택 속성(Greedy-Choice Property)** — 어떤 최적해는 반드시 그리디 선택을 포함한다. 보통 교환 논증(Exchange Argument)으로 증명한다.
2. **최적 부분 구조(Optimal Substructure)** — 그리디 선택 후 남은 부분 문제의 최적해가 전체 최적해의 일부가 된다.

### 그리디가 실패하는 반례
| 문제 | 그리디 결과 | 최적 결과 |
|---|---|---|
| 동전 {1, 3, 4}, 금액 6 | 4 + 1 + 1 = 3개 | 3 + 3 = 2개 |
| 동전 {1, 5, 7, 10}, 금액 14 | 10 + 1 + 1 + 1 + 1 = 5개 | 7 + 7 = 2개 |
| 0/1 배낭 (가치/무게 정렬) | 용량이 남아도 분수 사용 불가 | DP로 풀어야 한다 |

### 대표 알고리즘
| 문제 | 그리디 전략 | 시간 복잡도 |
|---|---|---|
| 동전 거스름돈 (표준 화폐) | 큰 단위부터 선택 | $O(k)$ |
| 분할 가능 배낭 | 가치/무게 비율 내림차순 | $O(n \log n)$ |
| 활동 선택 (최대 비겹침) | 가장 빨리 끝나는 활동(EFT) | $O(n \log n)$ |
| 작업 스케줄링 (최소 기계) | 가장 빨리 시작하는 작업(EST) | $O(n \log n)$ |
| 허프만 코딩 | 최저 빈도 두 노드를 병합 | $O(n \log n)$ |
| 크루스칼 MST | 가중치 오름차순 간선 + Union-Find | $O(m \log m)$ |
| 프림 MST | 시작 정점에서 트리를 키움 | $O(n^2)$ 또는 $O(m \log n)$ |
| 다익스트라 | 최소 거리 미확정 정점 확정 | $O(n^2)$ 또는 $O(m \log n)$ |

### 활동 선택 vs 작업 스케줄링 (혼동 주의)
| 구분 | 활동 선택 | 작업 스케줄링 |
|---|---|---|
| 목표 | 활동 수 최대화 | 기계 수 최소화 |
| 그리디 기준 | EFT (가장 빠른 종료 시간) | EST (가장 빠른 시작 시간) |
| 기계 수 | 1개 | 여러 개 |

### 허프만 코딩
```
1. 문자 빈도로 최소 힙 Q 를 구성한다.
2. while |Q| ≥ 2:
     A = extract-min(Q)
     B = extract-min(Q)
     N = 새 노드, N.freq = A.freq + B.freq
     N.left = A,  N.right = B
     insert(Q, N)
3. 루트에서 리프까지의 0/1 경로가 각 문자의 접두 코드이다.
```
**설명** — 빈도가 가장 낮은 두 노드를 뽑아 합친 뒤 새 노드로 힙에 다시 넣는 과정을 반복한다. 합쳐지는 노드는 트리에서 더 깊게 위치하므로 긴 코드가 배정되고, 빈도가 높은 노드는 얕게 위치해 짧은 코드가 배정된다. 루트에서 각 리프까지의 0/1 경로가 바로 접두 코드가 된다. 최적성은 교환 논증으로 증명되며, 핵심은 최저 빈도 두 문자가 반드시 트리에서 형제라는 점이다.

### 크루스칼 vs 프림
| 구분 | 크루스칼 | 프림 |
|---|---|---|
| 접근 | 간선 중심 | 정점 중심 |
| 자료구조 | Union-Find | 우선순위 큐 |
| 복잡도 | $O(m \log m)$ | $O(n^2)$ 또는 $O(m \log n)$ |
| 적합 그래프 | 희소 그래프 | 밀집 그래프 |

### 다익스트라
```
D[v] = ∞ ; D[s] = 0 ; S = {}
while S ≠ V:
    u = S 밖에서 D[u] 가 최소인 정점
    S += {u}
    for v in neighbors(u) \ S:
        if D[u] + w(u, v) < D[v]:
            D[v] = D[u] + w(u, v)     # 간선 완화
```
**설명** — 출발 정점 $s$ 에서 각 정점까지의 최단 거리 $D[v]$ 를 계산한다. 매 반복에서 아직 확정되지 않은 정점 중 $D$ 가 최소인 $u$ 를 확정 집합 $S$ 에 추가하고, $u$ 의 이웃들에 대해 간선 완화를 수행한다. 음이 아닌 가중치 가정 덕분에 $S$ 에 편입된 정점의 $D$ 값은 이후 바뀌지 않는다는 불변식이 성립한다.

**음수 간선 주의** — 음수 간선이 있으면 이미 확정된 $D[u]$ 가 나중에 더 작아질 수 있어 그리디 불변식이 깨진다. 이 경우 벨만-포드 알고리즘($O(VE)$)을 써야 한다.

## 6주차 동적 계획법

### DP 의 두 필수 조건
1. **최적 부분 구조(Optimal Substructure)** — 전체 최적해가 부분 문제 최적해를 포함한다.
2. **중복 부분 문제(Overlapping Subproblems)** — 같은 부분 문제가 재귀 구조에서 여러 번 등장한다.

### 메모이제이션 vs 테이블화
| 구분 | 메모이제이션 (하향식) | 테이블화 (상향식) |
|---|---|---|
| 구현 | 재귀와 캐시 결합 | 반복문으로 테이블을 채움 |
| 계산 범위 | 실제 필요한 부분 문제만 | 모든 부분 문제 |
| 오버헤드 | 재귀 호출 스택 | 타이트 루프 |
| 실전 속도 | 상대적으로 느리다 | 상대적으로 빠르다 |

### DP 6단계 레시피
1. 부분 문제를 명확히 정의한다 (예: `dp[i, j]` 가 무엇을 뜻하는가).
2. 점화식을 작성한다.
3. 기저 조건을 식별한다.
4. 부분 문제를 채우는 순서를 결정한다.
5. 최종 답이 어느 셀에 저장되는지 확인한다.
6. 필요하면 역추적으로 실제 해를 복원한다.

### 대표 DP 문제 요약
| 문제 | 부분 문제 | 점화식 요약 | 시간 | 공간 |
|---|---|---|---|---|
| 피보나치 | $f[i]$ | $f[i-1] + f[i-2]$ | $O(n)$ | $O(1)$ |
| 행렬 경로 | $c[i,j]$ | $m[i,j] + \max(c[i-1,j], c[i,j-1])$ | $O(n^2)$ | $O(n^2)$ |
| LCS | $c[i,j]$ | 일치 시 $+1$, 불일치 시 $\max$ | $O(mn)$ | $O(mn)$ |
| 편집 거리 | $E[i,j]$ | 일치 시 대각선, 불일치 시 $1 + \min$ | $O(mn)$ | $O(mn)$ |
| 0/1 배낭 | $K[i,w]$ | $\max(K[i-1,w], v_i + K[i-1, w-w_i])$ | $O(nC)$ | $O(nC)$ |
| 동전 거스름돈 (최소) | $C[j]$ | $1 + \min_{d_i \leq j} C[j - d_i]$ | $O(nk)$ | $O(n)$ |
| 플로이드-워셜 | $d[i,j,k]$ | $\min(d[i,j], d[i,k] + d[k,j])$ | $O(n^3)$ | $O(n^2)$ |

### LCS 점화식
```
if X[i] == Y[j]: c[i][j] = c[i-1][j-1] + 1
else:            c[i][j] = max(c[i-1][j], c[i][j-1])
```
**설명** — $c[i][j]$ 는 $X[1..i]$ 와 $Y[1..j]$ 의 LCS 길이이다. 마지막 문자가 일치하면 두 문자를 모두 제거한 부분 수열의 LCS 에 1 을 더하고, 불일치하면 $X$ 의 마지막을 뺀 경우와 $Y$ 의 마지막을 뺀 경우 중 더 긴 쪽을 선택한다. 실제 LCS 문자열은 역추적으로 복원한다.

### 편집 거리 점화식
```
if X[i] == Y[j]: E[i][j] = E[i-1][j-1]
else:            E[i][j] = 1 + min(E[i-1][j],       # 삭제
                                   E[i][j-1],       # 삽입
                                   E[i-1][j-1])     # 교체
```
**설명** — $E[i][j]$ 는 $X[1..i]$ 를 $Y[1..j]$ 로 바꾸는 최소 연산 수이다. 마지막 문자가 일치하면 대각선 값을 그대로 취하고(연산 0), 불일치하면 삭제·삽입·교체의 세 경로 중 최소에 1 을 더한다. 기저 조건은 $E[i][0] = i$ (i 번 삭제), $E[0][j] = j$ (j 번 삽입)이다.

### 0/1 배낭 점화식
```
if w[i] > w:  K[i][w] = K[i-1][w]
else:         K[i][w] = max(K[i-1][w],
                            v[i] + K[i-1][w - w[i]])
```
**설명** — $K[i][w]$ 는 물건 1..i 까지 고려하고 용량이 $w$ 일 때의 최대 가치이다. 각 물건을 넣을지 말지 두 선택을 비교한다. 넣으려면 용량이 $w_i$ 이상이어야 하고, 넣을 경우 가치는 "나머지 용량으로 $i-1$ 까지 담은 최대 가치 $+ v_i$" 가 된다. 넣지 않으면 $i-1$ 까지 담은 값을 그대로 가져간다. 둘 중 큰 쪽을 선택한다.

### 플로이드-워셜
```
for k = 1 to n:     # k 가 반드시 가장 바깥 루프
    for i = 1 to n:
        for j = 1 to n:
            D[i][j] = min(D[i][j], D[i][k] + D[k][j])
```
**설명** — 중간 경유 정점 후보를 $\{1\} \to \{1, 2\} \to \cdots \to \{1..n\}$ 으로 점차 확장하며 모든 쌍 $i \to j$ 의 최단 거리를 갱신한다. 핵심 아이디어는 "정점 $k$ 를 경유하면 더 가까워지는가?" 를 모든 $(i, j)$ 에 대해 매 단계 확인하는 것이다. $k$ 가 반드시 최외곽 루프여야 하는 이유는, 각 $k$ 의 갱신이 "$\{1..k-1\}$ 만 경유할 때의 결과" 를 바탕으로 이루어져야 하기 때문이다. 음수 간선은 허용되지만 음의 사이클은 없어야 한다. 복잡도는 $\Theta(n^3)$ 이다.

## 공식 치트시트

### 점화식 빠른 참조
| 점화식 | 해 | 대표 예시 |
|---|---|---|
| $T(n) = T(n/2) + 1$ | $\Theta(\log n)$ | 이진 탐색 |
| $T(n) = T(n-1) + 1$ | $\Theta(n)$ | 팩토리얼 |
| $T(n) = 2T(n/2) + n$ | $\Theta(n \log n)$ | 병합 정렬 |
| $T(n) = 2T(n/2) + 1$ | $\Theta(n)$ | — |
| $T(n) = T(n-1) + n$ | $\Theta(n^2)$ | 삽입 정렬, 퀵 정렬 최악 |
| $T(n) = 2T(n-1) + 1$ | $\Theta(2^n)$ | 하노이 탑 |
| $T(n) = T(n-1) + T(n-2) + 1$ | $\Theta(\phi^n)$ | 순진한 재귀 피보나치 |

### Big-O 증명 공식
다항식 $f(n) = \sum a_i n^i$ 에 대해 $c = \sum |a_i|$, $n_0 = 1$ 로 잡으면 대부분 성립한다.

### 유클리드 GCD
$$ \gcd(a, b) = \gcd(b, a \bmod b),  \gcd(a, 0) = a $$

### 비교 기반 정렬 하한
모든 비교 기반 정렬은 최악 $\Omega(n \log n)$ 비교가 필요하며, 결정 트리 모델로 증명된다.

## 패러다임 결정 플로우

```
                 문제 읽기
                    │
                    ▼
          제약이 작은가? (n ≤ 수백)
                   ╱  ╲
                예     아니오
                │       │
                ▼       ▼
           완전 탐색   독립 부분 문제로 분할 가능?
                          ╱   ╲
                       예     아니오
                        │       │
                        ▼       ▼
                   분할 정복   지역 최적이 전역 최적?
                                  ╱   ╲
                               예     아니오
                                │       │
                                ▼       ▼
                             그리디   중복 부분 문제 +
                                      최적 부분 구조?
                                            │
                                            ▼
                                           DP
```
**설명** — 새로운 문제를 받으면 위에서부터 질문에 따라 내려간다. (1) 제약이 작으면 $O(n^2)$ 도 허용되므로 완전 탐색을 쓴다. (2) 큰 입력이면 부분 문제가 독립적인지 확인하고, 그렇다면 분할 정복을 쓴다. (3) 그렇지 않으면 그리디 선택이 항상 안전한지(교환 논증으로 증명 가능한지) 확인한다. (4) 마지막으로 중복 부분 문제와 최적 부분 구조가 있다면 DP 를 쓴다. 실전에서는 그리디를 먼저 시도하고 반례가 보이면 DP 로 전환하는 것이 경험칙이다.

### 패러다임 비교
| 측면 | 분할 정복 | 그리디 | 동적 계획법 |
|---|---|---|---|
| 부분 문제 | 독립적 | 해당 없음 | 중복 |
| 방향 | 재귀 | 순차 | 상향식 또는 메모이제이션 |
| 최적 보장 | 문제에 따라 다름 | 그리디 속성 필요 | 항상 |
| 전형 복잡도 | $O(n \log n)$ | $O(n \log n)$ 또는 $O(n)$ | $O(n^2)$ 또는 $O(n^3)$ |
| 대표 예시 | 병합·퀵·이진 탐색 | Dijkstra·Huffman·MST | LCS·편집 거리·배낭·FW |

### 같은 문제, 다른 패러다임
| 문제 | 그리디 | DP | 분할 정복 |
|---|---|---|---|
| 동전 (표준 화폐) | ✓ | ✓ | — |
| 동전 (임의 화폐) | ✗ | ✓ | — |
| 분할 가능 배낭 | ✓ | ✓ | — |
| 0/1 배낭 | ✗ | ✓ | — |
| 최단 경로 (가중치 ≥ 0) | ✓ (Dijkstra) | ✓ | — |
| 최단 경로 (음수 허용) | — | ✓ (FW, 벨만-포드) | — |
| 정렬 | — | — | ✓ |
| k번째 원소 | — | — | ✓ |
| MST | ✓ | — | — |

## 시험 문제 스타일 예시

### 예시 1 — 마스터 정리
$T(n) = 4T(n/2) + n$ 의 복잡도를 구한다.
- $a = 4$, $b = 2$ 이므로 $n^{\log_b a} = n^2$ 이다.
- $f(n) = n = O(n^{2 - 1})$ 이므로 Case 1 에 해당한다.
- 따라서 $T(n) = \Theta(n^2)$ 이다.

### 예시 2 — 그리디 vs DP
동전 {1, 3, 4}, 금액 6 을 비교한다.
- **그리디** — 4 + 1 + 1 로 3개가 된다.
- **DP** — $C[6] = 1 + \min(C[5], C[3], C[2]) = 1 + \min(2, 1, 2) = 2$ 이므로 3 + 3 으로 2개가 된다.
- 이 예시에서 그리디는 최적이 아니다.

### 예시 3 — LCS 테이블
$X$ = "ABCB", $Y$ = "BDCAB" 의 LCS 를 구한다.

| | "" | B | D | C | A | B |
|---|---|---|---|---|---|---|
| "" | 0 | 0 | 0 | 0 | 0 | 0 |
| A | 0 | 0 | 0 | 0 | **1** | 1 |
| B | 0 | **1** | 1 | 1 | 1 | **2** |
| C | 0 | 1 | 1 | **2** | 2 | 2 |
| B | 0 | 1 | 1 | 2 | 2 | **3** |

답은 길이 3, LCS 의 예는 "BCB" 이다.

## 자주 하는 실수

| 실수 | 회피 방법 |
|---|---|
| O, Ω, Θ 를 혼동한다 | O 는 상한, Ω 은 하한, Θ 는 양쪽이 모두 꼭 맞을 때이다. |
| 비다항식 $f(n)$ 에 마스터 정리를 적용한다 | 정규성 조건을 확인하고, 안 되면 반복 대입이나 귀납법을 쓴다. |
| 증명 없이 그리디를 최적이라 주장한다 | 반드시 반례를 먼저 찾아보고, 없으면 교환 논증으로 증명한다. |
| DP 기저 조건을 잘못 잡는다 | $n = 1, 2, 3$ 을 손으로 추적해 검증한다. |
| DP 테이블 인덱스를 헷갈린다 | 0-index 와 1-index 중 무엇을 쓰는지 명시한다. |
| 분할 가능 배낭과 0/1 배낭을 혼동한다 | 분할 가능은 그리디, 0/1 은 DP 이다. |
| 퀵 정렬의 $n^2$ 최악을 간과한다 | 랜덤 피벗이나 Median-of-3 를 사용해야 한다. |
| 플로이드-워셜의 $k$ 루프 순서를 바꾼다 | $k$ 는 반드시 가장 바깥 루프여야 한다. |
| 다익스트라에 음수 간선을 넣는다 | 음수 간선이 있으면 벨만-포드를 써야 한다. |
| 귀납에서 $cn + 1 \leq cn$ 으로 막힌다 | 추측을 $cn - d$ 형태로 강화해 여유를 만든다. |
| 병합 정렬의 공간 $O(n)$ 을 빠뜨린다 | 보조 배열이 필요함을 반드시 명시한다. |
| 이진 탐색의 `left <= right` 등호를 뺀다 | 등호가 없으면 마지막 한 원소를 검사하지 못한다. |
| buildHeap 을 $O(n \log n)$ 이라 쓴다 | 실제로는 $O(n)$ 이며, 리프 절반을 처리하지 않는 타이트 분석 덕분이다. |
| 계수 정렬에서 순방향 순회로 안정성을 잃는다 | 반드시 역순 순회로 배치해야 안정성이 보장된다. |
| LCS 를 부분 문자열과 혼동한다 | LCS 는 연속하지 않아도 되는 부분 수열이다. |
