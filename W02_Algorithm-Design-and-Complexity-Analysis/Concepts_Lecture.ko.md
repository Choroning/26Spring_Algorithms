## 1. 알고리즘 기초

### 1.1 알고리즘이란?

- **알고리즘** 은 문제를 해결하기 위한 단계별 절차 또는 방법이다.
- 우리가 다루는 문제들은 **컴퓨터를 사용하여** 해결한다.
- 알고리즘은 **입력** 을 받아 **출력**(해답)을 생성한다.

```
┌───────┐      ┌─────────────┐      ┌────────┐
│  입력  │ ───► │   알고리즘    │ ───► │   출력  │
└───────┘      └─────────────┘      └────────┘
```

### 1.2 알고리즘의 속성

| 속성 | 설명 |
|:-----|:-----|
| **정확성(Correctness)** | 모든 유효한 입력에 대해 올바른 출력을 생성해야 한다 (같은 입력 → 같은 출력) |
| **실행 가능성(Executability)** | 각 단계가 컴퓨터에서 실행 가능해야 한다 |
| **유한성(Finiteness)** | 유한한 시간 내에 종료해야 한다 |
| **효율성(Efficiency)** | 시간과 공간 면에서 효율적일수록 가치 있다 |

> **핵심:** 특히 **유한성** 은 중요하다 — 무한 루프에 빠지는 프로그램은 알고리즘이 아니다. 예를 들어 `while True: pass`는 알고리즘이 아니다. 반면 유클리드 GCD는 b가 매번 작아지므로 반드시 0에 도달하여 종료한다.

### 1.3 유클리드 GCD 알고리즘

**가장 오래된 알고리즘** 으로, 기원전 ~300년에 만들어졌다.

- **최대공약수(GCD)**: 두 자연수를 모두 나누는 가장 큰 수
- 핵심 통찰: GCD(A, B) = GCD(B, A mod B)

**수학적 근거:**

$A > B$이고, $A = a \cdot G$, $B = b \cdot G$ ($G$ = GCD)일 때:

여기서 $a = A/G$, $b = B/G$는 정수이다 ($G$가 $A$와 $B$를 모두 나누므로). $a$와 $b$는 서로소(coprime)이다 (이들의 GCD는 1).

$$A \mod B = (a \mod b) \cdot G$$

이를 반복 적용하면 $B$가 $0$으로 줄어들고, 그 시점의 $A$가 GCD이다.

> **참고:** A를 B로 나눈 나머지 A mod B도 G의 배수가 된다. 즉, 나머지 연산을 해도 GCD는 보존된다. 그래서 GCD(A, B) = GCD(B, A mod B)가 성립한다.

> **참고:** 증명: $A = B \cdot q + r$ (q는 몫, r은 나머지)이므로 $a \cdot G = b \cdot G \cdot q + r$이다. 따라서 $r = (a - bq) \cdot G = (a \bmod b) \cdot G$이다. 나머지에도 G가 인수로 남아 있으므로 GCD가 보존된다.

**의사코드:**

```
Euclid(a, b)
  입력:  정수 a, b  (a >= b >= 0)
  출력:  GCD(a, b)

  if b == 0
      return a
  return Euclid(b, a mod b)
```

**추적: GCD(24, 14)**

| 호출 | a | b | a mod b |
|:-----|:--|:--|:--------|
| 1 | 24 | 14 | 10 |
| 2 | 14 | 10 | 4 |
| 3 | 10 | 4 | 2 |
| 4 | 4 | 2 | 0 |
| 5 | 2 | 0 | — (return 2) |

**반복(Iterative) 버전:**

```python
def gcd_iter(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

**재귀(Recursive) 버전:**

```python
def gcd_rec(a, b):
    if b == 0:
        return a
    return gcd_rec(b, a % b)
```

두 버전 모두 동일한 결과를 생성한다. 재귀는 **호출 스택(call stack)** 을 사용하며, 각 호출이 새 프레임을 push한다. 반복은 스택 오버헤드를 회피한다.

> **[자료구조]** **재귀(recursion)** 는 함수가 자기 자신을 호출하는 것이다. 모든 재귀 함수는 (1) **기저 조건(base case)**: 재귀를 멈추는 조건과 (2) **재귀 단계(recursive step)**: 문제를 더 작은 크기로 줄여 자기 자신을 호출하는 단계로 구성된다. 위 GCD 함수에서 `b == 0`이 기저 조건이고, `gcd_rec(b, a % b)`가 재귀 단계이다.

**재귀 호출 스택 추적:**

```
gcd_rec(24, 14)
  └─► gcd_rec(14, 10)
        └─► gcd_rec(10, 4)
              └─► gcd_rec(4, 2)
                    └─► gcd_rec(2, 0) → returns 2
                  returns 2
            returns 2
      returns 2
returns 2
```

각 재귀 호출은 스택에 프레임을 **push** 한다. `b == 0`이 되면 값이 다시 **pop** 되어 올라온다.

> **[자료구조]** 스택은 **후입선출(LIFO: Last In, First Out)** 자료구조이다. 프로그램의 함수 호출도 스택 구조로 관리된다 — 함수가 호출되면 push, 반환되면 pop된다. 재귀가 너무 깊으면 **스택 오버플로(Stack Overflow)** 가 발생할 수 있다. Python의 기본 재귀 깊이 제한은 1000이다.

### 1.4 알고리즘의 표현

알고리즘은 단계별 절차이며, 요리 레시피와 유사하다. 세 가지 일반적인 표현 방법이 있다:

| 방법 | 설명 |
|:-----|:-----|
| **자연어** | 각 단계를 말로 풀어서 설명 |
| **의사코드(Pseudocode)** | 프로그래밍 언어와 유사한 구조화된 표기법 (가장 일반적) |
| **순서도(Flowchart)** | 도형과 화살표로 된 시각적 다이어그램 |

**예시 — 최댓값 찾기:**

문제: $n$장의 숫자 카드에서 가장 큰 수를 찾아라.

![카드 비교](../images/ch02_p002_001.png)

**자연어:**

1. 첫 번째 카드의 숫자를 읽고 기억한다
2. 다음 카드를 읽어 기억한 숫자와 비교한다
3. 더 큰 수를 기억에 유지한다
4. 카드가 남아 있으면 2단계로 간다
5. 기억하고 있는 숫자가 최댓값이다

**의사코드:**

```
FindMax(A[], n)
  max = A[0]
  for i = 1 to n-1
      if A[i] > max
          max = A[i]
  return max
```

> **참고:** 의사코드(Pseudocode)는 특정 프로그래밍 언어가 아니라, 알고리즘의 논리를 명확히 표현하기 위한 약속된 표기법이다. CLRS 교재는 자체적인 의사코드 스타일을 사용하며, 시험에서도 이 형식으로 알고리즘을 작성하게 된다.

**순서도:**

```
  ┌─────────┐
  │   시작   │
  └────┬────┘
       ▼
  ┌─────────┐
  │max=A[0] │
  │ i = 1   │
  └────┬────┘
       ▼
  ◇ i < n ? ◇──아니오──► ┌────────────┐
       │               │ return max │
       예               └────────────┘
       ▼
  ◇ A[i]>max? ◇
  │           │
  예         아니오
  ▼           │
┌────────┐    │
│max=A[i]│    │
└───┬────┘    │
    │◄────────┘
    ▼
┌────────┐
│i = i+1 │
└───┬────┘
    │
    └──► (i < n ? 으로 돌아감)
```

순서도는 단순한 알고리즘에는 직관적이지만, 대규모 프로그램에서는 복잡해지므로 현대 교과서에서는 의사코드가 주로 사용된다.

### 1.5 알고리즘 분류

> **참고:** 이것은 학기 전체의 로드맵이다. 아직 이 용어들을 이해할 필요는 없다 — 각각 이후 주차에서 자세히 다룬다.

문제 해결 전략에 따른 6가지 주요 유형:

| 유형 | 핵심 아이디어 |
|:-----|:-------------|
| **분할 정복(Divide and Conquer)** | 부분 문제로 나누기 → 각각 해결 → 결과 합치기. *예: 병합 정렬(Merge Sort) — 배열을 반으로 나누고, 각 반쪽을 정렬한 뒤, 정렬된 반쪽을 합친다.* |
| **탐욕법(Greedy)** | 매 단계에서 지역적으로 최적인 선택. *예: Dijkstra 알고리즘 — 항상 가장 짧은 알려진 경로를 확장하여 그래프의 모든 노드에 도달한다.* |
| **동적 프로그래밍(Dynamic Programming)** | 부분 문제를 풀고, 결과를 저장하여 재사용. *예: 배낭 문제(Knapsack Problem) — 무게와 가치가 있는 물건들이 주어졌을 때, 가방에 넣을 수 있는 가장 가치 있는 조합을 찾는다.* |
| **근사(Approximation)** | NP-난해 문제에 대한 근사적 최적 해. *예: 외판원 문제(TSP) — 정확한 해를 구하기엔 비용이 너무 클 때, 모든 도시를 방문하는 거의 최단 경로를 찾는다.* |
| **백트래킹(Backtracking)** | 모든 가능성을 탐색하되, 불가능한 가지를 가지치기(pruning). *예: N-Queen 문제 — N x N 체스판에 N개의 퀸을 서로 공격하지 않도록 배치한다.* |
| **분기 한정(Branch and Bound)** | 백트래킹과 유사하나, 한계값(bound)을 이용해 더 공격적으로 가지치기. *예: 배낭 문제(정확 해) — 현재 최적보다 나아질 수 없는 가지를 잘라내어 최적 물건 선택을 찾는다.* |

> **핵심:** 이 6가지 분류는 과목 전체를 관통하는 핵심 프레임워크이다. 이 중 **분할 정복, 탐욕법, 동적 프로그래밍** 이 가장 집중적으로 다루는 3대 핵심 전략이다.

**분할 정복 (Divide and Conquer):**

큰 문제를 더 작은 부분 문제로 나누고, 각각을 재귀적으로 해결한 뒤, 결과를 합친다. **장점:** 문제 구조가 자연스럽게 재귀적일 때 효과적이다. 예시: 병합 정렬(Merge Sort), 퀵 정렬(Quick Sort), 이진 탐색(Binary Search).

![병합 정렬 트리](../images/ch02_p020_006.png)

> **[자료구조]** 분할 정복은 항상 3단계로 구성된다:
> 1. **분할(Divide)**: 문제를 더 작은 부분 문제로 나눈다
> 2. **정복(Conquer)**: 각 부분 문제를 재귀적으로 해결한다 (충분히 작으면 직접 해결)
> 3. **결합(Combine)**: 부분 문제들의 해를 합쳐 원래 문제의 해를 만든다
>
> 병합 정렬에서: 분할 = 배열을 반으로 나눔, 정복 = 각 반쪽을 재귀적으로 정렬, 결합 = 두 정렬된 반쪽을 합병(merge).

**탐욕 알고리즘 (Greedy Algorithms):**

매 단계에서 **지금 당장** 가장 좋아 보이는 선택을 한다. 항상 전역 최적해를 보장하는 것은 **아니지만**, 특정 조건 하에서는 보장된다. 예시: 최소 신장 트리 (Kruskal, Prim), Dijkstra 최단 경로, Huffman 코딩.

> **참고:** 탐욕 알고리즘이 최적해를 보장하려면 두 가지 조건이 필요하다:
> 1. **탐욕 선택 속성(Greedy Choice Property)**: 지역적으로 최적인 선택이 전역적 최적해로 이어진다
> 2. **최적 부분 구조(Optimal Substructure)**: 문제의 최적해가 부분 문제의 최적해를 포함한다
>
> 이 조건이 만족되지 않으면 탐욕법은 잘못된 해를 줄 수 있다. 5~6주차에서 엄밀하게 다룬다.

**동적 프로그래밍 (DP):**

문제를 부분 문제로 나누되 (분할 정복처럼), **겹치는 부분 문제의 결과를 저장하고 재사용** 한다.

핵심 조건:
- 겹치는 부분 문제 (Overlapping Subproblems)
- 최적 부분 구조 (Optimal Substructure)

예시: 피보나치 수열, 최장 공통 부분 수열 (LCS), 배낭 문제 (Knapsack).

**DP vs 분할 정복:** 분할 정복의 부분 문제는 **독립적** 이고, DP의 부분 문제는 **겹친다(overlap)**.

> **참고:** 피보나치로 차이를 살펴보면:
> - **분할 정복**: fib(5) → fib(4) + fib(3), fib(4) → fib(3) + fib(2)… 여기서 fib(3)이 **중복 계산** 된다. O(2^n).
> - **DP**: fib(1), fib(2)부터 차례로 계산하여 테이블에 저장하고 재사용한다. 중복 계산이 없어 O(n).

**근사, 백트래킹, 분기 한정:**

| 유형 | 설명 |
|:-----|:-----|
| **근사 알고리즘** | 정확한 해가 너무 비싼 NP-난해 문제용. 최적성을 속도와 교환한다. 예: TSP 근사, 정점 커버 |
| **백트래킹** | 모든 경우의 수를 탐색하되, 불가능한 가지를 가지치기(pruning)한다. 예: N-Queen, 미로 탐색 |
| **분기 한정** | 백트래킹과 유사하나 상한/하한(bound)을 계산하여 더 효과적으로 가지치기한다. 예: TSP, 배낭 문제 (정확 해) |

> **참고:** NP-난해(NP-hard) 문제란, 알려진 모든 알고리즘으로 효율적(다항 시간)으로 풀 수 없는 문제이다. 예를 들어, 외판원 문제(TSP)는 n이 커지면 모든 경우의 수를 확인하는 것이 사실상 불가능하다. 14주차에서 자세히 다룬다.

> **참고:** 백트래킹과 분기 한정의 핵심 차이:
> - **백트래킹**: "이 방향으로 가면 **해가 존재할 수 있는가?**" (가능성 검사)
> - **분기 한정**: "이 방향으로 가면 **현재까지의 최적해보다 나아질 수 있는가?**" (최적성 검사)
>
> 분기 한정은 상한 또는 하한을 계산하여, 현재 최적해보다 나아질 가능성이 없는 가지를 통째로 잘라낸다.

**기타 분류:**
- **문제 영역별:** 정렬, 그래프, 계산 기하학
- **컴퓨팅 환경별:** 병렬, 분산, 양자
- **기타:** 인공지능 / 기계학습 알고리즘

## 2. 시간 복잡도

### 2.1 시간 복잡도란?

**시간 복잡도** 는 입력 크기 $n$에 대한 **기본 연산** 횟수를 함수로 표현한 것이다. **기본 연산** 이란 비교, 읽기, 쓰기, 산술 연산 등 상수 시간이 걸리는 단순한 연산을 말한다.

이 연산 횟수 계산은 RAM(Random Access Machine) 모델을 전제로 한다. RAM 모델에서는 각 기본 연산(산술, 비교, 메모리 접근)이 정확히 1단위 시간이 소요된다.

**예시: $n$장의 카드에서 최댓값 찾기**

- 순차 스캔: 각 카드를 현재 최댓값과 비교
- 비교 횟수: $n - 1$
- 시간 복잡도: $T(n) = n - 1$

> **핵심:** 시간 복잡도를 구할 때 핵심은 **"입력 크기 n에 대해 기본 연산이 몇 번 수행되는가"** 를 세는 것이다. 실제 실행 시간(초)이 아니라 연산 횟수를 추상적으로 계산한다. 이렇게 하면 컴퓨터 성능에 관계없이 알고리즘의 본질적인 효율성을 비교할 수 있다.

### 2.2 실행 시간 예시

**예시 1 — O(1):**

```
sample1(A[], n)
    k = n / 2
    return A[k]
```

나눗셈 1번 + 배열 접근 1번 → $n$에 관계없이 **상수 시간**.

**예시 2 — O(n):**

```
sample2(A[], n)
    sum = 0
    for i = 1 to n
        sum = sum + A[i]
    return sum
```

루프가 $n$번 실행 → **선형 시간**.

**예시 3 — O(n²):**

```
sample3(A[], n)
    sum = 0
    for i = 1 to n
        for j = 1 to n
            sum = sum + A[i] * A[j]
    return sum
```

중첩 루프, 각각 $n$번 실행 → $n \times n$ = **이차 시간(quadratic time)**.

> **참고:** 중첩 루프의 시간 복잡도를 구하는 기본 원칙: **안쪽 루프의 반복 횟수 × 바깥쪽 루프의 반복 횟수** 이다. 여기서는 n × n = n²이다. 루프가 3중이면 O(n³)이 된다.

**예시 4 — O(n²) (삼각형 루프):**

```
sample4(A[], n)
    sum = 0
    for i = 1 to n-1
        for j = i+1 to n
            sum = sum + A[i] * A[j]
    return sum
```

안쪽 루프는 $(n-1) + (n-2) + \cdots + 1 = \frac{n(n-1)}{2}$번 실행 → 여전히 $O(n^2)$.

> **참고:** $1 + 2 + \cdots + (n-1) = \frac{n(n-1)}{2}$는 등차급수의 합 공식이다. Big-O에서는 최고차항만 남기고 계수를 무시하므로 O(n²)이 된다. 이런 "삼각형 형태의 이중 루프"는 매우 자주 등장한다 (예: 중복 검사, 모든 쌍 비교).

**예시 5 — O(n) (재귀):**

```
factorial(n)
    if n == 1 return 1
    return n * factorial(n - 1)
```

재귀 깊이가 $n$ → **선형 시간**.

> **참고:** 재귀 함수의 시간 복잡도는 "재귀 호출이 총 몇 번 발생하는가"로 판단한다. factorial(n)은 정확히 n번 호출되므로 O(n)이다. 각 호출에서 하는 일(곱셈 1번)이 O(1)이므로, 총 시간 복잡도는 O(n) × O(1) = O(n)이다.

### 2.3 복잡도 분석의 유형

| 유형 | 의미 |
|:-----|:-----|
| **최악의 경우(Worst-case)** | 상한 — "어떤 입력이든 이 이상은 걸리지 않는다" |
| **평균의 경우(Average-case)** | 확률 분포(보통 균등 분포)에서의 기대 시간 |
| **최선의 경우(Best-case)** | 가장 빠른 실행 — 최적 알고리즘을 찾는 데 사용 |
| **분할 상환(Amortized)** | 일련의 연산에 대한 연산당 평균 비용 |

분할 상환 분석은 평균 분석과 다르다: 일련의 *연산 시퀀스*에 대한 최악의 총 비용을 구한 뒤, 연산 횟수로 나눈 것이다. 예를 들어, 동적 배열은 가끔 크기를 재할당(비용이 큼)해야 하지만, 대부분의 삽입은 저렴하므로 삽입당 분할 상환 비용은 O(1)이다.

실무에서는 **최악의 경우 분석** 이 가장 일반적으로 사용된다.

> **핵심:** 최악의 경우를 주로 사용하는 이유는 **보장(guarantee)** 을 제공하기 때문이다. "이 알고리즘은 어떤 입력이 와도 절대 n² 시간을 넘지 않는다"는 확실한 약속이 된다. 반면 평균 분석은 "보통은 이 정도"라는 추정일 뿐이다. 예를 들어, 퀵 정렬은 평균 O(n log n)이지만 최악 O(n²)이다.

**통근 시간 비유:**

![서울 지하철](../images/seoul-metro.jpg)

시나리오: 집 → 역 (6분) → 지하철 (20분) → 강의실 (10분)

| 경우 | 시간 | 설명 |
|:-----|:-----|:-----|
| **최선** | 36분 | 열차가 바로 와서 |
| **최악** | 40분 | 방금 떠나서, 4분 대기 |
| **평균** | 38분 | 평균 대기 ~2분 |

알고리즘 분석을 생각하는 방식과 동일하다 — **최악의 경우** 가 보장을 제공한다.

## 3. 점근적 표기법

### 3.1 왜 점근적 표기법인가?

2절에서 우리는 $T(n) = n-1$이나 $T(n) = n(n-1)/2$ 같은 정확한 연산 횟수를 계산했다. 하지만 이런 정확한 횟수는 구현 세부사항에 따라 달라진다 (루프가 0부터 시작하는가 1부터 시작하는가? 마지막 비교를 세는가?). 점근적 표기법은 이런 무관한 세부사항을 제거하고, 진정으로 중요한 것에 집중하게 해준다: $n$이 커질 때 실행 시간이 어떻게 *증가(scale)* 하는가.

![점근적 표기법 그래프](../images/ch03_p003_001.png)

- 시간 복잡도는 $n$의 함수이다 (보통 여러 항이 있는 다항식).
- $n$이 커지면, **지배적인 항(dominant term)** 만 중요하다.
- **점근적 표기법** 은 증가율(growth rate)에 초점을 맞춰 표현을 단순화한다.

| 표기법 | 의미 | 직관적 해석 |
|:-------|:-----|:-----------|
| $O$ (Big-O) | 상한 | "최대 이 정도 속도" |
| $\Omega$ (Big-Omega) | 하한 | "최소 이 정도 속도" |
| $\Theta$ (Theta) | 엄밀한 한계 | 정확한 증가율 |

> **핵심:** "점근적(asymptotic)"이란 "n이 무한히 커질 때의 행동"을 뜻한다. 예를 들어 $3n^2 + 100n + 5000$에서 n이 1,000,000이면 $3n^2 = 3 \times 10^{12}$이고 $100n = 10^8$이므로, $100n$은 $3n^2$에 비해 무시할 수 있을 만큼 작다. 그래서 **최고차항만 남기고** 나머지는 무시한다.

### 3.2 Big-O 표기법

**공식 정의:**

$O(g(n))$ = $g(n)$보다 **최대 같은 속도로** 증가하는 함수들의 집합이다.

$$O(g(n)) = \{ f(n) \mid \exists\, c > 0,\; n_0 \geq 0 \;\text{s.t.}\; \forall\, n \geq n_0,\; f(n) \leq c \cdot g(n) \}$$

- $g(n)$은 $f(n)$의 **점근적 상한(asymptotic upper bound)**
- 관례: 기술적으로는 $f(n) \in O(g(n))$이지만 $f(n) = O(g(n))$이라고 쓴다

> **정의 읽는 법:** "상수 $c > 0$과 $n_0 \geq 0$이 존재하여, 모든 $n \geq n_0$에 대해 $f(n) \leq c \cdot g(n)$이다." 기호 $\exists$는 "존재한다(there exists)"를, $\forall$는 "모든(for all)"을 의미한다.

**직관적 의미:** $f(n) = O(g(n))$은 $f$가 $g$보다 **더 빠르게 증가하지 않는다** 는 뜻이다 (상수 배율 무시).

> **참고:** 정의를 풀어 설명하면: "충분히 큰 n(n ≥ n₀)부터는, f(n)이 항상 c × g(n) 이하"라는 뜻이다. 여기서 c와 n₀는 적절히 찾아야 하는 상수이다. 시험에서는 이 c와 n₀를 구체적으로 찾아 증명하는 문제가 나올 수 있다.

**증명 예시:** $f(n) = 2n^2 - 8n + 3 = O(n^2)$

$n \geq n_0$인 모든 $n$에 대해 $2n^2 - 8n + 3 \leq c \cdot n^2$을 만족하는 상수 $c > 0$과 $n_0 \geq 0$을 찾아야 한다.

$c = 5$를 선택:

$$2n^2 - 8n + 3 \leq 5n^2 \quad \text{(모든 } n \geq 1 \text{에서)}$$

이를 정리하면 $-3n^2 - 8n + 3 \leq 0$이고, 이는 $n \geq 1$에서 성립한다.

> **참고:** 전개 과정: $2n^2 - 8n + 3 \leq 5n^2$의 양변에서 $5n^2$을 빼면 $-3n^2 - 8n + 3 \leq 0$이다. $n \geq 1$이면 $-3n^2 \leq -3$이고 $-8n \leq -8$이므로 왼변은 $-3 - 8 + 3 = -8 \leq 0$이다. n이 커질수록 $-3n^2$이 지배하여 더욱 음수가 되므로 항상 성립한다.

따라서 $c = 5$, $n_0 = 1$로 $f(n) = O(n^2)$. $\square$

> **시험 팁:** c를 찾는 실전 방법 — 다항식 $a_k n^k + a_{k-1} n^{k-1} + \cdots + a_0$에 대해, **모든 계수의 절댓값 합** $|a_k| + |a_{k-1}| + \cdots + |a_0|$을 c로 잡으면 거의 항상 작동한다. 예: $2n^2 - 8n + 3$이면 $c = 2 + 8 + 3 = 13$, $n_0 = 1$.

**$O(n^2)$에 포함되는 함수:**
- $3n^2 + 2n$ ✓, $7n^2 - 100n$ ✓, $n \log n + 5n$ ✓, $3n$ ✓

**$O(n^2)$에 속하지 않는 함수:**
- $n^3$ ✗, $3^n$ ✗

**모범 사례:** 가능한 한 **촘촘하게(tight)** 쓸 것. $n \log n + 5n = O(n \log n)$이지, $O(n^2)$이라 쓰지 않는다. 느슨한 한계는 **정보의 손실** 을 의미한다.

> **참고:** "이 알고리즘은 O(n²)이다"와 "이 알고리즘은 O(n log n)이다"는 둘 다 참일 수 있지만, O(n log n)이 더 유용한 정보이다. "이 사과는 100원 이하이다"보다 "이 사과는 50원이다"가 더 유용한 것과 같다.

**더 많은 Big-O 예시:**

| 수식 | Big-O | 이유 |
|:-----|:------|:-----|
| $3n + 2$ | $O(n)$ | $3n+2 \leq 4n$ ($n \geq 2$일 때) |
| $100n + 6$ | $O(n)$ | $100n+6 \leq 101n$ ($n \geq 10$일 때) |
| $10n^2 + 4n + 2$ | $O(n^2)$ | $10n^2+4n+2 \leq 11n^2$ ($n \geq 5$일 때) |
| $6 \cdot 2^n + n^2$ | $O(2^n)$ | $6 \cdot 2^n + n^2 \leq 7 \cdot 2^n$ ($n \geq 4$일 때) |

### 3.3 Big-Omega 표기법

**공식 정의:**

$\Omega(g(n))$ = $g(n)$보다 **최소 같은 속도로** 증가하는 함수들의 집합이다.

$$\Omega(g(n)) = \{ f(n) \mid \exists\, c > 0,\; n_0 \geq 0 \;\text{s.t.}\; \forall\, n \geq n_0,\; f(n) \geq c \cdot g(n) \}$$

- $g(n)$은 $f(n)$의 **점근적 하한(asymptotic lower bound)**
- Big-O와 대칭적

**직관적 의미:** $f(n) = \Omega(g(n))$은 $f$가 $g$보다 **더 느리게 증가하지 않는다** 는 뜻이다.

**예시:**

| 수식 | Big-Omega | 이유 |
|:-----|:----------|:-----|
| $3n + 2$ | $\Omega(n)$ | $3n+2 \geq 3n$ ($n \geq 1$일 때) |
| $100n + 6$ | $\Omega(n)$ | $100n+6 \geq 100n$ ($n \geq 1$일 때) |
| $10n^2 + 4n + 2$ | $\Omega(n^2)$ | $10n^2+4n+2 \geq n^2$ ($n \geq 1$일 때) |
| $6 \cdot 2^n + n^2$ | $\Omega(2^n)$ | $6 \cdot 2^n + n^2 \geq 2^n$ ($n \geq 1$일 때) |

**모범 사례:** 가능한 한 촘촘하게 쓸 것. $n \log n + 5n = \Omega(n \log n)$이지, $\Omega(n)$이라 쓰지 않는다.

**증명 예시:** $10n^2 + 4n + 2 = \Omega(n^2)$

$c = 10$, $n_0 = 1$로 선택한다. 그러면 모든 $n \geq 1$에 대해 $10n^2 + 4n + 2 \geq 10n^2 \geq c \cdot n^2$이다. $\square$

### 3.4 Theta 표기법

**공식 정의:**

$\Theta(g(n))$ = $g(n)$과 **같은 속도로** 증가하는 함수들의 집합이다.

$$\Theta(g(n)) = O(g(n)) \;\cap\; \Omega(g(n))$$

$$\Theta(g(n)) = \{ f(n) \mid \exists\, c_1, c_2 > 0,\; n_0 \geq 0 \;\text{s.t.}\; \forall\, n \geq n_0,\; c_2 \cdot g(n) \leq f(n) \leq c_1 \cdot g(n) \}$$

**직관적 의미:** $f(n) = \Theta(g(n))$은 $f$와 $g$가 (상수 배율 차이를 제외하면) **같은 속도로** 증가한다는 뜻이다.

> **핵심:** Θ는 가장 정확한 표기법이다:
> - O(n²): "이 알고리즘은 **최대** n² 수준이다" (더 빠를 수도 있다)
> - Ω(n²): "이 알고리즘은 **최소** n² 수준이다" (더 느릴 수도 있다)
> - Θ(n²): "이 알고리즘은 **정확히** n² 수준이다"

**예시:**

| 수식 | Theta |
|:-----|:------|
| $3n + 2$ | $\Theta(n)$ — $3n \leq 3n+2 \leq 4n$ ($n \geq 2$일 때) |
| $10n^2 + 4n + 2$ | $\Theta(n^2)$ |
| $6 \cdot 2^n + n^2$ | $\Theta(2^n)$ |

**반례:** $2n^2 + 3n + 5 \neq \Theta(n^3)$ 이고 $2n^2 + 3n + 5 \neq \Theta(n)$ — Theta는 상한과 하한이 **모두** 일치해야 한다.

**증명 예시:** $3n + 2 = \Theta(n)$

상한: $3n + 2 \leq 5n$ ($n \geq 1$일 때, $c_2 = 5$). 하한: $3n + 2 \geq 3n$ ($n \geq 0$일 때, $c_1 = 3$). 따라서 $c_1 = 3$, $c_2 = 5$, $n_0 = 1$. $\square$

### 3.5 일반적인 복잡도 분류

| 표기법 | 이름 | 예시 |
|:-------|:-----|:-----|
| $O(1)$ | 상수 | 배열 인덱스 접근 |
| $O(\log n)$ | 로그 | 이진 탐색 |
| $O(n)$ | 선형 | 선형 탐색 |
| $O(n \log n)$ | 로그-선형 | 병합 정렬 |
| $O(n^2)$ | 이차 | 버블 정렬 |
| $O(n^3)$ | 삼차 | 행렬 곱셈 (단순) |
| $O(2^n)$ | 지수 | 부분집합 완전 탐색 |

**계층구조:** $O(1) \subset O(\log n) \subset O(n) \subset O(n \log n) \subset O(n^2) \subset O(n^3) \subset O(2^n)$

> **핵심:** 실전에서의 감각:
> - **O(1)**: 해시 테이블 조회 — 즉시
> - **O(log n)**: n=10억이어도 ~30번 — 매우 빠르다
> - **O(n)**: n=10억이면 10억번 — 보통 수 초
> - **O(n log n)**: 대부분의 정렬 — 실용적 한계
> - **O(n²)**: n=10만이면 100억번 — 느리기 시작한다
> - **O(2^n)**: n=40이면 1조번 — 사실상 불가능하다

**증가율 비교:**

| $n$ | $\log n$ | $n$ | $n \log n$ | $n^2$ | $n^3$ | $2^n$ |
|:----|:---------|:----|:-----------|:------|:------|:------|
| 10 | 3.3 | 10 | 33 | 100 | 1,000 | 1,024 |
| 100 | 6.6 | 100 | 664 | 10,000 | $10^6$ | $10^{30}$ |
| 1,000 | 10 | 1,000 | 10,000 | $10^6$ | $10^9$ | $10^{301}$ |
| $10^6$ | 20 | $10^6$ | $2 \times 10^7$ | $10^{12}$ | $10^{18}$ | — |

다항식과 지수의 격차는 **어마어마** 하다.

> **참고:** $2^{100} \approx 10^{30}$인데, 관측 가능한 우주에 있는 원자의 수가 대략 $10^{80}$이다. n=100만 되어도 $2^n$은 상상할 수 없는 수준이다. 이것이 "지수 시간 알고리즘은 실용적이지 않다"는 말의 의미이다.

> **참고:** 위 표에서 $\log n$은 $\log_2 n$이다. CS에서 별도 표기 없이 log라 쓰면 보통 **밑(base)이 2** 이다. Big-O에서는 밑이 무엇이든 상관없다 — $\log_a n = \frac{\log_b n}{\log_b a}$이므로 밑의 변환은 상수 배율일 뿐이고, Big-O에서 상수는 무시되기 때문이다.

### 3.6 효율적인 알고리즘이 중요한 이유

**10억 개의 숫자 정렬:**

| 알고리즘 | $n$ = 1,000 | $n$ = 1,000,000 | $n$ = 1,000,000,000 |
|:---------|:------------|:----------------|:--------------------|
| $O(n^2)$ (PC) | < 1초 | ~2시간 | **~300년** |
| $O(n^2)$ (슈퍼컴퓨터) | < 1초 | ~1초 | ~1주일 |
| $O(n \log n)$ (PC) | < 1초 | < 1초 | **~5분** |
| $O(n \log n)$ (슈퍼컴퓨터) | < 1초 | < 1초 | < 1초 |

> *PC는 초당 ~10^8 연산, 슈퍼컴퓨터는 초당 ~10^12 연산을 수행한다고 가정한다.*

> 효율적인 알고리즘은 **슈퍼컴퓨터보다 가치 있다**.
>
> 더 좋은 하드웨어에 투자하는 것보다 더 나은 알고리즘에 투자하는 것이 훨씬 경제적이다.

> **핵심:** O(n²)을 슈퍼컴퓨터에서 돌리는 것보다 O(n log n)을 일반 PC에서 돌리는 것이 **수만 배 빠르다**. 하드웨어는 상수 배율의 개선만 줄 수 있지만, 알고리즘 개선은 **차수(order)** 자체를 바꾼다.

## 4. 점화식

### 4.1 점화식이란?

재귀 알고리즘을 분석할 때는 2절에서처럼 단순히 루프 반복 횟수를 셀 수 없다. 대신, 실행 시간 $T(n)$을 더 작은 입력에서의 $T$ 값으로 표현하게 되는데 — 이것이 점화식이다. 우리의 목표는 점화식을 *풀어서* $O(n \log n)$ 같은 닫힌 형태의 표현을 얻는 것이다.

**점화식(Recurrence Relation)** 은 함수를 **더 작은 입력에서의 값** 으로 표현한 식이다.

예시:
- $a_n = a_{n-1} + 2$
- $f(n) = n \cdot f(n-1)$ — 팩토리얼
- $f(n) = f(n-1) + f(n-2)$ — 피보나치
- $f(n) = f(n/2) + n$ — 이진 감소

많은 재귀 알고리즘이 실행 시간에 대한 점화식을 자연스럽게 생성한다.

> **참고:** 점화식은 "재귀적으로 정의된 함수"라고 생각하면 된다. 재귀 알고리즘은 직접 T(n) = … 형태로 표현하기 어렵고, 대신 "T(n)을 T(n/2)나 T(n-1) 등으로 표현"하게 된다. 이렇게 세운 점화식을 풀어서 닫힌 형태(closed form)로 만드는 것이 목표이다.

**병합 정렬의 점화식:**

$$T(n) = 2T(n/2) + n$$

- $2T(n/2)$: 반쪽에 대한 두 번의 재귀 호출
- $n$: 합병 오버헤드 (선형 스캔)
- $T(1) = 1$

> **[자료구조]** 병합 정렬의 동작:
> 1. 배열 [5, 2, 4, 1, 3]을 반으로 나눈다: [5, 2] / [4, 1, 3]
> 2. 각 반쪽을 재귀적으로 정렬한다: [2, 5] / [1, 3, 4]
> 3. 두 정렬된 배열을 합병한다: [1, 2, 3, 4, 5]
>
> 합병(merge) 과정은 두 배열의 앞에서부터 비교하며 작은 것을 꺼내므로 O(n)이다. 점화식의 $n$이 바로 이 합병 비용이다.

**점화식을 풀기 위한 세 가지 방법:**

| 방법 | 아이디어 |
|:-----|:---------|
| **반복 대입법(Repeated Substitution)** | 점화식을 단계별로 전개하여 패턴을 찾는다 |
| **추측 후 검증(Guess and Verify)** | 형태를 추측한 뒤, 수학적 귀납법으로 증명한다 |
| **마스터 정리(Master Theorem)** | 특정 형태의 점화식에 공식을 직접 적용한다 |

### 4.2 방법 1: 반복 대입법

**예시:** $T(n) = T(n-1) + c$, $T(1) \leq c$

$$
\begin{aligned}
T(n) &= T(n-1) + c \\
     &= (T(n-2) + c) + c = T(n-2) + 2c \\
     &= T(n-3) + 3c \\
     &\;\;\vdots \\
     &= T(1) + (n-1)c \\
     &\leq c + (n-1)c = cn = O(n)
\end{aligned}
$$

이것은 **팩토리얼** 의 점화식이며, 선형 시간이다.

> **참고:** 반복 대입법의 핵심 전략: T(n)에 점화식을 반복해서 대입하면서 패턴을 찾고, 그 패턴을 일반화한다. 위 예시에서 패턴은 "k번 대입하면 T(n-k) + kc"이다. k = n-1을 대입하면 T(1) + (n-1)c가 된다.

**병합 정렬에 적용:** $T(n) = 2T(n/2) + n$, $T(1) = 1$

$n = 2^k$라 가정한다:

$$
\begin{aligned}
T(n) &= 2T(n/2) + n \\
     &= 2(2T(n/4) + n/2) + n = 4T(n/4) + 2n \\
     &= 8T(n/8) + 3n \\
     &\;\;\vdots \\
     &= 2^k T(n/2^k) + kn \\
     &= n \cdot T(1) + n \log_2 n \\
     &= n + n \log n = O(n \log n)
\end{aligned}
$$

> **참고:** 전개 과정의 패턴:
> - 1번 대입: $2T(n/2) + n$ → 계수 2, 문제 크기 n/2
> - 2번 대입: $4T(n/4) + 2n$ → 계수 4, 문제 크기 n/4
> - k번 대입: $2^k T(n/2^k) + kn$
>
> $n = 2^k$이면 $k = \log_2 n$이고, $2^k = n$이므로 $n \cdot T(1) + n\log n$이 된다. **$n = 2^k$라 가정하는 것은 계산을 깔끔하게 하기 위한 것** 이며, 일반적인 n에 대해서도 동일한 결과가 성립한다.

### 4.3 방법 2: 추측 후 검증 (귀납법)

**점화식:** $T(n) = 2T(n/2) + n$

**추측:** $T(n) = O(n \log n)$, 즉 어떤 상수 $c > 0$에 대해 $T(n) \leq cn \log n$.

> **[이산수학]** 수학적 귀납법의 3단계:
> 1. **기저 사례(Base Case)**: 가장 작은 경우에 명제가 참임을 보인다
> 2. **귀납 가정(Inductive Hypothesis)**: n = k일 때 명제가 참이라고 가정한다
> 3. **귀납 단계(Inductive Step)**: n = k+1일 때도 참임을 보인다
>
> 이 세 단계를 완성하면, 모든 자연수에 대해 명제가 참이 된다.

**수학적 귀납법으로 증명:**

*기저 사례:* $T(2) = 2T(1) + 2 = 2(1) + 2 = 4$이다. $4 \leq c \cdot 2 \cdot \log_2(2) = 2c$가 필요하므로, $c \geq 2$이면 충분하다.

*귀납 가정:* $T(n/2) \leq c(n/2)\log(n/2)$라 가정한다.

*귀납 단계:*

$$
\begin{aligned}
T(n) &= 2T(n/2) + n \\
     &\leq 2 \cdot c(n/2)\log(n/2) + n \\
     &= cn\log n - cn\log 2 + n \\
     &= cn\log n + (-c\log 2 + 1)n \\
     &\leq cn\log n \quad \text{(} c \geq 1/\log 2 \text{일 때)}
\end{aligned}
$$

따라서 $T(n) = O(n \log n)$. $\square$

**흔한 함정:**

**주의:** 가정에서의 상수 $c$는 결론에서의 $c$와 **동일** 해야 한다.

잘못된 증명 시도 — $T(n) = 2T(n/2) + 1$, 추측 $T(n) = O(n)$:

$$T(n) \leq 2 \cdot c(n/2) + 1 = cn + 1$$

이것은 $cn + 1$이지, $\leq cn$이 **아니다**. 증명 실패!

수정 — 추측을 $T(n) \leq cn - 2$로 강화한다:

$$T(n) \leq 2(c(n/2) - 2) + 1 = cn - 4 + 1 = cn - 3 \leq cn - 2 \;\checkmark$$

> **시험 팁:** 이 함정은 시험에서 자주 나온다! 귀납법 증명에서 가장 흔한 실수는 "cn + (상수)"를 얻고서 "O(n)이니까 됐다"고 하는 것이다. 귀납법에서는 **정확히 같은 형태** ($\leq cn$)를 보여야 한다. 해결법은 추측을 $cn - d$ (적절한 상수 d)로 "약간 줄여서" 여유를 만드는 것이다.

> **참고:** 상수 $d$를 체계적으로 결정하는 방법: 원래 증명에서 $T(n) \leq cn + 1$이 나왔는데 $\leq cn$이 필요하다. 추측을 $cn - d$로 놓고 귀납 단계를 전개하면: $T(n) \leq 2(c(n/2) - d) + 1 = cn - 2d + 1$이다. 이것이 $\leq cn - d$이려면 $-2d + 1 \leq -d$, 즉 $d \geq 1$이면 충분하다. 핵심은 "초과분을 흡수할 수 있는 여유(slack)를 만드는 것"이다.

### 4.4 방법 3: 마스터 정리

다음 형태의 점화식에 대해:

$$T(n) = aT(n/b) + f(n) \quad \text{(단, } a \geq 1, \; b > 1\text{)}$$

$h(n) = n^{\log_b a}$로 놓는다. $f(n)$과 $h(n)$을 비교한다:

| 경우 | 조건 | 결과 |
|:-----|:-----|:-----|
| **Case 1** | $f(n) = O(n^{\log_b a - \varepsilon})$ (어떤 $\varepsilon > 0$) | $T(n) = \Theta(n^{\log_b a})$ |
| **Case 2** | $f(n) = \Theta(n^{\log_b a})$ | $T(n) = \Theta(n^{\log_b a} \log n)$ |
| **Case 3** | $f(n) = \Omega(n^{\log_b a + \varepsilon})$ (어떤 $\varepsilon > 0$), 그리고 $af(n/b) \leq cf(n)$ (어떤 $c < 1$) | $T(n) = \Theta(f(n))$ |

> **참고:** "다항식적으로(polynomially) 더 작다/크다"의 의미: $f(n) = O(n^{\log_b a - \varepsilon})$에서 $\varepsilon > 0$이 존재해야 한다는 것은, f(n)이 $n^{\log_b a}$보다 **$n^{\varepsilon}$ 배만큼** 작아야 한다는 뜻이다. 예를 들어 $n^{\log_b a} = n$이고 $f(n) = n / \log n$이면, 이것은 n보다 작지만 "다항식적으로" 작지는 않다. 이런 경우 마스터 정리를 직접 적용할 수 없다.

> **핵심:** 마스터 정리는 **분할 정복 알고리즘의 점화식을 한 번에 풀어주는 강력한 도구** 이다. 핵심은 간단하다:
> - $a$: 재귀 호출 횟수
> - $b$: 문제가 줄어드는 비율
> - $f(n)$: 분할과 결합에 드는 비용
> - $n^{\log_b a}$: 재귀 트리의 리프(leaf) 수준 비용
>
> 3가지 Case는 "리프 비용이 이기는가(Case 1), 루트 비용이 이기는가(Case 3), 아니면 비기는가(Case 2)"를 판단한다.

**직관:**

$$T(n) = aT(n/b) + f(n)$$

재귀 트리의 각 레벨이 작업을 수행한다. **누가 지배하는가?**

![레벨별 비용이 있는 재귀 트리](../images/recursion-tree-costs.png)

| 시나리오 | 승자 | 결과 |
|:---------|:-----|:-----|
| 리프가 지배 | 리프 작업 승 | $T(n) = \Theta(n^{\log_b a})$ |
| 루트가 지배 | 루트 작업 승 | $T(n) = \Theta(f(n))$ |
| 균형 | 레벨당 동일 | $T(n) = \Theta(n^{\log_b a} \log n)$ |

트리의 높이는 $\log_b n$이고, 각 레벨의 총 비용이 합산된다.

> **[자료구조]** 재귀 트리(recursion tree)는 트리의 응용이다. 루트는 원래 문제, 각 내부 노드는 재귀 호출, 리프는 기저 사례이다. 트리의 높이가 $\log_b n$인 이유는, 각 레벨에서 문제 크기가 $1/b$로 줄어들기 때문이다 (n → n/b → n/b² → … → 1).

**예시:**

**예시 1:** $T(n) = 2T(n/3) + c$

- $a = 2$, $b = 3$, $h(n) = n^{\log_3 2} \approx n^{0.63}$, $f(n) = c = O(1)$
- $f(n)$이 다항식적으로 더 작다 → **Case 1**
- $T(n) = \Theta(n^{\log_3 2})$

**예시 2:** $T(n) = 2T(n/4) + n$

- $a = 2$, $b = 4$, $h(n) = n^{\log_4 2} = n^{0.5} = \sqrt{n}$, $f(n) = n$
- $f(n)$이 다항식적으로 더 크다 → **Case 3**
- $T(n) = \Theta(n)$

**예시 3:** $T(n) = 2T(n/2) + n$

- $a = 2$, $b = 2$, $h(n) = n^{\log_2 2} = n$, $f(n) = n$
- $f(n) = \Theta(h(n))$ → **Case 2**
- $T(n) = \Theta(n \log n)$

> **참고:** 마스터 정리 적용 순서:
> 1. a, b, f(n)을 식별한다
> 2. $n^{\log_b a}$를 계산한다
> 3. f(n)과 $n^{\log_b a}$를 비교한다
> 4. 해당하는 Case를 적용한다
>
> 예시 3은 바로 **병합 정렬** 의 점화식이다! 마스터 정리를 쓰면 반복 대입이나 귀납법 없이 바로 O(n log n)을 얻을 수 있다.

> **참고:** 정칙 조건(regularity condition)은 $f(n)$이 "잘 행동하는(well-behaved)" 함수임을 보장한다 — 구체적으로, 재귀 트리의 각 레벨에서 수행되는 작업이 기하급수적으로 감소한다는 것이다. 대부분의 흔한 함수(다항식, 지수 함수)는 이 조건을 자동으로 만족한다.

> **시험 팁:** 마스터 정리의 **Case 3은 출제 빈도가 낮다**. 정칙 조건($af(n/b) \leq cf(n)$)의 검증이 까다롭기 때문이다. 대부분의 시험 문제는 Case 1 또는 Case 2로 해결된다. 우선 $n^{\log_b a}$를 계산하고 f(n)과 비교하는 데 집중하는 것이 효과적이다.

| 복잡도 분석 유형 | **최악**(가장 일반적 — 보장 제공), 평균, 최선, 분할 상환 |
| Big-O ($O$) | 점근적 **상한**; $f(n) \leq c \cdot g(n)$ for all $n \geq n_0$; "최대 이 정도" |
| Big-Omega ($\Omega$) | 점근적 **하한**; $f(n) \geq c \cdot g(n)$ for all $n \geq n_0$; "최소 이 정도" |
| Theta ($\Theta$) | **엄밀한 한계**; $O \cap \Omega$; "정확히 이 수준" |
| 촘촘하게 쓸 것 | $O(n \log n)$이 $O(n^2)$보다 유용한 정보; 느슨한 한계는 정보 손실 |
| 복잡도 계층 | $O(1) \subset O(\log n) \subset O(n) \subset O(n \log n) \subset O(n^2) \subset O(n^3) \subset O(2^n)$ |
| 알고리즘 > 하드웨어 | O(n log n)을 PC에서 돌리는 것이 O(n²)를 슈퍼컴퓨터에서 돌리는 것보다 빠르다 |
| 점화식 | 함수를 더 작은 입력에서의 값으로 표현; 재귀 알고리즘의 시간 복잡도 분석에 사용 |
| 반복 대입법 | 점화식을 반복 전개하여 패턴을 찾고 일반화 |
| 추측 후 검증 | 형태를 추측한 뒤 수학적 귀납법으로 증명; **상수 c가 동일해야 하는 함정** 주의 |
| 마스터 정리 | $T(n) = aT(n/b) + f(n)$에 적용; $n^{\log_b a}$와 $f(n)$ 비교로 3가지 Case 판단 |
| 교재 범위 | CLRS 1~3장 |

## 점검 문제

1. **유클리드 GCD:** GCD(48, 18)을 단계별로 추적하라. 몇 번의 반복이 필요한가?
2. **연산 횟수 세기:** `for i in range(n): for j in range(i, n): count += 1` 코드에서 `count += 1`은 몇 번 실행되는가? n의 함수로 표현하라.
3. **Big-O 증명:** 적절한 c와 n₀를 찾아 5n³ + 2n² + 7 = O(n³)을 증명하라.
4. **Big-Omega vs Big-O:** n² = O(n³)은 참인가? n² = Ω(n³)은 참인가? 이유를 설명하라.
5. **Theta:** f(n) = Θ(g(n))이면 f와 g의 관계에 대해 무엇을 알 수 있는가? f(n) = Θ(n²)과 f(n) = O(n³)이 동시에 참일 수 있는가?
6. **마스터 정리:** T(n) = 4T(n/2) + n에 마스터 정리를 적용하라. a, b, f(n)을 식별하고, n^(log_b a)를 계산하고, 경우를 판정하고, T(n)을 구하라.
7. **추측 후 검증:** T(n) = T(n/2) + 1에 대해 T(n) = O(log n)을 추측하고 귀납법으로 검증하라. 기저 사례를 잊지 말 것!
8. **왜 중요한가?** O(n log n) 알고리즘을 실행하는 PC vs O(n²) 알고리즘을 실행하는 슈퍼컴퓨터 — 어떤 입력 크기에서 PC가 이기는가?
