## 1. 기초 (W01)

### 1.1 알고리즘의 정의와 성질

**알고리즘**은 입력이 주어지면 유한 단계 내에 올바른 출력을 내놓는, 체계적이고 명확한 풀이 절차다.

| 성질 | 의미 |
|------|------|
| **정확성(Correctness)** | 모든 유효 입력에 대해 올바른 출력을 생성 |
| **실행 가능성(Executability)** | 각 단계가 컴퓨터에서 실행 가능 |
| **유한성(Finiteness)** | 유한 단계 내 종료 |
| **효율성(Efficiency)** | 시간·공간 사용이 최소 |
| **명확성(Unambiguity)** | 각 단계가 모호하지 않게 정의 ("상황에 맞게 처리" 같은 표현은 알고리즘 아님) |

> **알고리즘 + 자료구조 = 프로그램** — 니클라우스 비르트

### 1.2 표현 방식

자연어 / **의사코드**(표준) / 순서도. 시험에서는 CLRS 스타일 의사코드로 쓴다.

### 1.3 유클리드 GCD (가장 오래된 알고리즘, ~기원전 300년)

**핵심 항등식:** `gcd(a, b) = gcd(b, a mod b)`; 기저 `gcd(a, 0) = a`.

```python
def gcd(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

최악의 반복 횟수 ≈ `log(min(a,b))` (연속한 피보나치 수일 때 최악).

### 1.4 고전 문제와 log₂(n) 테마

| 문제 | 전략 | 최악 (n 입력) |
|------|------|----------------|
| 최댓값 찾기 | 순차 탐색 | 비교 `n − 1`회, O(n) |
| 이진 탐색 (정렬된 배열) | 분할 정복 | 비교 `log₂(n)`회, O(log n) |
| 동전 거스름돈 (표준 화폐) | 그리디 (큰 것부터) | 화폐 종류 k개면 O(k) |
| 오일러 회로 | DFS, 다리 회피 | 홀수 차수 정점이 0개 또는 2개여야 |
| 미로 (벽 연결) | 오른손 법칙 (DFS 변종) | 항상 출구 도달 |
| 가짜 동전 (1,024개) | 반으로 나눠 측정 | **10회 측정** (log₂ 1024) |
| 독이 든 포도주 (n개 항아리) | 이진 인코딩 | `⌈log₂(n)⌉`명 시종 |

> **log₂(n) 모티프:** 매 단계 문제를 절반으로 줄이면 로그 성능. 이진 탐색, 가짜 동전, 독이 든 포도주 — 그리고 모든 효율적 D&C 알고리즘의 토대.

### 1.5 오일러 회로 조건

연결 무방향 그래프에서 **오일러 회로**는 **모든 정점의 차수가 짝수**일 때 존재한다. **오일러 경로**(회로 아닐 수 있음)는 홀수 차수 정점이 **정확히 0개 또는 2개**일 때 존재한다. 주의: **다리**(제거 시 그래프가 분리되는 간선)를 너무 일찍 건너지 말 것.

### 1.6 독이 든 포도주 — 이진 인코딩

항아리 `k`에 이진수 ID `k`를 부여. 시종 `i`는 비트 `i`가 1인 모든 항아리를 마심. 일주일 후 죽은 시종의 패턴 = 독 든 항아리의 이진수. `⌈log₂(n)⌉`명 필요. 각 시종 = 1비트 정보; n개 구분에는 `log₂(n)` 비트 필요.

## 2. 복잡도 분석 (W02)

### 2.1 시간 복잡도 기본

`T(n)` = 입력 크기 `n`에 대한 **기본 연산**(비교, 대입, 사칙연산, 배열 접근) 횟수. **RAM 모델** 가정(기본 연산 1회 = 시간 1).

### 2.2 분석 유형 네 가지

| 유형 | 의미 |
|------|------|
| **최악(Worst-case)** | 상한 — 보장 (가장 자주 사용) |
| **평균(Average-case)** | 확률 분포 하의 기댓값 |
| **최선(Best-case)** | 가장 빠른 실행 |
| **분할 상환(Amortized)** | 연산 *시퀀스*에 대한 평균 비용 |

### 2.3 점근적 표기 — 형식적 정의

| 표기 | 집합적 정의 |
|------|-------------|
| **O(g(n))** | `{ f : ∃ c > 0, n₀ ≥ 0 s.t. ∀ n ≥ n₀, f(n) ≤ c·g(n) }` — 상한 |
| **Ω(g(n))** | `{ f : ∃ c > 0, n₀ ≥ 0 s.t. ∀ n ≥ n₀, f(n) ≥ c·g(n) }` — 하한 |
| **Θ(g(n))** | `O(g(n)) ∩ Ω(g(n))` — 꼭 맞는 한계 |

**`f(n) = O(g(n))` 증명 레시피:** 다항식 `aₖnᵏ + ... + a₀`라면 `c = |aₖ| + ... + |a₀|`, `n₀ = 1`. 예: `2n² − 8n + 3 ≤ 13n²` (`n ≥ 1`) → `c = 13, n₀ = 1`.

> **원칙:** 경계는 언제나 **최대한 타이트하게**. `n log n + 5n = O(n log n)`이지 `O(n²)`이 아니다.

### 2.4 복잡도 계층

```
O(1) ⊂ O(log n) ⊂ O(n) ⊂ O(n log n) ⊂ O(n²) ⊂ O(n³) ⊂ O(2ⁿ) ⊂ O(n!)
```

| 클래스 | 이름 | 예시 | 체감 (n=10⁶) |
|--------|------|------|---------------|
| O(1) | 상수 | 배열 접근 | 즉시 |
| O(log n) | 로그 | 이진 탐색 | ~20 연산 |
| O(n) | 선형 | 선형 스캔 | ~몇 초 |
| O(n log n) | 선형로그 | 병합/힙 정렬 | 실용적 상한 |
| O(n²) | 이차 | 버블 정렬 | 분~시간 |
| O(n³) | 삼차 | 플로이드-워셜 | 작은 n만 가능 |
| O(2ⁿ) | 지수 | 순진한 피보나치 | n≈40 이상 불가 |

**Big-O에서 로그 밑이 상관없는 이유:** `logₐ n = (logᵦ n) / (logᵦ a)`이고 `1/logᵦ a`는 상수. 상수는 O()에 흡수.

### 2.5 점화식 — 세 가지 방법

**방법 1: 반복 대입(Repeated Substitution).** 패턴이 보일 때까지 점화식을 전개한 뒤 기저 대입.

예: `T(n) = 2T(n/2) + n`, `T(1) = 1`. `n = 2ᵏ`로 가정:
```
T(n) = 2T(n/2) + n
     = 4T(n/4) + 2n
     = 2ᵏ·T(n/2ᵏ) + k·n
     = n·T(1) + n·log₂ n
     = Θ(n log n)
```

**방법 2: 추측 후 증명(Guess and Verify).** 형태를 추측해서 수학적 귀납법으로 증명.

> **함정:** 귀납 가정의 상수 `c`와 결론의 `c`는 **정확히 같은** 값이어야 한다. `T(n) ≤ cn + 1`은 `≤ cn`이 **아니다**. 가정을 `cn − d`로 강화해 여유를 만들 것.

**방법 3: 마스터 정리(Master Theorem).** `T(n) = a·T(n/b) + f(n)`(`a ≥ 1, b > 1`), `h(n) = n^(logᵦ a)`라 할 때:

| Case | 조건 | 결과 |
|------|------|------|
| **Case 1** | `f(n) = O(n^(logᵦ a − ε))` (`ε > 0`) | `T(n) = Θ(n^(logᵦ a))` |
| **Case 2** | `f(n) = Θ(n^(logᵦ a))` | `T(n) = Θ(n^(logᵦ a) · log n)` |
| **Case 3** | `f(n) = Ω(n^(logᵦ a + ε))` (`ε > 0`), **그리고** 정규성 조건 `a·f(n/b) ≤ c·f(n)` (`c < 1`) | `T(n) = Θ(f(n))` |

**마스터 정리 적용 레시피:** (1) `a, b, f(n)` 식별. (2) `n^(logᵦ a)` 계산. (3) `f(n)`과 `n^(logᵦ a)` 비교. (4) 해당 Case 적용.

**마스터 정리 예제 모음:**

| 점화식 | `a, b, n^(logᵦ a)` | `f(n)` | Case | 결과 |
|--------|-------------------|--------|------|------|
| `T(n) = 2T(n/2) + n` | 2, 2, `n` | `n` | Case 2 | `Θ(n log n)` — 병합 정렬 |
| `T(n) = T(n/2) + 1` | 1, 2, `1` | `1` | Case 2 | `Θ(log n)` — 이진 탐색 |
| `T(n) = 2T(n/2) + 1` | 2, 2, `n` | `1` | Case 1 | `Θ(n)` |
| `T(n) = 4T(n/2) + n` | 4, 2, `n²` | `n` | Case 1 | `Θ(n²)` |
| `T(n) = 4T(n/2) + n²` | 4, 2, `n²` | `n²` | Case 2 | `Θ(n² log n)` |
| `T(n) = 2T(n/4) + n` | 2, 4, `√n` | `n` | Case 3 | `Θ(n)` |
| `T(n) = 7T(n/2) + n²` | 7, 2, `n^log₂7 ≈ n^2.81` | `n²` | Case 1 | `Θ(n^log₂7)` — 스트라센 |
| `T(n) = 3T(n/4) + n²` | 3, 4, `n^log₄3 ≈ n^0.79` | `n²` | Case 3 | `Θ(n²)` |

> **시험 팁:** Case 3는 정규성 조건 검증이 까다로워 잘 안 나온다. `f(n)`과 `n^(logᵦ a)` 비교로 Case 1 vs Case 2만 정확히 구별해도 충분.

**마스터 정리 형태가 아닌 경우:** `T(n) = T(n−1) + Θ(n)` (크기가 1씩 감소) → 반복 대입으로 `Θ(n²)`. 퀵 정렬 최악, 삽입 정렬, 팩토리얼, 피보나치 계열.

## 3. 자료구조와 정렬 (W03)

### 3.1 핵심 자료구조

| 구조 | 접근 방식 | 주요 연산 | 복잡도 | 응용 |
|------|-----------|-----------|--------|------|
| **배열/리스트** | 인덱스 | 접근 O(1), 삽입/삭제 O(n) | 랜덤 접근 | 이진 탐색, 대부분 알고리즘의 기반 |
| **연결 리스트** | 순차 | 접근 O(n), 삽입/삭제 O(1) (노드 알 때) | 포인터 기반 | 동적 시퀀스 |
| **스택** | **LIFO** | push, pop — O(1) | 후입선출 | 함수 호출, 괄호 매칭, DFS |
| **큐** | **FIFO** | enqueue, dequeue — O(1) | 선입선출 | BFS, 스케줄러 |
| **힙** | 우선순위 | insert, extract-min/max — O(log n) | 배열로 저장하는 완전 이진 트리 | 우선순위 큐, 힙 정렬, Dijkstra, Prim |

**힙의 배열 표현 (1-인덱스):** `A[i]`의 자식은 `A[2i], A[2i+1]`, 부모는 `A[⌊i/2⌋]`. 힙 속성: 최대 힙은 `key(부모) ≥ key(자식)`, 최소 힙은 반대.

### 3.2 정렬 — 완벽 비교표

| 알고리즘 | 최악 | 평균 | 최선 | 공간 | 안정? | 핵심 포인트 |
|----------|------|------|------|------|-------|-------------|
| **선택 정렬** | Θ(n²) | Θ(n²) | Θ(n²) | O(1) | 아니오 | 항상 같음; swap 횟수 최소 |
| **버블 정렬** | Θ(n²) | Θ(n²) | **Θ(n)** (최적화 시) | O(1) | 예 | 최선 Θ(n)은 `swapped` 플래그 사용할 때만 |
| **삽입 정렬** | Θ(n²) | Θ(n²) | **Θ(n)** | O(1) | 예 | 거의 정렬된 입력에 최적 |
| **병합 정렬** | Θ(n log n) | Θ(n log n) | Θ(n log n) | **O(n)** | 예 | 항상 보장; 안정 |
| **퀵 정렬** | **Θ(n²)** | Θ(n log n) | Θ(n log n) | O(log n) | 아니오 | 실무에서 가장 빠름 (랜덤 피벗!) |
| **힙 정렬** | Θ(n log n) | Θ(n log n) | Θ(n log n) | O(1) | 아니오 | 최악도 보장, 제자리(in-place) |
| **계수 정렬** | Θ(n + k) | Θ(n + k) | Θ(n + k) | O(k) | 예 | 값 범위가 작아야 |
| **기수 정렬** | Θ(d(n + k)) | Θ(d(n + k)) | Θ(d(n + k)) | O(n + k) | 예 | d, k 상수면 Θ(n) |

### 3.3 비교 기반 정렬 하한

> **정리:** 임의의 비교 기반 정렬은 최악의 경우 Ω(n log n)번 비교가 필요하다.

**증명 아이디어:** 결정 트리 모델. 순열 `n!`개 → 리프 `≥ n!`. 리프 `L`개인 이진 트리의 높이는 `≥ log₂ L`. 스털링 근사로 `log₂(n!) = Θ(n log n)`.

계수/기수 정렬은 **비교를 쓰지 않아서** 이 하한을 우회 — 키의 구조(제한된 범위, 고정 자릿수)를 이용.

### 3.4 정렬의 재귀 구조

세 기본 정렬은 모두 `T(n) = T(n − 1) + Θ(n) = Θ(n²)`를 공유 — 문제 크기가 매 단계 1씩만 줄어서.
병합·퀵은 `T(n) = 2T(n/2) + Θ(n) = Θ(n log n)` — 절반씩 쪼개는 것이 근본적 속도 향상.

### 3.5 퀵 정렬 파티션 (마지막 원소 피벗)

```
PARTITION(A, p, r):
    pivot = A[r]
    i = p − 1
    for j = p to r − 1:
        if A[j] ≤ pivot:
            i = i + 1
            swap A[i], A[j]
    swap A[i+1], A[r]
    return i + 1
```

**루프 불변식:** `A[p..i] ≤ pivot`, `A[i+1..j−1] > pivot`, `A[j..r−1]`는 미확인.

**랜덤화 변형:** `[p..r]`에서 무작위 인덱스 `r'` 선택, `A[r'] ↔ A[r]` 스왑 후 파티션. 정렬된 입력에서의 `Θ(n²)` 최악을 피함.

### 3.6 힙 정렬 — 두 서브루틴

- `buildHeap`: O(n) (O(n log n) 아님 — 타이트 분석, 리프는 처리 불필요).
- `heapify(A, k, n)`: O(log n) — 아래로 내리기.
- `heapSort`: `buildHeap` + `(n−1) × heapify` = **최악 O(n log n)**, 제자리.

### 3.7 계수 정렬 핵심 아이디어

1. 개수 세기: `C[i]` = `i`와 같은 원소 수.
2. 누적 합: `C[i]` = `i` 이하 원소 수.
3. **입력을 뒤에서 앞으로** 순회(안정성): `B[C[A[j]]] = A[j]; C[A[j]]−−`.

### 3.8 기수 정렬

최하위 자리(LSD)부터 최상위 자리(MSD)까지 **안정** 정렬(보통 계수 정렬)로 처리. LSD→MSD여야 이전 자리 순서가 유지됨. `T(n) = Θ(d · (n + k))`.

## 4. 분할 정복 (W04)

### 4.1 3단계 패러다임

```
DivideAndConquer(P):
    if P가 충분히 작으면: 직접 해결    (기저)
    else:
        DIVIDE     P를 부분 문제 P₁,...,Pₖ로 분할
        CONQUER    각 Pᵢ를 재귀적으로 해결
        COMBINE    부분 해들을 원본 P의 해로 결합
```

**필수 조건:** 부분 문제가 **독립적**(겹치지 않음). 겹치면 → DP 사용.

**전형적인 점화식 형태:** `T(n) = a·T(n/b) + f(n)`.

### 4.2 D&C 알고리즘 카탈로그

| 알고리즘 | 분할 | 정복 | 결합 | 점화식 | 시간 |
|----------|------|------|------|--------|------|
| 병합 정렬 | 반으로 쪼갬 | 각 반 정렬 | 두 정렬된 반 병합 | `2T(n/2) + Θ(n)` | Θ(n log n) |
| 이진 탐색 | 중간과 비교 | 한쪽만 재귀 | 자명 | `T(n/2) + O(1)` | Θ(log n) |
| 퀵 정렬 (평균) | 파티션 | 양쪽 재귀 | 자명 | `2T(n/2) + Θ(n)` | Θ(n log n) |
| 퀵 정렬 (최악) | 불균형 파티션 | — | — | `T(n−1) + Θ(n)` | Θ(n²) |
| 랜덤 선택 | 파티션 | **한쪽**만 재귀 | 자명 | `T(3n/4) + Θ(n)` (평균) | **Θ(n)** 평균 |
| 중앙값의 중앙값 선택 | 5개씩 그룹화 | 양쪽 재귀 | 파티션 | `T(n/5) + T(7n/10) + Θ(n)` | **Θ(n)** 최악 |
| 최근접 쌍 | x로 분할 | 각 반 재귀 | 스트립 검사 | `2T(n/2) + O(n log n)` | O(n log² n) |
| 스트라센 (시험 외) | 2×2 행렬 7번 곱 | | | `7T(n/2) + Θ(n²)` | Θ(n^log₂7) |

### 4.3 선택 문제 (Selection)

**문제:** 정렬되지 않은 배열에서 `i`번째로 작은 원소 찾기.

**랜덤 선택** — 피벗으로 파티션, 순위 `i`가 있는 쪽만 재귀:
```
SELECT(A, p, r, i):
    if p == r: return A[p]
    q = PARTITION(A, p, r)
    k = q − p + 1          # A[p..r] 내에서 피벗의 순위
    if i == k: return A[q]
    else if i < k: return SELECT(A, p, q−1, i)
    else:          return SELECT(A, q+1, r, i − k)
```
평균 `Θ(n)`, 최악 `Θ(n²)`.

**중앙값의 중앙값**(최악 선형):
1. 5개씩 그룹화, 각 그룹 중앙값 찾기.
2. 중앙값들의 중앙값 `M`을 재귀로 찾기.
3. `M`으로 파티션; 순위 `i`가 있는 쪽만 재귀.

**왜 5개씩?** 최소 `3n/10`개 원소가 `≤ M`임이 보장되고, **별도로** 최소 `3n/10`개 원소가 `≥ M`임이 보장됨 → 3단계의 재귀는 최대 `7n/10`개에 대해 이루어짐. 점화식 `T(n) = T(n/5) + T(7n/10) + Θ(n)`이 선형인 이유는 `1/5 + 7/10 = 9/10 < 1`.

### 4.4 최근접 점 쌍 (D&C)

1. 점을 x좌표로 정렬 (1회만, O(n log n)).
2. x 중앙값에서 분할 → 좌/우 반.
3. 재귀로 `d_L`, `d_R` 계산; `d = min(d_L, d_R)`.
4. 분할선 주위 폭 `2d`의 **스트립**을 y 기준으로 정렬해 검사.
5. 스트립의 각 점에 대해 y 기준 **이웃 7개**만 검사하면 충분.

**왜 최대 7개?** `2d × d` 직사각형 안에서 서로 `≥ d` 떨어진 점은 최대 8개; 따라서 각 점이 확인할 상대는 7개 이하.

`T(n) = 2T(n/2) + O(n log n) = O(n log² n)` (y 기준 사전 정렬 시 `O(n log n)`).

### 4.5 D&C가 실패하는 경우 — 중복 부분 문제

순진한 재귀 피보나치는 부분 문제가 겹친다(같은 `f(k)`를 여러 번 계산). `O(2ⁿ)`. 겹치면 **DP**(저장 후 재사용) 사용.

## 5. 그리디 알고리즘 (W05)

### 5.1 그리디 템플릿

```
Greedy(C):
    S = {}
    while C ≠ ∅ 이고 S가 미완성:
        x = C에서 가장 좋아 보이는 원소
        C = C − {x}
        if S ∪ {x}가 실행 가능:
            S = S ∪ {x}
    return S
```

### 5.2 정확성을 위한 두 조건

1. **그리디 선택 속성(Greedy-choice property)** — 어떤 최적해는 그리디(지역 최적) 선택을 포함한다.
2. **최적 부분 구조(Optimal substructure)** — 그리디 선택 후 남은 부분 문제도 같은 구조를 가지며, 전체 최적해는 부분 문제 최적해를 포함한다.

> **원칙:** 먼저 그리디를 시도. 반례가 나오면 DP로 전환.

### 5.3 그리디가 실패하는 대표 반례

- **비표준 동전** `{1, 3, 4}`, 금액 6: 그리디 `4 + 1 + 1 = 3`개; 최적 `3 + 3 = 2`개.
- **비표준 동전** `{16, 10, 5, 1}`, 금액 20: 그리디 `16 + 1+1+1+1 = 5`개; 최적 `10 + 10 = 2`개.
- **0/1 배낭 (가치/무게 비율)**: 용량을 낭비할 수 있음.
- **이진 트리 최대 경로 합**: 지역 최적 자식이 전역 최적 경로를 막을 수 있음.

### 5.4 그리디 알고리즘 카탈로그

| 문제 | 전략 | 정확한가? | 시간 |
|------|------|----------|------|
| 동전 거스름돈 (표준) | 큰 것부터 | ✓ 배수 관계일 때 | O(k) |
| **분할 가능 배낭** | 가치/무게로 정렬 후 탐욕적 | ✓ 항상 | O(n log n) |
| 0/1 배낭 | — | **✗ DP 사용** | — |
| **작업 스케줄링** (기계 최소화) | 가장 빠른 시작 시간 | ✓ | O(n log n) + O(mn) |
| **활동 선택** (활동 최대, 기계 1개) | **가장 빠른 종료 시간** | ✓ | O(n log n) |
| **허프만 코딩** | 최저 빈도 두 노드 반복 병합 | ✓ 최적 접두 코드 | O(n log n) |
| **크러스컬 MST** | 사이클 없는 최소 간선 추가 | ✓ | O(m log m) |
| **프림 MST** | 한 트리 성장: 최소 교차 간선 추가 | ✓ | O(n²) 배열 / O(m log n) 힙 |
| **다익스트라 최단 경로** | 최소 거리 미확정 정점 선택, 이웃 완화 | ✓ 음이 아닌 가중치일 때 | O(n²) / O(m log n) |
| 다익스트라 + 음수 간선 | — | **✗ 벨만-포드 사용** | — |

### 5.5 작업 스케줄링 vs 활동 선택 (주의!)

**다른** 문제이고 **다른** 전략이다:

| | 작업 스케줄링 | 활동 선택 |
|--|---------------|-----------|
| 목표 | **모든** 작업 수행에 필요한 **기계 수 최소화** | **한 기계**로 처리할 수 있는 **활동 수 최대화** |
| 전략 | **가장 빠른 시작 시간** | **가장 빠른 종료 시간** |

### 5.6 허프만 코딩

- 문자 빈도로 최소 힙 구성.
- `|Q| ≥ 2`인 동안: 최소 둘 추출, 합친 빈도의 새 노드로 병합, 재삽입.
- 최종 트리가 **접두 코드** 제공: 루트→리프 경로(좌=0, 우=1) = 해당 문자 코드.
- **최적 접두 코드** 생성: 다른 어떤 접두 코드도 더 짧은 기대 길이를 낼 수 없다.

### 5.7 MST 알고리즘 — 크러스컬 vs 프림

| | 크러스컬 | 프림 |
|--|----------|------|
| 관점 | 간선 중심 (숲 → 트리) | 정점 중심 (트리 하나 성장) |
| 자료구조 | **유니온-파인드** (사이클 검출) | 우선순위 큐 / 거리 배열 `D[v]` |
| 복잡도 | O(m log m) (정렬이 지배) | O(n²) 배열 / O(m log n) 힙 |
| 적합 | 희소 그래프 | 밀집 그래프 |

### 5.8 다익스트라 — 간선 완화(Edge Relaxation)

```
Dijkstra(G, s):
    D[v] = ∞, 모든 v; D[s] = 0; S = ∅
    while S ≠ V:
        vmin = S에 없으면서 D[v] 최소인 정점
        S = S ∪ {vmin}
        for vmin의 각 이웃 w (w ∉ S):
            if D[vmin] + weight(vmin, w) < D[w]:     # RELAX
                D[w] = D[vmin] + weight(vmin, w)
```

**그리디 불변식:** `vmin`이 확정되면 `D[vmin]`은 실제 최단 거리 — **모든 간선이 음이 아닐 때**만 유효.

## 6. 동적 프로그래밍 (W06)

### 6.1 정의와 두 필수 조건

**DP**는 다음으로 문제를 푼다:
1. **중복** 부분 문제로 분해.
2. 각 부분 문제를 **한 번만** 풀고 결과 저장.
3. **작은** 것에서 **큰** 것으로 쌓아 올림.

**필수 조건:**
- **최적 부분 구조:** 전체 최적해가 부분 문제 최적해를 포함.
- **중복 부분 문제:** 같은 부분 문제가 반복 등장.

> "프로그래밍"은 *테이블화(tabulation)* 의미 — 수학적 최적화 용어이지 컴퓨터 프로그래밍이 아님.

### 6.2 메모이제이션 vs 타뷸레이션

| | 메모이제이션 (하향식) | 타뷸레이션 (상향식) |
|--|----------------------|---------------------|
| 방식 | 재귀 + 캐시 | 반복문으로 테이블 채움 |
| 구현 난이도 | 보통 쉬움 (재귀에 캐시 추가) | 순서를 명시해야 |
| 해결 부분 문제 | 실제 필요한 것만 (lazy) | **모든** 부분 문제 |
| 오버헤드 | 재귀 스택, 캐시 조회 | 타이트 루프, 캐시 친화 |
| 보통 유리 | 희소하게 필요할 때 | 조밀하게 필요할 때 (실무에서 더 빠름) |

### 6.3 DP 레시피 (6단계)

1. **부분 문제 정의** — `OPT[i, j, ...]`가 무엇을 나타내는가?
2. **점화식 작성** — 작은 부분 문제와 어떻게 연결되는가?
3. **기저 조건 식별** — 자명한 경계값.
4. **계산 순서 결정** — 의존성을 만족하도록 작은 것부터 채움.
5. **답 추출** — 어느 셀에 최종 값이?
6. **(선택) 역추적** — 값뿐 아니라 실제 해 복원.

### 6.4 고전 DP 문제 — 완벽 정리

| 문제 | 부분 문제 | 점화식 | 기저 | 시간 | 공간 |
|------|-----------|--------|------|------|------|
| **피보나치** | `f[i]` | `f[i] = f[i−1] + f[i−2]` | `f[1]=f[2]=1` | O(n) | O(n) → O(1) |
| **행렬 경로** (최대) | `c[i,j]` = `(i,j)`까지 최대 합 | `c[i,j] = m[i,j] + max(c[i−1,j], c[i,j−1])` | `c[i,0]=c[0,j]=0` | O(n²) | O(n²) → O(n) |
| **LCS** | `c[i,j]` = `X₁..ᵢ, Y₁..ⱼ`의 LCS | 일치: `c[i−1,j−1]+1`; 불일치: `max(c[i−1,j], c[i,j−1])` | `c[i,0]=c[0,j]=0` | O(mn) | O(mn) → O(min(m,n)) |
| **편집 거리** | `E[i,j]` = `X₁..ᵢ, Y₁..ⱼ` 편집 거리 | 일치: `E[i−1,j−1]`; 불일치: `1 + min(E[i−1,j], E[i,j−1], E[i−1,j−1])` | `E[i,0]=i, E[0,j]=j` | O(mn) | O(mn) |
| **0/1 배낭** | `K[i,w]` = 물건 1..i, 용량 w일 때 최대 가치 | `wᵢ > w`: `K[i−1,w]`; 아니면: `max(K[i−1,w], K[i−1,w−wᵢ]+vᵢ)` | `K[0,w]=K[i,0]=0` | O(nC) | O(nC) |
| **동전 거스름돈** (최소) | `C[j]` = 금액 `j` 만드는 최소 동전 수 | `C[j] = min over dᵢ ≤ j of {C[j−dᵢ] + 1}` | `C[0]=0` | O(nk) | O(n) |
| **플로이드-워셜** | `d[i,j,k]` 정점 `{1..k}` 경유 | `min(d[i,j,k−1], d[i,k,k−1] + d[k,j,k−1])` | `d[i,j,0]=w(i,j)` | O(n³) | O(n²) |

### 6.5 0/1 배낭이 O(nC)인 이유 — "의사다항시간(pseudo-polynomial)"

용량 `C`의 입력 크기는 O(log C) 비트지만, 알고리즘은 O(C) = O(2^log C) — 비트 길이에 대해 지수. 따라서 0/1 배낭은 고전적 의미에서 **NP-hard**이며, DP는 *수치 값*에 대해서만 다항 시간.

### 6.6 플로이드-워셜 — 반복문 순서가 결정적

```
FloydWarshall(W, n):
    D = W
    for k = 1 to n:          # ← 가장 바깥 (이 순서 고정)
        for i = 1 to n:
            for j = 1 to n:
                D[i,j] = min(D[i,j], D[i,k] + D[k,j])
    return D
```

`k`가 **반드시** 가장 바깥 루프. `k` 반복마다 "허용 중간 정점"을 `{1..k−1}`에서 `{1..k}`로 확장. 순서를 바꾸면 **오답**.

### 6.7 LCS 역추적

`c[m, n]`에서 시작:
- `X[i] == Y[j]`: 해당 문자 포함, `(i−1, j−1)`로 대각선 이동.
- 아니면: `c[i−1, j]`와 `c[i, j−1]` 중 큰 쪽으로 이동.

## 7. 패러다임 비교

### 7.1 D&C vs 그리디 vs DP — 한눈에

| 측면 | 분할 정복 | 그리디 | 동적 프로그래밍 |
|------|-----------|--------|-----------------|
| 접근 | 분할, 재귀, 결합 | 매 단계 지역 최적 선택 | 모든 부분 문제 풀고 결합 |
| 부분 문제 | 독립적 (트리) | N/A (부분 문제 재귀 없음) | **중복** (DAG) |
| 최적 보장? | 문제 의존 | 그리디 선택 속성 있을 때만 | 예 (최적 부분 구조 있을 때) |
| 전형적 시간 | 흔히 O(n log n) | 흔히 O(n log n) 또는 O(n) | 흔히 O(n²) 또는 O(n³) |
| 공간 | O(log n) 스택 | O(1) ~ O(n) | O(n) ~ O(n²) 테이블 |
| 전형적 용도 | 정렬, 탐색 | 스케줄링, MST, 최단 경로 | 시퀀스 문제, 배낭, 체인 최적화 |

### 7.2 어느 기법을 쓸까 — 결정 흐름

```
최적값(min/max)을 구하는 문제인가?
├── 아니오 → D&C 또는 완전 탐색
└── 예
    ├── 안전한 그리디 선택을 증명할 수 있는가? → 그리디 (정확성 증명!)
    └── 아니오
        ├── 부분 문제가 겹치는가?   → DP
        └── 독립적인가?            → D&C
```

### 7.3 같은 문제, 다른 패러다임

| 문제 | 그리디 | DP | D&C |

## 8. 공식 치트시트

**정렬 복잡도:** §3.2의 전체 표를 외울 것.

**마스터 정리:** `T(n) = a·T(n/b) + f(n)`; `h = n^(logᵦ a)`.

| | `f(n)` vs `h` | 결과 |
|--|---------------|------|
| Case 1 | `f(n) = O(h / n^ε)` | `Θ(h)` |
| Case 2 | `f(n) = Θ(h)` | `Θ(h · log n)` |
| Case 3 | `f(n) = Ω(h · n^ε)` + 정규성 | `Θ(f(n))` |

**꼭 외워야 할 점화식:**

| 점화식 | 해 | 예시 |
|--------|-----|------|
| `T(n) = T(n/2) + Θ(1)` | `Θ(log n)` | 이진 탐색 |
| `T(n) = T(n−1) + Θ(1)` | `Θ(n)` | 팩토리얼 |
| `T(n) = 2T(n/2) + Θ(n)` | `Θ(n log n)` | 병합 정렬 |
| `T(n) = 2T(n/2) + Θ(1)` | `Θ(n)` | — |
| `T(n) = T(n−1) + Θ(n)` | `Θ(n²)` | 퀵 정렬 최악, 삽입 정렬 |
| `T(n) = 2T(n−1) + Θ(1)` | `Θ(2ⁿ)` | 하노이 탑 |
| `T(n) = T(n−1) + T(n−2) + Θ(1)` | `Θ(φⁿ)` ≈ O(1.618ⁿ) | 순진한 재귀 피보나치 |
| `T(n) = T(n/5) + T(7n/10) + Θ(n)` | `Θ(n)` | 중앙값의 중앙값 |

**핵심 DP 점화식:**
- LCS: `c[i,j] = c[i−1,j−1] + 1` (일치) / `max(c[i−1,j], c[i,j−1])` (불일치)
- 편집 거리: `E[i,j] = E[i−1,j−1]` (일치) / `1 + min(E[i−1,j], E[i,j−1], E[i−1,j−1])` (불일치)
- 0/1 배낭: `K[i,w] = max(K[i−1,w], vᵢ + K[i−1, w−wᵢ])` (`wᵢ ≤ w`일 때)
- 행렬 경로 (최대): `c[i,j] = m[i,j] + max(c[i−1,j], c[i,j−1])`
- 동전 거스름돈: `C[j] = min over dᵢ ≤ j of {C[j−dᵢ] + 1}`
- 플로이드-워셜: `d[i,j] = min(d[i,j], d[i,k] + d[k,j])` (`k`가 가장 바깥)

## 9. 연습 문제 (시험 스타일)

### 문제 1 — 마스터 정리

**Q:** `T(n) = 4T(n/2) + n`의 복잡도는?

**A:** `a = 4, b = 2, f(n) = n, n^(logᵦ a) = n²`. `n = O(n^(2−1))`이므로 **Case 1** → `T(n) = Θ(n²)`.

### 문제 2 — Big-O 증명

**Q:** `5n³ + 2n² + 7 = O(n³)`을 증명하라.

**A:** `c = 5 + 2 + 7 = 14`, `n₀ = 1`로 잡으면: 모든 `n ≥ 1`에 대해 `5n³ + 2n² + 7 ≤ 5n³ + 2n³ + 7n³ = 14n³ = c·n³`. ∎

### 문제 3 — 그리디 vs DP (동전 거스름돈)

**Q:** 동전 `{1, 3, 4}`, 금액 6. 그리디와 DP 비교.

**그리디:** 4 + 1 + 1 = 3개.

**DP:**

| j     | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|-------|---|---|---|---|---|---|---|
| C[j]  | 0 | 1 | 2 | 1 | 1 | 2 | **2** |

`C[6] = min(C[5], C[3], C[2]) + 1 = min(2, 1, 2) + 1 = 2` → 최적은 **{3, 3}** (2개).

### 문제 4 — LCS 채우기와 역추적

**Q:** `X = "ABCB"`, `Y = "BDCAB"`의 LCS.

|     |   | B | D | C | A | B |
|-----|---|---|---|---|---|---|
|     | 0 | 0 | 0 | 0 | 0 | 0 |
| A   | 0 | 0 | 0 | 0 | **1** | 1 |
| B   | 0 | **1** | 1 | 1 | 1 | **2** |
| C   | 0 | 1 | 1 | **2** | 2 | 2 |
| B   | 0 | 1 | 1 | 2 | 2 | **3** |

**답:** LCS 길이 **3**, 예: `"BCB"`. 우하단에서 시작, 일치에서 대각선 이동.

### 문제 5 — 0/1 배낭 추적

**Q:** 물건 `(w,v) = (2,12), (1,10), (3,20), (2,15)`, 용량 `C = 5`. `K[4,5]`를 구하라.

| i \ w | 0 | 1 | 2 | 3 | 4 | 5 |
|-------|---|---|---|---|---|---|
| 0 | 0 | 0 | 0 | 0 | 0 | 0 |
| 1 (2,12) | 0 | 0 | 12 | 12 | 12 | 12 |
| 2 (1,10) | 0 | 10 | 12 | 22 | 22 | 22 |
| 3 (3,20) | 0 | 10 | 12 | 22 | 30 | 32 |
| 4 (2,15) | 0 | 10 | 15 | 25 | 30 | **37** |

**답:** `K[4,5] = 37`. 역추적: 물건 4 + 물건 2 + 물건 1 (무게 2+1+2 = 5, 가치 15+10+12 = 37).

### 문제 6 — 퀵 정렬 최악

**Q:** 마지막 원소 피벗 퀵 정렬이 `Θ(n²)`로 도는 입력은? 랜덤 피벗이 이를 어떻게 피하는가?

**A:** 이미 정렬된 (또는 역정렬된) 입력: 매 파티션이 `0 : (n−1)`, `T(n) = T(n−1) + Θ(n) = Θ(n²)`. 랜덤 피벗은 균일 랜덤 인덱스를 먼저 선택; 반복적 최악 발생 확률은 극히 낮아져 기대 `Θ(n log n)` 회복.

### 문제 7 — 다익스트라 + 음수 간선

**Q:** 다익스트라가 오답을 내는 그래프 예시.

**A:** 방향 간선: `s→a = 4, s→b = 2, a→b = −5`.
- 1단계: `s` 확정. 완화: `D[a] = 4, D[b] = 2`.
- 2단계: `b` 선택 (최소 = 2). 확정. (`b`에서 나가는 간선 없음.)
- 3단계: `a` 선택 (D = 4). 확정. `a → b` 완화 시도: `D[b] = 4 − 5 = −1`이 되어야 하지만, **`b`가 이미 확정됨** → 갱신 무시.

다익스트라 결과: `D[b] = 2`. 실제 최단 경로: `s → a → b = 4 − 5 = −1`. "한번 확정되면 거리가 최적" 그리디 불변식은, **아직 확정되지 않은 정점에서 나가는 음수 간선이 이미 확정된 정점을 개선할 수 있을 때** 깨진다.

### 문제 8 — 활동 선택

**Q:** 활동 `(s, f)`: `(1,3), (2,5), (4,7), (1,8), (5,9), (8,10)`. 가장 빠른 종료 시간 적용.

종료 시간 정렬: `(1,3), (2,5), (4,7), (1,8), (5,9), (8,10)`.
- `(1,3)` 선택 → `last = 3`
- `(2,5)`: `2 < 3` → 스킵
- `(4,7)`: `4 ≥ 3` → 선택; `last = 7`
- `(1,8)`: `1 < 7` → 스킵
- `(5,9)`: `5 < 7` → 스킵
- `(8,10)`: `8 ≥ 7` → 선택; `last = 10`

**답:** 3개 — `{(1,3), (4,7), (8,10)}`.

## 10. 자주 하는 실수

| 실수 | 예방법 |
|------|--------|
| O / Ω / Θ 혼동 | O=상한, Ω=하한, Θ=양쪽 모두 타이트 |
| 느슨한 Big-O | 둘 다 성립하면 O(n log n)을 O(n²)보다 선호 |
| `f(n)`이 다항식 아닌데 마스터 정리 적용 | 반복 대입/귀납법으로 전환 |
| 증명 없이 그리디 주장 | 먼저 반례부터 찾아볼 것 |
| DP 기저 잘못 | `n = 0, 1, 2`로 점화식 손 계산 검증 |
| DP 테이블 off-by-one | 0-인덱스인지 1-인덱스인지 명시 |
| 분할 가능 배낭 vs 0/1 배낭 | 분할 가능 = 그리디; 0/1 = DP |
| 정렬 입력에서 퀵 정렬 `Θ(n²)` | 랜덤 또는 중앙값-3 피벗 사용 |
| 플로이드-워셜 반복문 순서 틀림 | `k`가 반드시 가장 바깥 |
| 다익스트라 + 음수 간선 | 벨만-포드 사용 |
| "직관적으로" 그리디 선택 속성 주장 | 교환 논증(exchange argument)으로 증명 |
| 귀납법: `cn + 1 ≤ cn`?? | 가정을 `cn − d`로 강화 |
| "병합 정렬은 O(n log n)" (느슨) | `Θ(n log n)` (타이트)로 쓰기 |
| 병합 정렬의 O(n) 공간 망각 | 보조 배열 필요함을 기억 |
| 무제한 키에 계수 정렬 | Θ(n)이 되려면 `k = O(n)` 필요 |
