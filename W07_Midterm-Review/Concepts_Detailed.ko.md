## 0. 시험 개요

중간고사는 1–6주차를 다룹니다. 주당 3시간의 강의·랩, 총 18시간 분량의 내용을 **60분 필기 시험** 하나로 평가합니다. 전자기기 없음, 참조 없음. 의사코드 작성, DP 테이블 채우기, 알고리즘 추적, 점근적 경계 증명, 정확성 논증을 직접 손으로 해야 합니다.

| 주차 | 주제 | CLRS | 한 줄 요약 |
|------|------|------|-------------|
| 01 | 알고리즘 입문 | — | 알고리즘은 유한하고 명확한 절차; "반으로 나누기"가 로그 성능의 근원 |
| 02 | 복잡도 분석 | 1–3 | Big-O는 불필요한 디테일을 걷어냄; 마스터 정리는 D&C 점화식을 해결 |

지금까지 배운 모든 주요 알고리즘은 이 사슬 어딘가에 위치합니다. 이를 내면화하면 새로운 알고리즘이 임의적이 아니라 익숙하게 느껴집니다.

### 0.1 시험이 실제로 묻는 것

- **정의와 용어.** Big-O, 마스터 정리, 그리디 선택 속성, 최적 부분 구조를 자유롭게 서술할 수 있어야.
- **손으로 추적하기.** 파티션, 병합, heapify, buildHeap, 크러스컬, 프림, 다익스트라, 허프만, LCS, 배낭, 플로이드-워셜, 편집 거리 — 이 중 최소 하나는 작은 예제에서 실행하라고 물을 것.
- **증명과 논증.** `f(n) = O(g(n))` 증명, 점화식 풀이, 그리디 선택의 최적성 증명, 또는 반례 구성.
- **알고리즘 설계.** 새 문제를 주고 패러다임(D&C / 그리디 / DP)을 식별한 뒤 개략적 해법을 제시.

### 0.2 전략

1. **모든 문제를 먼저 읽고** 배점에 비례해 시간 배분.
2. **풀이 과정을 보여라.** 최종 답이 틀렸어도 올바른 추론은 부분 점수.
3. **표와 트리를 그려라.** DP 테이블, 재귀 트리, 허프만 트리, 간트 차트 — 시각화가 산문보다 빠르고 오류가 적다.
4. **작은 입력으로 검증.** 점화식이나 DP 테이블을 `n = 1, 2, 3`에 대해 손으로 확인한 뒤 신뢰하라.
5. **사용 기법을 명시.** "마스터 정리 Case 2에 의해" 또는 "교환 논증에 의해"를 쓰면 채점자가 당신의 접근을 알 수 있고, 실행이 비틀거려도 점수를 받을 수 있다.

## 1. 기초 (W01)

### 1.1 알고리즘이란 정확히 무엇인가?

알고리즘은 **체계적이고 유한하며 명확한 절차**다. 어떤 선조건을 만족하는 입력이 주어지면, 후조건을 만족하는 출력을 생성해야 한다. 모든 단계는 기계적으로 실행 가능해야 한다 — "적절히 처리" 혹은 "현명하게 선택" 같은 것은 안 된다.

교과서의 세 가지보다 더 깊이 들어가는 다섯 가지 성질:

| 성질 | 의미 | 실패 모드 |
|------|------|-----------|
| **정확성** | 모든 유효 입력에 대해 출력이 명세를 만족 | 잘못된 답이나 정의되지 않은 동작 |
| **실행 가능성** | 각 단계가 기계적으로 실행 가능 | 오라클이나 계산 불가능한 연산에 의존 |
| **유한성** | 유한한 단계에 종료 | `while True: pass`는 알고리즘이 **아니다** |
| **효율성** | 합리적인 시간·공간 사용 | 작은 입력에서 지수 시간 |
| **명확성** | 각 단계가 정확히 명시 | "상황에 맞게 처리" — 너무 모호 |

> **왜 명확성이 시험에서 중요한가.** 의사코드에서는 독자가 손으로 시뮬레이션할 수 있는 연산을 써야 한다. `for i = 1 to n: A[i] = A[i] + 1`은 알고리즘이지만, `repeat until it's right`는 아니다.

니클라우스 비르트의 유명한 등식 `알고리즘 + 자료구조 = 프로그램`은 이 분야 전체를 요약한다: 알고리즘은 *무엇을 할지*를 말하고, 자료구조는 *정보를 어떻게 조직할지*를 말하며, 둘은 분리 불가능하다. 같은 검색 문제가 정렬되지 않은 배열에서는 `O(n)`, 정렬된 배열에서는 `O(log n)` — 자료구조가 바뀌면 알고리즘도 바뀐다.

### 1.2 알고리즘 표현

세 가지 표현:

1. **자연어** — 직관에는 적합하지만 형식 분석에는 너무 모호.
2. **의사코드** — 구조적이고 프로그래밍 언어 유사. CLRS가 사용하며 시험에서도 쓴다.
3. **순서도** — 시각적, 제어 흐름에 유용, 몇 개 이상의 분기에는 부적합.

시험 관행: CLRS 스타일 의사코드, 달리 명시되지 않으면 1-인덱스 배열, 대입은 `=`, 비교는 `==`. `↔`은 교환(swap).

### 1.3 유클리드 GCD — 가장 오래된 알고리즘 (~기원전 300년)

**핵심 항등식:** `gcd(a, b) = gcd(b, a mod b)`, 기저 `gcd(a, 0) = a`.

**이 항등식이 성립하는 이유.** `G = gcd(a, b)`라 하자. `a = q·b + r` (`r = a mod b`). `a`와 `b`의 임의의 공약수는 `r = a − q·b`도 나눈다. 역으로, `b`와 `r`의 임의의 공약수는 `a = q·b + r`도 나눈다. 따라서 `(a, b)`의 공약수 집합과 `(b, r)`의 공약수 집합이 같으며, 특히 그들의 최댓값도 일치한다.

```python
def gcd(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

**`gcd(48, 18)` 추적:**

| 호출 | a | b | a mod b |
|------|---|---|---------|
| 1 | 48 | 18 | 12 |
| 2 | 18 | 12 | 6 |
| 3 | 12 | 6 | 0 |
| 4 | 6 | 0 | — 6 반환 |

**종료 보장 이유.** 매 반복마다 `b`가 엄격히 감소하고 음이 아니므로, 유한한 단계에 반드시 0에 도달한다.

**얼마나 빠른가?** 최악 `O(log min(a, b))` 반복. 연속한 피보나치 수에서 최악 발생 — 고전적인 결과로, 시험에서 상세히 묻지는 않지만 알아두면 좋다: GCD는 피보나치 유사 쌍을 "싫어한다."

### 1.4 고전 문제와 log₂(n) 테마

W01 강의가 일곱 문제를 소개하는 이유는, 이들이 이후 반복해서 만나게 될 다섯 가지 알고리즘 전략 — 순차 스캔, 이진 반분, 그리디 선택, 그래프 순회, 정보 이론적 인코딩 — 을 포괄하기 때문이다.

| 문제 | 전략 | 최악 분석 |
|------|------|-----------|
| 최댓값 찾기 | 순차 스캔 | 비교 `n − 1`회, `O(n)` |
| 이진 탐색 (정렬된 입력) | 분할 정복 | 비교 `⌈log₂(n)⌉`회, `O(log n)` |
| 동전 거스름돈 (표준 화폐) | 그리디 (큰 것부터) | 화폐 종류 `k`개면 `O(k)` |
| 오일러 회로 / 한붓그리기 | 그래프 순회, 다리 회피 | 모든 정점 차수가 짝수일 때 존재 |
| 미로 (벽 연결) | 오른손 법칙 (DFS 변종) | 항상 출구 도달 |
| 1,024개 중 가짜 동전 | 반으로 나누기 | **10회 측정** |
| n개 항아리 독 든 포도주 | 이진 인코딩 | **⌈log₂(n)⌉**명 시종 |

**log₂(n) 후렴.** W01의 세 가지 다른 문제 — 이진 탐색, 가짜 동전, 독 든 포도주 — 모두 각 단계마다 미지의 정보를 **절반**으로 자르기 때문에 `O(n)` 또는 `O(n/2)`에서 `O(log n)`으로 축소된다. 이는 병합 정렬이 `n²` 대신 `n log n`이 되는 것과 같은 수학적 현상이다: 반분 × 반분 × ... × 반분하여 1에 도달하려면 `log₂ n`번 필요. 효율적인 분할 정복 알고리즘은 모두 이 사실을 타고 움직인다.

#### 1.4.1 최댓값 찾기 (풀이 추적)

```python
def find_max(cards):
    max_val = cards[0]
    for i in range(1, len(cards)):
        if cards[i] > max_val:
            max_val = cards[i]
    return max_val
```

입력 `[3, 8, 2, 11, 5]`에서:
- 시작 `max = 3`.
- `i = 1`: `8 > 3` → `max = 8`.
- `i = 2`: `2 ≤ 8` → 변화 없음.
- `i = 3`: `11 > 8` → `max = 11`.
- `i = 4`: `5 ≤ 11` → 변화 없음.
- `11` 반환.

**루프 불변식**으로 정확성 증명: 반복 `i` 시작 시점에 `max_val`은 `cards[0..i−1]`의 최댓값이다. 이것이 반복 알고리즘 귀납 증명의 청사진이다.

#### 1.4.2 이진 탐색 (풀이 추적)

`[15, 20, 25, 35, 45, 55, 60, 75, 85, 90]`에서 `85` 탐색:

| 단계 | `left` | `right` | `mid` | `A[mid]` | 판단 |
|------|--------|---------|-------|----------|------|
| 1 | 0 | 9 | 4 | 45 | `45 < 85` → 오른쪽 |
| 2 | 5 | 9 | 7 | 75 | `75 < 85` → 오른쪽 |
| 3 | 8 | 9 | 8 | 85 | 일치 → 인덱스 8 반환 |

10개 중에서 3번 비교. `n = 10⁶`이면 이진 탐색은 최대 20번 비교; 순차 탐색은 최대 100만번.

> **`left <= right` (등호 포함)인 이유.** 탐색 구간이 단일 원소로 좁혀졌을 때(`left == right`) 그 원소를 검사해야 한다. 등호를 빼면 타깃이 마지막 위치에 있을 때 놓치는 off-by-one 버그가 생긴다. 코딩 테스트에서 이진 탐색의 단일 최빈 실수.

#### 1.4.3 오일러 회로 조건

**정리 (오일러).** 연결 무방향 그래프는 **모든 정점의 차수가 짝수**일 때만 **오일러 회로**(모든 간선을 정확히 한 번씩 사용하는 닫힌 걸음)를 가진다. **오일러 경로**(반드시 닫힐 필요 없음)는 홀수 차수 정점이 정확히 0개 또는 2개일 때만 존재한다.

**직관.** 펜이 어떤 정점에 들어갈 때마다 반드시 나와야 하므로 간선을 쌍으로 사용한다. 홀수 차수 정점이 있으면 언젠가 들어가서 나올 간선이 없어진다. 홀수 차수 0개: 경로가 닫힌다. 2개: 하나에서 시작해 다른 하나에서 끝난다.

**알고리즘 원칙 — 다리를 너무 일찍 건너지 말 것.** **다리**는 제거 시 그래프를 분리하는 간선이다. 여러 선택이 있는 정점에서, 다리가 *아닌* 간선(돌아올 수 있는 사이클이 있는 간선)을 선호하라. 다리를 일찍 건너면 반대쪽에 갇혀 도달하지 못할 간선이 남는다.

**다리 / 사이클 검출:** 후보 이웃에서 DFS를 시작; DFS가 다른 간선으로 현재 정점에 도달하면 사이클 존재, 그 간선은 안전.

#### 1.4.4 가짜 동전 — 세 가지 접근

`n`개 동전 중 하나가 약간 가볍다. 양팔 저울로 찾는다.

| 접근 | 방법 | `n = 1024` 최악 |
|------|------|-----------------|
| A (하나씩) | 동전 1을 2..n과 차례로 비교 | **1,023** |
| B (쌍) | 쌍으로 측정; 불균형 쌍에서 한 번 더 | **~512** |
| C (반분) | 두 반을 측정; 가벼운 쪽에 가짜; 재귀 | **10** |

**접근 C 추적 `n = 1024`:**
```
1024 → 512 → 256 → 128 → 64 → 32 → 16 → 8 → 4 → 2 → 1  (10회 측정)
```

매 측정이 후보를 반으로 줄임 → `log₂ n`번 측정. 이 수업에서 만나는 첫 번째 분할 정복 분석.

> **3분할은 왜 안 쓰나?** 실제로 더 좋다: 3더미로 나누어 두 개를 측정, 가벼운 더미 판단. `log₃ n`회 측정 — 여전히 `O(log n)`, 상수만 다름. 중요한 것은 **로그 함수**이지 밑이 아니다.

#### 1.4.5 독 든 포도주 — 이진 인코딩

항아리 `k`에 `k`와 같은 `⌈log₂(n)⌉`-비트 이진 ID를 부여. 시종 `i`는 비트 `i`가 1인 모든 항아리에서 마심. 일주일 후 죽은 시종들의 비트 패턴이 독 든 항아리의 이진 ID와 정확히 일치.

**4개 항아리, 2명 시종:**

| 항아리 | 이진 | 시종 A | 시종 B |
|--------|------|--------|--------|
| 0 | 00 | — | — |
| 1 | 01 | — | ✓ |
| 2 | 10 | ✓ | — |
| 3 | 11 | ✓ | ✓ |

A 사망, B 생존 → 이진 `10` → 항아리 2.

**왜 작동하는가.** 각 시종은 1비트 정보(생/사)를 보고. `n`개 가능성을 구별하려면 `⌈log₂(n)⌉` 비트 필요. 알고리즘은 정확히 이 정보 이론적 최소치만큼 지불.

**`n`이 2의 거듭제곱이 아닐 때:** `⌈log₂(n)⌉`명 시종 사용; 일부 비트 패턴은 미사용. 예: `n = 5`는 시종 3명(8 패턴), 3 패턴 미사용. 그래도 인코딩 체계는 (있다면) 어느 항아리가 독 든 항아리인지 식별한다.

## 2. 복잡도 분석 (W02)

### 2.1 정확한 연산 수에서 점근적 성장으로

알고리즘에 대해 `T(n)` = 입력 크기 `n`의 함수로서 **기본 연산**(비교, 대입, 사칙연산, 배열 접근) 수를 정의. **RAM 모델**에서 기본 연산 1회 = 시간 1단위이므로 `T(n)`은 상수 배까지는 실시간의 충실한 대용이다.

왜 초가 아닌 연산을 세는가? 초는 하드웨어에 의존. 연산은 알고리즘에만 의존. `O(n²)` 알고리즘은 슈퍼컴퓨터에서도 당신 노트북의 `O(n log n)` 알고리즘에 결국 진다 — 하드웨어 상수는 최대 1–2자릿수 이득이지만, 알고리즘 개선은 임의로 많은 자릿수를 산다.

### 2.2 복잡도 분석 네 가지

| 유형 | 의미 | 사용 시기 |
|------|------|-----------|
| **최악** | 크기 `n` 입력 전체에 대한 상한 | 기본 — 보장을 제공 |
| **평균** | 확률 분포(보통 균등) 하 기댓값 | 입력 분포를 알 때 |
| **최선** | 가장 빠른 실행 | 단독으로는 거의 쓸모 없음; 하한에 좋음 |
| **분할 상환** | `m`회 연산 시퀀스의 최악 *총* 비용을 `m`으로 나눔 | 때때로 비싼 연산이 있는 자료구조(동적 배열, 스플레이 트리) |

통근 비유:

| 경우 | 집 → 강의실 | 의미 |
|------|-------------|------|
| 최선 | 36분 (열차 대기 중) | 하한 |
| 최악 | 40분 (방금 놓침) | 약속할 수 있는 보장 |
| 평균 | 38분 | 전형적 일상 |

**왜 시험에서 최악이 지배적인가.** 보장은 계약이다. "퀵 정렬은 어떤 상수 `c`에 대해 최대 `c · n²` 시간 걸린다"는 지속 가능한 진술; "보통 `n log n` 걸린다"는 희망. 보장은 합성 가능; 희망은 그렇지 않다.

### 2.3 점근적 표기 — 형식적 정의

| 표기 | 집합적 정의 | 쉬운 말 |
|------|-------------|---------|
| `O(g(n))` | `{ f : ∃ c > 0, n₀ ≥ 0 s.t. ∀ n ≥ n₀, f(n) ≤ c·g(n) }` | **상한**: `f`는 `g`보다 빠르게 자라지 않음 |
| `Ω(g(n))` | `{ f : ∃ c > 0, n₀ ≥ 0 s.t. ∀ n ≥ n₀, f(n) ≥ c·g(n) }` | **하한**: `f`는 `g`보다 느리게 자라지 않음 |
| `Θ(g(n))` | `O(g(n)) ∩ Ω(g(n))` | **타이트 한계**: `f`와 `g`는 정확히 같은 속도로 자람 (상수 배까지) |

정의는 "충분히 큰 모든 `n`에 대해, `f`를 `g`의 상수 배로 경계 지을 수 있다." 상수와 작은 `n`은 무관 — 오직 *성장률*만 중요.

**정의 소리 내어 읽기.** "상수 `c`와 임계점 `n₀`이 존재하여, 모든 `n ≥ n₀`에 대해 `f(n)`이 최대 `c · g(n)`이다." `∃ c`는 약속; `c`를 원하는 만큼 크게 골라 부등식을 성립시킬 자유가 있다.

### 2.4 `f(n) = O(g(n))` 증명 — 주력 기술

**다항식 레시피.** `f(n) = aₖnᵏ + aₖ₋₁nᵏ⁻¹ + ... + a₀`라면 `f(n) = O(nᵏ)`를 다음으로 증명:

- `c = |aₖ| + |aₖ₋₁| + ... + |a₀|`
- `n₀ = 1`

**왜 이게 작동하는가.** 모든 `n ≥ 1`에 대해 각 항 `|aᵢ|·nⁱ ≤ |aᵢ|·nᵏ` (`n ≥ 1`, `i ≤ k`이므로 `nⁱ ≤ nᵏ`). 합하면, `f(n) ≤ (|aₖ| + ... + |a₀|)·nᵏ = c·nᵏ`.

**풀이 예시.** `2n² − 8n + 3 = O(n²)` 증명.

`c = |2| + |−8| + |3| = 13`, `n₀ = 1`. 모든 `n ≥ 1`에 대해:

`|2n² − 8n + 3| ≤ 2n² + 8n + 3 ≤ 2n² + 8n² + 3n² = 13n² = c·n²`. ∎

(음 계수를 깔끔히 처리하려고 절댓값으로 감싼다.)

**Θ 증명 예시.** `3n + 2 = Θ(n)` 증명.

*상한:* `3n + 2 ≤ 5n` (`n ≥ 1`)이므로 `3n + 2 = O(n)`.

*하한:* `3n + 2 ≥ 3n` (모든 `n ≥ 0`)이므로 `3n + 2 = Ω(n)`.

`c₁ = 3, c₂ = 5, n₀ = 1`로: `3n ≤ 3n + 2 ≤ 5n` (`n ≥ 1`). 따라서 `3n + 2 = Θ(n)`. ∎

**타이트성 원칙.** 항상 정당화할 수 있는 가장 타이트한 경계를 보고하라. `n log n + 5n = O(n²)`는 기술적으로 참이지만 무용; `O(n log n)` 또는 더 나아가 `Θ(n log n)`로 써라. 느슨한 경계는 정보 손실.

### 2.5 복잡도 계층

```
O(1) ⊂ O(log n) ⊂ O(√n) ⊂ O(n) ⊂ O(n log n) ⊂ O(n²) ⊂ O(n³) ⊂ O(2ⁿ) ⊂ O(n!)
```

| 클래스 | 이름 | 예시 | PC에서 `n = 10⁶` |
|--------|------|------|-------------------|
| O(1) | 상수 | 배열 접근, 해시 조회 | 즉시 |
| O(log n) | 로그 | 이진 탐색 | ~20 연산 |
| O(√n) | 준선형 | 시행 제수 소수 판정 | ~10³ 연산 |
| O(n) | 선형 | 선형 스캔, 계수 정렬 | ~초 |
| O(n log n) | 선형로그 | 병합 / 힙 정렬 | 큰 `n`의 실용적 상한 |
| O(n²) | 이차 | 버블 정렬 | 분~시간 |
| O(n³) | 삼차 | 플로이드-워셜, 행렬 곱 | 작은 `n`만 |
| O(2ⁿ) | 지수 | 부분집합 열거, 순진한 재귀 피보나치 | `n ≈ 40` 이상 불가능 |
| O(n!) | 팩토리얼 | 모든 순열, 순진한 TSP | `n ≈ 12` 이상 불가능 |

**체감 표. 10억 개 숫자 정렬:**

| 알고리즘 | `n = 10³` | `n = 10⁶` | `n = 10⁹` |
|----------|-----------|-----------|-----------|
| PC O(n²) | < 1 s | ~2 h | ~**300년** |
| 슈퍼컴 O(n²) | < 1 s | ~1 s | ~1주 |
| PC O(n log n) | < 1 s | < 1 s | **~5분** |
| 슈퍼컴 O(n log n) | < 1 s | < 1 s | < 1 s |

더 나은 알고리즘이 더 큰 컴퓨터를 이긴다. 항상. 이 수업의 가장 중요한 엔지니어링 교훈이다.

**왜 Big-O에서 로그 밑이 상관없는가.** 밑 변환으로 `logₐ n = (logᵦ n) / (logᵦ a)`, 여기서 `1/logᵦ a`는 양의 상수. 상수는 `O(·)`에서 사라지므로, 밑 `≥ 2`인 어떤 로그도 호환 가능.

### 2.6 점화식 — 세 가지 방법

재귀 알고리즘은 `T(n)`을 닫힌 형태로 직접 주지 않고, `T(n) = 2T(n/2) + n` 같은 점화식을 준다. 알고리즘의 복잡도를 알려면 점화식을 **풀어야** 한다.

#### 2.6.1 방법 1 — 반복 대입 (전개)

패턴이 보일 때까지 점화식을 레벨별로 전개한 뒤 기저를 대입.

**예: 병합 정렬.** `T(n) = 2T(n/2) + n`, `T(1) = 1`. `n = 2ᵏ`라 가정.

```
T(n)   = 2T(n/2) + n
       = 2·[2T(n/4) + n/2] + n      = 4T(n/4) + 2n
       = 4·[2T(n/8) + n/4] + 2n     = 8T(n/8) + 3n
       ⋮
       = 2ᵏ·T(n/2ᵏ) + kn
       = n·T(1) + n·log₂ n          (2ᵏ = n ⇒ k = log₂ n이므로)
       = n + n log n
       = Θ(n log n)
```

재귀 트리의 매 레벨이 총 `n`의 일; `log₂ n` 레벨; 곱은 `n log n`. 트리를 그리면 밑바닥 패턴이 보인다.

**예: 팩토리얼.** `T(n) = T(n − 1) + c`, `T(1) ≤ c`.

```
T(n) = T(n−1) + c = T(n−2) + 2c = ... = T(1) + (n−1)c ≤ c + (n−1)c = cn = O(n)
```

단순 등차 급수로 전개.

#### 2.6.2 방법 2 — 추측 후 증명 (귀납)

닫힌 형태를 추측하고 귀납법으로 증명.

**예.** `T(n) = 2T(n/2) + n`이 `T(n) = O(n log n)`을 만족함 증명.

**추측:** 어떤 상수 `c > 0`에 대해 `T(n) ≤ c · n log n`.

**기저:** `T(2) = 2T(1) + 2 = 4`. `4 ≤ c · 2 · log₂ 2 = 2c` 필요, `c ≥ 2`면 충분.

**귀납 단계:** `T(n/2) ≤ c · (n/2) · log(n/2)` 가정. 그러면

```
T(n) = 2T(n/2) + n
     ≤ 2 · c(n/2) log(n/2) + n
     = cn log(n/2) + n
     = cn log n − cn log 2 + n
     = cn log n + (1 − c log 2)n
     ≤ cn log n           (c · log 2 ≥ 1, 즉 c ≥ 1/log 2일 때)
```

`c ≥ max(2, 1/log 2)`를 선택하여 귀납 완료. ∎

> **고전적 함정.** 귀납을 "거의" 끝낼 수는 없다. 귀납 단계에서 `T(n) ≤ cn + 1`이 나오면, 그것은 `≤ cn`이 **아니고** 증명이 실패한 것이다. 해결은 추측을 강화하는 것 — 더 강한 명제 `T(n) ≤ cn − d` (상수 `d`)를 증명하라. 그러면 `T(n) ≤ 2(c(n/2) − d) + 1 = cn − 2d + 1 ≤ cn − d` (`d ≥ 1`일 때). 여분의 여유가 `+1`을 흡수한다.

귀납은 교육학적으로 부담되지만 완전한 엄밀성을 준다. 귀납을 완성하도록 요구하는 시험 문제가 최소 하나는 있을 것으로 예상.

#### 2.6.3 방법 3 — 마스터 정리

`T(n) = a · T(n/b) + f(n)` 형태 (`a ≥ 1, b > 1`), `h(n) = n^(log_b a)`라 할 때, `f(n)`과 `h(n)` 비교:

| Case | `f(n)` 조건 | 해 |
|------|-------------|-----|
| **Case 1** | 어떤 `ε > 0`에 대해 `f(n) = O(n^(log_b a − ε))` | `T(n) = Θ(n^(log_b a))` |
| **Case 2** | `f(n) = Θ(n^(log_b a))` | `T(n) = Θ(n^(log_b a) · log n)` |
| **Case 3** | 어떤 `ε > 0`에 대해 `f(n) = Ω(n^(log_b a + ε))`, **그리고** 정규성 `a · f(n/b) ≤ c · f(n)` (어떤 `c < 1`, 충분히 큰 `n`) | `T(n) = Θ(f(n))` |

**직관 — 재귀 트리.** `T(n) = aT(n/b) + f(n)`의 재귀 트리를 상상하라. 레벨 0이 `f(n)`의 일. 레벨 1은 `a`개 부분 문제, 각자 `f(n/b)`의 일 — 총 `a · f(n/b)`. 레벨 `k`는 `aᵏ`개 부분 문제, 각자 `f(n/bᵏ)`의 일 — 총 `aᵏ · f(n/bᵏ)`. 트리 깊이는 `log_b n`, 리프 수는 `a^(log_b n) = n^(log_b a)` — 여기서 그 신비한 `n^(log_b a)`가 나온다 (리프가 상수 일을 할 때의 총 리프 비용).

- **Case 1.** 결합 비용이 루트에서 리프로 가며 기하적으로 감소; **리프 지배**. 총 = `Θ(#리프수)` = `Θ(n^(log_b a))`.
- **Case 2.** 모든 레벨이 같은 일 `Θ(n^(log_b a))`; `log n` 레벨이므로 **총 = Θ(n^(log_b a) · log n)**.
- **Case 3.** 결합 비용이 리프에서 루트로 가며 증가; **루트 지배**. 총 = `Θ(f(n))`. 정규성이 기하적 구조를 보장.

**풀이 예시 — 요약의 8개 예제 전부, 비교 단계 포함:**

| 점화식 | `a, b, n^(log_b a)` | `f(n)` | 비교 | Case | 결과 |
|--------|----------------------|--------|------|------|------|
| `T(n) = 2T(n/2) + n` | 2, 2, `n` | `n` | `f = Θ(h)` | 2 | `Θ(n log n)` — 병합 정렬 |
| `T(n) = T(n/2) + 1` | 1, 2, `1` | `1` | `f = Θ(h)` | 2 | `Θ(log n)` — 이진 탐색 |
| `T(n) = 2T(n/2) + 1` | 2, 2, `n` | `1` | `1 = O(n^(1−ε))` (어떤 `0 < ε < 1`) | 1 | `Θ(n)` |
| `T(n) = 4T(n/2) + n` | 4, 2, `n²` | `n` | `n = O(n^(2−1))`, `ε = 1` | 1 | `Θ(n²)` |
| `T(n) = 4T(n/2) + n²` | 4, 2, `n²` | `n²` | `f = Θ(h)` | 2 | `Θ(n² log n)` |
| `T(n) = 2T(n/4) + n` | 2, 4, `√n` | `n` | `n = Ω(n^(0.5+0.5))`, 정규성 ✓ | 3 | `Θ(n)` |
| `T(n) = 7T(n/2) + n²` | 7, 2, `n^(log₂ 7) ≈ n^2.81` | `n²` | `n² = O(n^(2.81 − 0.81))` | 1 | `Θ(n^(log₂ 7))` — 스트라센 |
| `T(n) = 3T(n/4) + n²` | 3, 4, `n^(log₄ 3) ≈ n^0.79` | `n²` | `n² = Ω(n^(0.79 + 1.21))`, 정규성 ✓ (`3·(n/4)² = 3n²/16 ≤ c · n²` `c = 3/16 < 1`) | 3 | `Θ(n²)` |

> **시험 팁.** 필기 시험에서 Case 3는 정규성 검증이 번거롭다. Case 1 vs Case 2에 집중 — 둘 다 그냥 `f(n)`과 `n^(log_b a)`를 비교.

**마스터 정리가 풀지 **못하는** 점화식:**
- `T(n) = T(n − 1) + Θ(n)` — `a · T(n/b)` 형태 아님. 대입으로: `Θ(n²)`. (퀵 정렬 최악, 삽입 정렬.)
- `T(n) = T(n − 1) + T(n − 2) + Θ(1)` — MT 형태 아님. 특성 다항식으로: `Θ(φⁿ)` ≈ O(1.618ⁿ). (순진한 재귀 피보나치.)
- `T(n) = T(n/5) + T(7n/10) + Θ(n)` — 불균등 분할. `1/5 + 7/10 = 9/10 < 1`로 `T(n) = O(n)` 귀납 증명. (중앙값의 중앙값.)
- `T(n) = 2T(n/2) + n log n` — `f(n)`이 다항식이 아니라 Case 3 직접 적용 불가. 결과 `Θ(n log² n)` (기본 MT 범위 밖이지만 확장판으로 처리 가능).

## 3. 자료구조와 정렬 (W03)

### 3.1 다섯 가지 기본 구조

| 구조 | 순서 | 주요 연산과 비용 | 전형적 용도 |
|------|------|------------------|-------------|
| **배열 / 리스트** | 인덱스 | 접근 `O(1)`; 삽입/삭제 `O(n)` | 랜덤 접근, 대부분 알고리즘 기반 |
| **연결 리스트** | 순차 | 접근 `O(n)`; 알려진 노드 삽입/삭제 `O(1)` | 연속 메모리 없는 동적 시퀀스 |
| **스택** | **LIFO** | `push`, `pop` — `O(1)` | 함수 호출 스택, 괄호 매칭, DFS |
| **큐** | **FIFO** | `enqueue`, `dequeue` — `O(1)` | BFS, 작업 스케줄링 |
| **힙** | 우선순위 | `insert`, `extract-min/max` — `O(log n)`; 빌드 — `O(n)` | 우선순위 큐, 힙 정렬, 다익스트라, 프림, 허프만 |

**힙 표현 (1-인덱스 배열).** 배열에 저장된 완전 이진 트리. 인덱스 `i`의 노드에 대해:
- 자식: `A[2i]`, `A[2i + 1]`,
- 부모: `A[⌊i/2⌋]`.

힙 속성 (max-heap): 모든 노드에서 `key(부모) ≥ key(자식)`. (min-heap은 쌍대.) 중요: 힙은 **정렬된 배열이 아니다** — 형제 간에는 순서 관계 없음.

**왜 힙을 배열로?** 인덱스 산술이 포인터 추적을 대체하여 모든 연산이 캐시 친화적이고 메모리 효율적. 이것이 힙이 실무에서 단순 포인터 기반 트리보다 빠른 이유.

### 3.2 기본 정렬 — O(n²) 트리오

셋 모두 재귀 구조 `T(n) = T(n − 1) + Θ(n) = Θ(n²)`를 공유 — 매 단계 원소 하나씩 줄이므로 총 일은 삼각 합 `1 + 2 + ... + (n−1) = n(n−1)/2`.

#### 3.2.1 선택 정렬

매 라운드: 미정렬 영역에서 최댓값(또는 최솟값) 찾기, 끝(또는 시작)과 교환, 미정렬 영역을 1 줄임.

```
selectionSort(A[1..n]):
    for last = n downto 2:
        k = A[1..last]에서 최댓값 인덱스
        swap A[k] ↔ A[last]
```

**분석.** 내부 스캔이 `last − 1`회 비교. 총 = `(n−1) + (n−2) + ... + 1 = n(n−1)/2 = Θ(n²)`. *입력 순서와 무관* — 최선 = 평균 = 최악 = `Θ(n²)`. 변하는 것은 교환 횟수(최대 `n − 1`, 이미 정렬되면 0).

**언제 쓰는가.** 실무에서는 거의 안 씀. 단순한 재귀 구조 때문에 교육적으로 좋음.

#### 3.2.2 버블 정렬

매 라운드: 왼쪽에서 오른쪽으로 걸으며 인접 역순 쌍 교환. 최대 미정렬 원소가 끝으로 "떠오름".

```
bubbleSort(A[1..n]):
    for last = n downto 2:
        for i = 1 to last − 1:
            if A[i] > A[i+1]: swap A[i] ↔ A[i+1]
```

**분석.** 최악/평균 `Θ(n²)`. `swapped` 플래그로 **정렬된 입력에서 최선 = Θ(n)**: 교환 0번인 한 번의 패스가 알고리즘 종료. 이 최적화는 실제 구현에서 표준.

#### 3.2.3 삽입 정렬

매 라운드: 다음 원소를 이미 정렬된 접두사에서 올바른 위치에 삽입, 더 큰 원소는 오른쪽으로 이동.

```
insertionSort(A[1..n]):
    for i = 2 to n:
        key = A[i]
        j = i − 1
        while j ≥ 1 and A[j] > key:
            A[j+1] = A[j]
            j = j − 1
        A[j+1] = key
```

**분석.** 최악 역정렬 입력 → `Θ(n²)`. **정렬된 입력에서 최선 = Θ(n)**: 각 키가 제자리에 있음을 확인하는 데 1회 비교만 필요. 이것이 삽입 정렬이 "거의 정렬된" 데이터의 표준 선택이자, 하이브리드 정렬(파이썬 Timsort, C++ introsort)의 작은 부분 배열에서 쓰이는 이유.

**루프 불변식으로 정확성 증명.** 반복 `i` 시작 시점에 `A[1..i−1]` 정렬됨. 기저 `i = 2`: 자명하게 참(단일 원소는 정렬됨). 귀납 단계: `A[i]`를 올바른 위치에 삽입한 뒤 `A[1..i]` 정렬됨. `i = n + 1`에서 종료: `A[1..n]` 정렬됨. 수학적 귀납법을 알고리즘에 직접 적용 — 반복 재사용할 템플릿.

### 3.3 고급 정렬 — O(n log n) 트리오

매 단계 문제를 대략 반으로 나눠 `n log n` 달성: `T(n) = 2T(n/2) + Θ(n)`.

#### 3.3.1 병합 정렬 — 보장 Θ(n log n), 안정, O(n) 공간

```
mergeSort(A, p, r):
    if p < r:
        q = ⌊(p + r) / 2⌋
        mergeSort(A, p, q)
        mergeSort(A, q+1, r)
        merge(A, p, q, r)
```

**병합 단계** (정렬된 두 반을 두 포인터로 걷기, 더 작은 앞을 복사):

```
[3, 8, 31, 65, 73]과 [11, 15, 20, 29, 48] 병합:
  3 < 11 → tmp = [3];
  8 < 11 → tmp = [3, 8];
  31 > 11 → tmp = [3, 8, 11];
  31 > 15 → tmp = [3, 8, 11, 15];
  ...
  결과: [3, 8, 11, 15, 20, 29, 31, 48, 65, 73]
```

각 병합은 `Θ(n)` 일. 점화식은 `Θ(n log n)` (MT Case 2, 또는 재귀 트리로).

**항상 Θ(n log n)**: 최악 = 최선 = 평균. **안정**: 동등한 키가 원래 상대 순서 유지. **트레이드오프**: 병합용 `O(n)` 보조 배열 — 병합 정렬은 제자리가 아님.

#### 3.3.2 퀵 정렬 — 실무에서 가장 빠름, 최악 Θ(n²)

```
quickSort(A, p, r):
    if p < r:
        q = PARTITION(A, p, r)
        quickSort(A, p, q − 1)
        quickSort(A, q + 1, r)

PARTITION(A, p, r):    // 마지막 원소 피벗의 Lomuto 파티션
    pivot = A[r]
    i = p − 1
    for j = p to r − 1:
        if A[j] ≤ pivot:
            i = i + 1
            swap A[i], A[j]
    swap A[i+1], A[r]
    return i + 1
```

**파티션 불변식.** 루프의 매 단계에:
- `A[p..i]`는 `≤ pivot`인 원소 포함,
- `A[i+1..j−1]`은 `> pivot`인 원소 포함,
- `A[j..r−1]`은 미검사,
- `A[r]`은 피벗 자체.

루프 후 `A[i+1]`과 피벗 교환 — 피벗이 최종 정렬 위치로; 불변식이 파티션을 준다.

**풀이 추적:** `[31, 8, 48, 73, 11, 3, 20, 29, 65, 15]`, 피벗 `15`:

```
j=1: A[1]=31 > 15 → 스킵
j=2: A[2]=8 ≤ 15 → i=1, swap A[1]↔A[2]: [8, 31, 48, 73, 11, 3, 20, 29, 65, 15]
j=3: A[3]=48 > 15 → 스킵
j=4: A[4]=73 > 15 → 스킵
j=5: A[5]=11 ≤ 15 → i=2, swap A[2]↔A[5]: [8, 11, 48, 73, 31, 3, 20, 29, 65, 15]
j=6: A[6]=3 ≤ 15 → i=3, swap A[3]↔A[6]: [8, 11, 3, 73, 31, 48, 20, 29, 65, 15]
j=7..9: 모두 > 15 → 스킵
최종 swap A[4]↔A[10]: [8, 11, 3, 15, 31, 48, 20, 29, 65, 73]
피벗 인덱스 4.
```

**복잡도.**
- 최선 / 평균: 파티션이 대략 반 분할 → `T(n) = 2T(n/2) + Θ(n) = Θ(n log n)`.
- 최악: 매 호출마다 `0 : (n−1)` 분할 → `T(n) = T(n−1) + Θ(n) = Θ(n²)`. 마지막 원소 피벗으로 이미 정렬된 / 역정렬 입력에서 발생.

**균형 깨진 분할에서도 평균이 Θ(n log n)인 이유.** 9:1 분할조차 깊이 `log_{10/9}(n) = O(log n)`. 불균형이 상수 비율로 제한되는 한 재귀 트리는 얕음 — 퇴화된 `0 : n−1` 경우만 폭발.

**실무적 해결.** 피벗 랜덤화(균일 랜덤 인덱스 선택, 마지막 위치로 교환 후 파티션). 입력과 무관하게 기대 시간 `O(n log n)`. 또는 **중앙값-3** 휴리스틱으로 `A[p], A[⌊(p+r)/2⌋], A[r]`의 중앙값을 피벗으로.

**퀵 vs 병합 vs 힙:**

| | 병합 | 퀵 | 힙 |
|--|------|----|----|
| 최악 | Θ(n log n) | Θ(n²) | Θ(n log n) |
| 평균 | Θ(n log n) | Θ(n log n) | Θ(n log n) |
| 공간 | **O(n)** | O(log n) | **O(1)** |
| 안정 | **예** | 아니오 | 아니오 |
| 캐시 친화 | 괜찮음 | **훌륭함** | 나쁨 |
| 실무 속도 | 좋음 | **가장 빠름** | 괜찮음 |

작은 상수와 훌륭한 지역성으로 퀵이 실무에서 우승. 안정성이나 최악 보장이 필요하면 병합. 여분 공간 `O(1)`과 최악 보장이 모두 필요하면 힙.

#### 3.3.3 힙 정렬

두 단계: 미정렬 배열로 힙 빌드, 루트 반복 추출.

```
heapSort(A[1..n]):
    buildHeap(A, n)
    for i = n downto 2:
        swap A[1] ↔ A[i]
        heapify(A, 1, i − 1)

buildHeap(A, n):
    for i = ⌊n/2⌋ downto 1:
        heapify(A, i, n)

heapify(A, k, n):           // A[1..n] 안에서 A[k]를 아래로 내림
    left = 2k; right = 2k + 1
    smaller = k  (min-heap; max-heap은 largest)
    if left ≤ n and A[left] < A[smaller]: smaller = left
    if right ≤ n and A[right] < A[smaller]: smaller = right
    if smaller ≠ k:
        swap A[k] ↔ A[smaller]
        heapify(A, smaller, n)
```

**왜 `buildHeap`이 O(n)인가 (O(n log n) 아님).** 순진한 분석은 `n · O(log n) = O(n log n)`. 타이트 분석: 깊이 `h`(리프는 `h = 0`)의 노드에서 heapify는 `O(h)`의 일. 깊이 `h`에는 대략 `n/2^(h+1)`개 노드가 있음. 총 일은

```
Σ_{h=0}^{log n}  (n / 2^(h+1)) · h  =  (n/2) · Σ h / 2^h  =  O(n)
```

`Σ h/2^h`이 상수로 수렴하기 때문. 대부분의 노드는 heapify가 거의 아무 일도 안 하는 리프 근처에 있음.

**왜 `⌊n/2⌋`에서 시작?** 인덱스 `⌊n/2⌋ + 1`부터 `n`까지의 노드는 리프 — 크기 1의 힙으로 이미 자명. 내부 노드에만, 가장 깊은 것에서 루트까지, heapify 실행하면 heapify가 부모에 도달할 때 자식 부분트리가 이미 힙.

**추출 단계.** 루트(min 또는 max)를 마지막 원소와 교환, 힙을 1 줄임, 새 루트를 heapify. 각 추출 `O(log n)`; 총 `n − 1`회 → `O(n log n)`.

**총:** `buildHeap (O(n)) + (n − 1) · heapify (O(log n)) = O(n log n)` 보장, 제자리, 비안정.

### 3.4 비교 기반 Ω(n log n) 하한

**정리.** 임의의 비교 기반 정렬 알고리즘은 최악의 경우 `Ω(n log n)` 비교가 필요하다.

**결정 트리 증명.** 임의의 비교 기반 정렬은 각 내부 노드가 "`A[i] ≤ A[j]`?"를 묻고 각 리프가 특정 출력 순열에 대응하는 이진 결정 트리로 표현 가능. 가능한 순열이 `n!`개이므로 트리는 최소 `n!`개 리프. 리프 `L`개인 이진 트리의 높이는 `≥ ⌈log₂ L⌉`. 따라서 최악 비교 횟수는 `≥ log₂(n!)`. 스털링 근사로 `log₂(n!) = Θ(n log n)`. ∎

**귀결.** 선택, 버블, 삽입, 병합, 퀵, 힙 정렬 — 원소를 비교만 하는 한 `O(n log n)`을 이길 수 없다.

### 3.5 선형 시간 정렬 — Ω(n log n) 우회

원소를 비교하지 않고 **키의 구조**를 이용.

#### 3.5.1 계수 정렬 — 값이 `[1..k]` 범위일 때 O(n + k)

```
countingSort(A[1..n], B[1..n], k):
    for i = 1 to k: C[i] = 0
    for j = 1 to n: C[A[j]]++            // 개수 세기
    for i = 2 to k: C[i] += C[i − 1]      // 누적 합: C[i] = i 이하 원소 수
    for j = n downto 1:
        B[C[A[j]]] = A[j]
        C[A[j]]--
```

**3단계 통찰.** 누적 합 이후 `C[v]`는 "`B`에서 `v`가 가야 할 마지막 인덱스". `A`를 오른쪽에서 왼쪽으로 걸으면, 동등한 키 중 `A`에서 늦은 것이 `B`에서도 늦게 놓임 — 안정성 보존. 왼쪽에서 오른쪽으로 걸으면 동등 키가 역전되어 안정성 깨짐.

**왜 안정성이 중요한가.** 계수 정렬은 종종 기수 정렬의 *서브루틴*. 기수 정렬은 서브루틴이 안정적인 데 의존 — 안정성 없으면 이전 자릿수 순서가 이후 패스에서 파괴됨.

**제한.** 총이 `Θ(n)`이려면 `k = O(n)` 필요. `k = n²`이면 계수 정렬이 비교 정렬보다 느림.

#### 3.5.2 기수 정렬 — `d` 자릿수 키에 Θ(d(n + k))

```
radixSort(A[1..n], k):
    for i = 1 to d:
        A를 i번째 자릿수로 안정 정렬        // 보통 계수 정렬
```

**LSD에서 MSD로** (최하위→최상위 자릿수) 처리. 각 패스가 안정이므로, 이후 패스가 동률일 때 이전(덜 중요한) 패스의 순서가 보존됨.

**왜 MSD부터 안 하나?** MSD부터 하면 최상위 자릿수로 정렬되지만, 각 그룹 내부를 나머지 자릿수로 재귀 정렬해야 — 그것은 다른 알고리즘(MSD 기수 정렬, 더 복잡). 안정 서브루틴과 함께한 LSD 기수 정렬이 단순한 고전.

**`[0123, 2154, 0222, 0004, 0283, 1560, 1061, 2150]` 추적:**

1의 자리, 10의 자리, 100의 자리, 1000의 자리로 정렬하면 완전 정렬된 시퀀스. 각 라운드가 동률일 때 이전 순서 보존.

### 3.6 정렬 완벽 비교

| 알고리즘 | 최악 | 평균 | 최선 | 공간 | 안정 | 핵심 포인트 |
|----------|------|------|------|------|------|-------------|
| 선택 | Θ(n²) | Θ(n²) | Θ(n²) | O(1) | 아니오 | 항상 같음; 최소 교환 횟수 |
| 버블 | Θ(n²) | Θ(n²) | **Θ(n)** | O(1) | 예 | 최선 Θ(n)은 `swapped` 플래그 필요 |
| 삽입 | Θ(n²) | Θ(n²) | **Θ(n)** | O(1) | 예 | 거의 정렬된 데이터에 최적 |
| 병합 | Θ(n log n) | Θ(n log n) | Θ(n log n) | O(n) | 예 | 보장 + 안정 |

## 4. 분할 정복 (W04)

### 4.1 3단계 패러다임

```
DivideAndConquer(P):
    if P가 충분히 작으면: 직접 해결 (기저)
    else:
        DIVIDE:   P를 부분 문제 P₁, ..., Pₖ로 분할
        CONQUER:  각 Pᵢ를 재귀적으로 해결
        COMBINE:  부분 해들을 P의 해로 결합
```

**양보 불가 전제.** 부분 문제들이 **독립적** — 입력 크기의 합이 최대 원본. 겹치면(같은 부분 문제가 여러 번 해결됨) 지수 중복 작업을 하는 것이므로 DP로 전환해야.

일반 점화식 `T(n) = a · T(n/b) + f(n)`. 이것이 마스터 정리의 정확한 템플릿.

### 4.2 D&C 알고리즘 카탈로그

| 알고리즘 | 분할 | 정복 | 결합 | 점화식 | 시간 |
|----------|------|------|------|--------|------|
| 병합 정렬 | 반으로 쪼갬 | 각 반 정렬 | 병합 | `2T(n/2) + Θ(n)` | Θ(n log n) |
| 이진 탐색 | 중간과 비교 | 한쪽 재귀 | 자명 | `T(n/2) + O(1)` | Θ(log n) |
| 퀵 정렬 (평균) | 파티션 | 양쪽 재귀 | 자명 | `2T(n/2) + Θ(n)` | Θ(n log n) |
| 퀵 정렬 (최악) | 파티션 (0 : n−1) | 재귀 | 자명 | `T(n−1) + Θ(n)` | Θ(n²) |
| 랜덤 선택 | 파티션 | **한쪽** 재귀 | 자명 | (평균) `T(3n/4) + Θ(n)` | **Θ(n)** 평균 |
| 중앙값의 중앙값 선택 | 5개씩 그룹, 재귀 중앙값 | 파티션 + 재귀 | 자명 | `T(n/5) + T(7n/10) + Θ(n)` | **Θ(n)** 최악 |
| 최근접 쌍 | x-중앙값으로 분할 | 각 반 재귀 | 스트립 검사 | `2T(n/2) + O(n log n)` | O(n log² n) |
| 스트라센 행렬 곱 | 7번 재귀 2×2 곱 | | | `7T(n/2) + Θ(n²)` | Θ(n^log₂ 7) ≈ Θ(n^2.81) |

### 4.3 선택 문제

정렬되지 않은 `n` 원소 배열 `A`와 정수 `i`가 주어졌을 때, `i`번째 작은 원소 찾기.

**자명한 접근:** `A`를 `O(n log n)`에 정렬 후 `A[i]` 반환. 더 잘할 수 있나?

#### 4.3.1 랜덤 선택 — 기대 Θ(n)

```
SELECT(A, p, r, i):
    if p == r: return A[p]
    q = PARTITION(A, p, r)         // 랜덤 피벗 Lomuto 파티션
    k = q − p + 1                   // A[p..r] 내 피벗 순위
    if i == k: return A[q]
    else if i < k: return SELECT(A, p, q − 1, i)
    else:          return SELECT(A, q + 1, r, i − k)
```

퀵 정렬과 같은 파티션, 단 순위 `i`를 포함하는 **한쪽에만** 재귀.

**기대 시간이 Θ(n)인 이유.** 랜덤 피벗이 중간 절반(25–75 백분위)에 확률 ≥ 1/2로 떨어짐. "좋은" 분할 조건 하에 재귀 호출은 최대 `3n/4`. 평균적으로,

```
T(n) ≤ T(3n/4) + Θ(n)
     ≤ cn + c(3n/4) + c(3/4)²n + ... = cn · 4 = O(n)
```

기하 급수. `Ω(n)`은 자명(모든 원소 검사 필요)하므로 `T(n) = Θ(n)`.

**최악.** 피벗이 항상 최소 또는 최대면 `T(n) = T(n−1) + Θ(n) = Θ(n²)` — 퀵 정렬 최악과 동일. 랜덤화가 이를 거의 제거.

#### 4.3.2 중앙값의 중앙값 — 최악 Θ(n)

고전적 결정적 선형 시간 알고리즘:

```
LINEAR-SELECT(A, p, r, i):
    1. if |A[p..r]| ≤ 5: 정렬 후 i번째 반환
    2. 원소를 ⌈n/5⌉ 그룹 (5개씩)으로 분할
    3. 각 그룹 중앙값 찾기 (정렬 후 중간) — O(n)
    4. M = LINEAR-SELECT로 중앙값들의 중앙값 찾기
    5. M 기준 A 파티션
    6. 적절한 쪽에 재귀
```

**왜 5개씩.** 4단계 후 `⌈n/5⌉` 그룹 중앙값 중 최소 절반이 `≤ M`. 각 그룹은 최소 3개 원소 `≤ M` 기여(그룹 중앙값 + 그룹 내 2개 작은 것). 따라서 최소

```
3 · ⌈⌈n/5⌉ / 2⌉ ≈ 3n/10
```

원소가 `≤ M`. 대칭으로 최소 `3n/10`개가 `≥ M`. 따라서 6단계 재귀는 최대 `n − 3n/10 = 7n/10`개에 대해.

**점화식.** `T(n) = T(n/5) + T(7n/10) + Θ(n)`. `1/5 + 7/10 = 9/10 < 1`이므로 "분수 합 < 1 ⇒ O(n)" 원칙에 의해 **선형**. 귀납으로 증명: `T(n) ≤ cn` 추측, 상수가 맞도록 `c` 선택.

**왜 정확히 5?** 3개씩이면 `T(n) = T(n/3) + T(2n/3) + Θ(n)`, `1/3 + 2/3 = 1` — 기하 합이 발산, 선형 아님. 7개씩도 작동하지만 그룹 내 정렬 시간 낭비. 5가 선형 시간을 사는 최소 크기.

**실무 노트.** 중앙값의 중앙값은 점근적으로 최적이지만 상수가 크다. 실무에서는 랜덤 선택이 이김.

### 4.4 O(n log² n) 최근접 쌍

평면의 `n`점에서 유클리드 거리 최소 쌍 찾기.

**브루트 포스:** 모든 `C(n, 2) = n(n−1)/2`거리 → `O(n²)`.

**분할 정복:**

1. **전처리:** x좌표로 점 정렬 — `O(n log n)`.
2. **분할:** x 중앙값으로 왼쪽/오른쪽 반.
3. **정복:** 재귀로 `d_L` (왼쪽 반 최근접), `d_R` (오른쪽 반 최근접).
4. **결합:** `d = min(d_L, d_R)`. 분할선 주위 폭 `2d` **스트립**에서 `d`보다 가까운 교차 쌍 검사.
5. **반환:** `d_L`, `d_R`, 최선 스트립 거리의 최솟값.

**스트립 검사 — 영리한 부분.** 스트립 점을 y로 정렬. 스트립의 각 점 `P`에 대해, `P` 위로 y 거리 `d` 이내 점만 `d`보다 가까울 수 있음. 더 나아가, `P` 위 너비 `2d` × 높이 `d` 직사각형에는 최대 7개 다른 점이 존재 — 따라서 각 점의 다음 7개 y 이웃만 검사.

**왜 최대 7개 다른 점?** `2d × d` 직사각형을 `(d/2) × (d/2)` 크기 8개 셀로 재분할. 같은 셀의 두 점은 거리 `≤ (d/2) · √2 < d`로 `d_L, d_R ≥ d`에 모순. 따라서 각 셀에 최대 1개 점 → 직사각형 최대 8개 점, 즉 자신 + 최대 7개 다른 점.

**복잡도.** 각 재귀 호출의 스트립 검사는 `O(n)` (기본판에서는 호출당 y 정렬 `O(n log n)`). 점화식: `T(n) = 2T(n/2) + O(n log n) → T(n) = O(n log² n)`. 재귀 내내 y 정렬 순서를 유지하는 더 신중한 구현은 `O(n log n)` 달성.

### 4.5 D&C가 잘못된 도구일 때

고전적 경우: 피보나치. 순진한 재귀 `F(n) = F(n−1) + F(n−2)`는 **부분 문제 크기 합이 원본보다 큼** (`(n−1) + (n−2) = 2n − 3`), 같은 부분 문제가 지수적으로 재계산. 시간: `Θ(φⁿ) ≈ O(1.618ⁿ)`.

**증상.** 부분 문제가 겹치면(부분해 공유), D&C가 지수 시간 낭비. **해결:** DP — 각 부분 문제 한 번 풀고, 저장, 재사용.

```
fibDP(n):
    F[0] = 0; F[1] = 1
    for i = 2 to n: F[i] = F[i−1] + F[i−2]
    return F[n]
```

`Θ(n)` 시간, `Θ(n)` 공간 (마지막 두 값만 유지하면 `O(1)`로 축소 가능).

## 5. 그리디 알고리즘 (W05)

### 5.1 그리디 패러다임

그리디 알고리즘은 각 단계에서 **지역 최적** 선택을 하고 재고하지 않는 방식으로 **최적화 문제**를 푼다. 단순하고 빠르지만 보통 틀림 — 문제에 지역 선택을 안전하게 만드는 특별한 구조가 있을 때를 제외하고.

```
Greedy(C):
    S = {}                                       // 부분 해
    while C ≠ ∅ and S가 완전한 해가 아님:
        x = C에서 "가장 좋아 보이는" 원소
        C = C − {x}
        if S ∪ {x}가 실행 가능:
            S = S ∪ {x}
    return S
```

### 5.2 정확성을 위한 두 조건

1. **그리디 선택 속성.** 그리디 선택을 포함하는 최적해가 존재. **교환 논증**으로 증명: 그리디 선택을 포함하지 않는 임의 최적해를 취해, 목적값을 악화시키지 않고 원소를 그리디 선택과 교환.

2. **최적 부분 구조.** 그리디 선택 후 남은 부분 문제의 최적해가 전체 최적해에 포함.

두 속성 모두 주어진 문제에 대해 **증명**되어야 — 직관만으로는 불충분.

### 5.3 그리디 실패 — 대표 반례

- **동전 `{1, 3, 4}`, 금액 6.** 그리디: `4 + 1 + 1 = 3`개. 최적: `3 + 3 = 2`. 그리디 선택 속성 실패.
- **동전 `{16, 10, 5, 1}`, 금액 20.** 그리디: `16 + 1 + 1 + 1 + 1 = 5`. 최적: `10 + 10 = 2`. 첫 큰 선택에 그리디가 묶임.
- **가치/무게로 0/1 배낭.** 물건 `(w=10, v=60), (w=20, v=100), (w=30, v=120)`, 용량 50. 그리디 물건 1(최고 비율) → `60 + 100 = 160`. 최적 물건 2+3 → `220`. 그리디가 용량 낭비 가능.
- **이진 트리 최대 경로 합.** 더 큰 자식으로 그리디 하강이 전역 최적 루트-리프 경로 놓칠 수 있음.

**교훈.** 그리디 알고리즘 신뢰 전에 반례를 먼저 찾아라. 못 찾으면 그리디 선택 속성을 증명.

### 5.4 그리디 알고리즘 카탈로그

| 문제 | 전략 | 최적? | 시간 |
|------|------|-------|------|
| 동전 거스름돈 (표준) | 큰 동전부터 | ✓ 배수 관계일 때 | O(k) |
| **분할 가능 배낭** | 가치/무게로 정렬 후 탐욕적 | ✓ 항상 | O(n log n) |
| 0/1 배낭 | — | ✗ DP 사용 | — |
| **작업 스케줄링** (기계 최소) | 가장 빠른 시작 시간 | ✓ | O(n log n) + O(mn) |
| **활동 선택** (한 기계, 활동 최대) | **가장 빠른 종료 시간** | ✓ | O(n log n) |
| **허프만 코딩** | 최저 빈도 두 노드 병합 | ✓ 최적 접두 코드 | O(n log n) |
| **크러스컬 MST** | 사이클 없는 최소 간선 | ✓ | O(m log m) |
| **프림 MST** | 현재 트리 떠나는 최소 간선 | ✓ | O(n²) 배열 / O(m log n) 힙 |
| **다익스트라 최단 경로** | 가장 가까운 미확정 정점, 완화 | ✓ 음 아닌 가중치 | O(n²) / O(m log n) |

### 5.5 작업 스케줄링 vs 활동 선택 — 고전적 혼동

학생들이 이 둘을 혼동한다. **다른** 문제이고 **다른** 최적 전략이다.

|                 | **작업 스케줄링** | **활동 선택** |
|-----------------|-------------------|----------------|
| 기계 | 많음 | **하나** |
| 목표 | **모든** 작업 실행에 필요한 기계 수 최소화 | 들어갈 수 있는 작업 수 최대화 |
| 전략 | **가장 빠른 시작 시간** | **가장 빠른 종료 시간** |
| 이유 | 부하 균형; 기계가 조기에 놀지 않음 | 일찍 끝내면 미래 활동을 위한 공간 최대 |

**활동 선택 증명 스케치 (교환 논증).** 가장 빠른 종료 활동을 `g`라 하자. 최적해 `OPT`가 `g`를 포함하지 않는다고 가정. `OPT`에서 가장 빨리 끝나는 활동을 `a`라 하자. `g`가 `a`보다 늦지 않게 끝나므로, `OPT`의 다른 활동을 무효화하지 않고 `a`를 `g`로 교환 가능 — 남은 활동은 모두 `a`의 종료 이후에 시작하고, 이는 `g`의 종료 이후. 교환은 같은 크기의 해이면서 `g`를 포함. ∎

### 5.6 허프만 코딩 — 최적 접두 코드

**설정.** 빈도가 주어진 문자들. 각 문자에 가변 길이 이진 코드 부여. 빈번한 문자는 짧은 코드; 드문 문자는 긴 코드. 코드는 **접두 자유**(prefix-free): 어떤 코드도 다른 코드의 접두가 아님(모호하지 않은 디코딩 필요).

**알고리즘.**
```
Huffman(chars, freqs):
    Q = 리프 노드(문자당 하나) 최소 우선순위 큐
    while |Q| ≥ 2:
        A = extract-min(Q)
        B = extract-min(Q)
        N = 새 내부 노드, left=A, right=B, freq = A.freq + B.freq
        N을 Q에 삽입
    return Q[루트]
```

**풀이 예시.** 빈도 `A:450, C:270, G:120, T:90`.

```
1단계: Q = [T:90, G:120, C:270, A:450]
       T(90) + G(120) 병합 → N1(210)

2단계: Q = [N1:210, C:270, A:450]
       N1(210) + C(270) 병합 → N2(480)

3단계: Q = [A:450, N2:480]
       A(450) + N2(480) 병합 → Root(930)
```

트리:
```
           Root:930
           /      \
       A:450      N2:480
                  /    \
               N1:210   C:270
               /   \
            T:90   G:120
```

코드 (왼쪽 = 0, 오른쪽 = 1): `A = 0`, `C = 10`, `T = 110`, `G = 111`.

**압축비:**

| 문자 | 빈도 | 코드 길이 | 비트 |
|------|------|-----------|------|
| A | 450 | 1 | 450 |
| C | 270 | 2 | 540 |
| T | 90 | 3 | 270 |
| G | 120 | 3 | 360 |
| 총 930자 | | | **1,620 비트** |

순진한 8비트 ASCII: 930 × 8 = 7,440 비트. 허프만은 1,620 / 7,440 ≈ 22% — 약 5배 압축.

**왜 최적인가.** 두 최저 빈도 심볼에 대한 교환 논증으로 증명: 임의의 최적 접두 코드 트리에서, 두 가장 깊은 리프가 두 최저 빈도 심볼이고 형제인 최적해가 존재. 따라서 이들을 먼저 병합하는 것이 안전.

### 5.7 MST 알고리즘

#### 5.7.1 크러스컬 알고리즘 — 간선 중심

가중치로 간선 정렬, 사이클을 만들지 않는 한 각 간선 추가, `n − 1`개 간선까지.

```
Kruskal(G):
    가중치 오름차순으로 간선 정렬
    T = {}
    정렬 순서의 각 간선 (u, v)에 대해:
        if Find(u) ≠ Find(v):           // u와 v가 다른 성분
            T = T ∪ {(u, v)}
            Union(u, v)
    return T
```

**왜 작동하는가.** MST의 "컷 속성": 그래프의 임의 컷에 대해, 컷을 가로지르는 최소 가중치 간선은 어떤 MST에 속함. 크러스컬은 항상 사이클을 만들지 않는 최소 간선을 추가 — 이 간선은 그 끝점의 성분 사이 컷을 가로지르며, 따라서 어떤 MST에 반드시 속함.

**유니온-파인드.** 랭크 기반 유니온 + 경로 압축으로 각 `Find`와 `Union`이 거의 `O(1)` (엄밀히 `O(α(n))`, 역 애커만 — 실제 사용할 어떤 `n`에 대해서도 ≤ 4).

**복잡도.** 간선 정렬이 지배: `O(m log m) = O(m log n)` (`m ≤ n²`이므로 `log m = O(log n)`).

#### 5.7.2 프림 알고리즘 — 정점 중심

임의 정점에서 시작해 단일 트리를 성장, 항상 트리를 확장하는 최소 간선 추가.

```
Prim(G, start):
    D[v] = ∞ 모든 v; D[start] = 0
    T = ∅
    while T ≠ V:
        u = T에 없고 D[u]가 최소인 정점
        T = T ∪ {u}
        T에 없는 u의 각 이웃 w에 대해:
            if weight(u, w) < D[w]: D[w] = weight(u, w)
    return T (그리고 사용된 간선)
```

**왜 작동하는가.** 같은 컷 속성. 매 반복에서 선택된 간선은 `T`와 `V − T` 사이 컷을 가로지르며 구성상 가장 가벼움.

**복잡도.** 최소 거리 정점을 선형 스캔하면 `O(n²)`; 이진 힙으로 `O(m log n)` (각 간선이 최대 한 번의 `decrease-key` 유발).

#### 5.7.3 언제 어느 것?

- **크러스컬:** `m`이 `n`에 가까운 희소 그래프. `m` 간선 정렬이 저렴.
- **프림:** `m ≈ n²`인 밀집 그래프. 간선 정렬 없는 선형 스캔이 빠름.

작은 그래프에 대한 시험 문제에서는 (간선 가중치가 보통 고유하므로) 둘 다 같은 MST를 주고 순서만 다름.

### 5.8 다익스트라 최단 경로

**음이 아닌** 간선 가중치 가정 하에 소스 `s`에서 모든 다른 정점까지 최단 거리 찾기.

```
Dijkstra(G, s):
    D[v] = ∞ 모든 v; D[s] = 0
    S = {}
    while S ≠ V:
        u = S에 없고 D[u]가 최소인 정점
        S = S ∪ {u}
        S에 없는 u의 각 이웃 w에 대해:
            if D[u] + weight(u, w) < D[w]:      // 간선 완화
                D[w] = D[u] + weight(u, w)
    return D
```

**간선 완화**가 핵심 단계: `u`가 정확한 거리 `D[u]`로 확정되었을 때, `u`를 거치는 경로가 현재 알려진 `w` 거리를 개선하는지 확인.

**그리디 불변식 (왜 작동).** `u`가 최소 거리 미확정 정점으로 선택될 때 `D[u]`는 실제 최단 거리. 귀류법 증명: `u`로 가는 더 짧은 경로는 아직 미확정인 어떤 정점 `y`를 지나야. 그러면 `D[y] ≥ D[u]` (`u`가 최소였으므로), 따라서 `y`를 경유하는 경로 길이는 `≥ D[y] + (음 아닌 꼬리) ≥ D[u]`, 모순. 음이 아닌 가중치 가정이 이 논증의 사활.

**왜 음수 간선이 다익스트라를 깨뜨리나.** 미확정 정점 `y`에서 나가는 간선이 음 가중치이면, 경로 `y → u`가 `< D[u]`로 `u`에 도달할 수 있음 — 그러나 `u`는 이미 확정. 음수 간선 그래프(그러나 음수 사이클 없음)에는 **벨만-포드**(`O(VE)`) 사용.

**풀이 반례.** 방향 간선 `s→a = 4, s→b = 2, a→b = −5`.
- `s` 확정. 완화: `D[a] = 4, D[b] = 2`.
- `b` 선택 (최소 = 2). 확정. 나가는 간선 없음.
- `a` 선택 (D = 4). 확정. `a → b` 완화 시도: `D[b] = −1`이 되어야 하지만, `b`가 이미 확정 → 갱신 무시.
- 다익스트라 `D[b] = 2` 반환. 진실: `s → a → b = −1`.

## 6. 동적 프로그래밍 (W06)

### 6.1 DP란 정확히 무엇인가

동적 프로그래밍은 다음으로 문제를 푼다:

1. **중복** 부분 문제로 분해,
2. 각 부분 문제를 **한 번** 풀고 결과 저장,
3. 작은 것에서 큰 것으로 쌓아 올림,
4. 테이블에서 답 추출.

**필수 속성.**

| 속성 | 의미 | DP에 왜 필요한가 |
|------|------|-------------------|
| **최적 부분 구조** | 전체 문제의 최적해가 부분 문제의 최적해 포함 | 작은 답으로 큰 답 조립 가능 |
| **중복 부분 문제** | 같은 부분 문제가 여러 번 필요 | 메모이제이션이 이득 — 중복 없으면 순수 D&C로 충분 |

**"프로그래밍"은 테이블화를 의미.** 리처드 벨만이 1950년대 수학적 최적화 연구를 위해 "dynamic programming"을 고안; 이 단어는 컴퓨터 프로그래밍과 무관. "dynamic"은 다단계 의사결정; "programming"은 "표 형태 계획 수립"을 의미.

### 6.2 두 가지 DP 구현 전략

| | 메모이제이션 (하향식) | 타뷸레이션 (상향식) |
|--|----------------------|---------------------|
| 스타일 | 재귀 + 캐시 | 반복, 테이블 채움 |
| 구현 난이도 | 보통 더 쉬움 (자연 재귀에 캐시 추가) | 부분 문제의 명시적 순서 필요 |
| 해결 부분 문제 | 실제 필요한 것만 (lazy) | 범위 내 **모든** 부분 문제 |
| 오버헤드 | 재귀 스택 + 캐시 조회 | 타이트 루프, 캐시 친화적 메모리 접근 |
| 보통 승자 | 필요한 부분 문제가 적을 때 | 대부분 필요할 때 |

중간고사의 고전 문제(피보나치, LCS, 편집 거리, 배낭, 플로이드-워셜)에서는 둘 다 작동; 타뷸레이션이 제시하기 더 깔끔.

### 6.3 DP 레시피 — 6단계

1. **부분 문제 정의.** `OPT[i, j, ...]`가 무엇을 나타내는가? 완전한 한국어 문장으로 쓴다.
2. **점화식 작성.** 더 작은 부분 문제로 `OPT`를 표현. 모든 경우 포괄(일치/불일치, 담기/버리기, 머물기/이동).
3. **기저 조건 식별.** 자명한 값은? (보통 `OPT[0, ...] = 0` 또는 `∞` 또는 자연 경계.)
4. **계산 순서 결정.** 의존성이 그것을 필요로 하는 셀보다 먼저 채워져야 (2D 테이블은 좌상→우하, 플로이드-워셜은 `k` 가장 바깥).
5. **답 추출.** 어느 셀이 최종 값을 담는가?
6. **(선택) 역추적.** 각 셀 값으로 이끈 선택을 따라 실제 해(어느 물건, 어느 경로) 복원.

이 레시피가 DP를 "영리한 트릭"에서 체계적 절차로 바꾼다.

### 6.4 고전 DP 문제 — 완전 카탈로그

#### 6.4.1 피보나치 (예열)

- 부분 문제: `f[i]` = `i`번째 피보나치 수.
- 점화식: `f[i] = f[i−1] + f[i−2]`.
- 기저: `f[1] = f[2] = 1`.
- 순서: `i = 3, 4, ..., n`.
- 답: `f[n]`.
- 시간: `O(n)`. 공간: `O(n)`, 두 이전 값만 유지해 `O(1)`로 축소 가능.

`Θ(φⁿ)` (순진한 재귀)에서 `Θ(n)` (DP)으로 — DP 가치의 전형적 시연.

#### 6.4.2 행렬 경로 (최대 합)

**문제.** 양의 정수 `n × n` 행렬이 주어질 때, `(1,1)`에서 `(n,n)`까지 (오른쪽 또는 아래로만 이동) 합 최대화 경로 찾기.

- 부분 문제: `c[i, j]` = `(1,1)`에서 `(i,j)`까지 임의 경로의 최대 합.
- 점화식: `c[i, j] = m[i, j] + max(c[i−1, j], c[i, j−1])`.
- 기저: `c[i, 0] = c[0, j] = 0`.
- 순서: 행별 (또는 열별).
- 답: `c[n, n]`.
- 시간 / 공간: `O(n²)`.

**추적 (4×4 예제):**

```
m = | 6  7 12  5 |            c = | 0  0  0  0  0 |  (0 행/열로 패딩)
    | 5  3 11 18 |                | 0  6 13 25 30 |
    | 7 17  3  3 |                | 0 11 16 36 54 |
    | 8 10 14  9 |                | 0 18 35 39 57 |
                                  | 0 26 45 59 68 |

답: c[4, 4] = 68.
```

#### 6.4.3 최장 공통 부분 수열 (LCS)

**문제.** 문자열 `X`, `Y`가 주어졌을 때, 둘 다에 부분 수열로 나타나는(순서 유지, 연속 불필요) 가장 긴 수열.

- 부분 문제: `c[i, j]` = `X₁..ᵢ`와 `Y₁..ⱼ`의 LCS 길이.
- 점화식:
  - `X[i] == Y[j]`면: `c[i, j] = c[i−1, j−1] + 1` (일치 연장).
  - 아니면: `c[i, j] = max(c[i−1, j], c[i, j−1])` (한 문자 제거 후 재시도).
- 기저: `c[i, 0] = c[0, j] = 0`.
- 순서: 행별.
- 답: `c[m, n]`.
- 시간 / 공간: `O(mn)`, 길이만 필요하면 공간을 `O(min(m, n))`으로 축소 가능.

**풀이 추적 — `X = "ABCB"`, `Y = "BDCAB"`:**

|     |   | B | D | C | A | B |
|-----|---|---|---|---|---|---|
|     | 0 | 0 | 0 | 0 | 0 | 0 |
| A   | 0 | 0 | 0 | 0 | **1** | 1 |
| B   | 0 | **1** | 1 | 1 | 1 | **2** |
| C   | 0 | 1 | 1 | **2** | 2 | 2 |
| B   | 0 | 1 | 1 | 2 | 2 | **3** |

LCS 길이 = 3. 한 LCS: `"BCB"`.

**역추적 규칙.** `c[m, n]`에서:
- `X[i] == Y[j]`면: 이 문자가 LCS에 있음; `(i−1, j−1)`로 이동.
- 아니면: `c[i−1, j]`와 `c[i, j−1]` 중 큰 쪽으로 이동 (동률이면 아무 쪽).

#### 6.4.4 편집 거리 (레벤슈타인)

**문제.** 문자열 `X`, `Y`가 주어졌을 때, `X`를 `Y`로 바꾸는 최소 단일-문자 연산(삽입, 삭제, 치환) 수.

- 부분 문제: `E[i, j]` = `X₁..ᵢ`와 `Y₁..ⱼ` 사이 편집 거리.
- 점화식:
  - `X[i] == Y[j]`면: `E[i, j] = E[i−1, j−1]` (무료 — 연산 없음).
  - 아니면: `E[i, j] = 1 + min(E[i−1, j], E[i, j−1], E[i−1, j−1])`.
    - `E[i−1, j] + 1`: `X[i]` 삭제.
    - `E[i, j−1] + 1`: `Y[j]` 삽입.
    - `E[i−1, j−1] + 1`: `X[i]`를 `Y[j]`로 치환.
- 기저: `E[i, 0] = i`, `E[0, j] = j` (모두 삭제, 또는 모두 삽입).
- 순서: 행별.
- 답: `E[m, n]`.
- 시간 / 공간: `O(mn)`.

**추적 — "kitten" → "sitting":**

| E   | ε | s | i | t | t | i | n | g |
|-----|---|---|---|---|---|---|---|---|
| ε   | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
| k   | 1 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
| i   | 2 | 2 | 1 | 2 | 3 | 4 | 5 | 6 |
| t   | 3 | 3 | 2 | 1 | 2 | 3 | 4 | 5 |
| t   | 4 | 4 | 3 | 2 | 1 | 2 | 3 | 4 |
| e   | 5 | 5 | 4 | 3 | 2 | 2 | 3 | 4 |
| n   | 6 | 6 | 5 | 4 | 3 | 3 | 2 | 3 |

`E[6, 7] = 3`: 세 연산 (치환 `k → s`, 치환 `e → i`, 삽입 `g`).

**응용.** 맞춤법 검사, DNA 정렬, `diff` 유틸리티, 퍼지 검색.

#### 6.4.5 0/1 배낭

**문제.** `n`개 물건, 각각 무게 `wᵢ`, 가치 `vᵢ`. 배낭 용량 `C`. 물건 분할 불가. 총 가치 최대화.

- 부분 문제: `K[i, w]` = 물건 `1..i` 사용, 용량 `w`일 때 최대 가치.
- 점화식:
  - `wᵢ > w`면: `K[i, w] = K[i−1, w]` (물건 안 들어감).
  - 아니면: `K[i, w] = max(K[i−1, w], K[i−1, w − wᵢ] + vᵢ)` (버리기 또는 담기).
- 기저: `K[0, w] = 0`, `K[i, 0] = 0`.
- 순서: `i = 1..n`, `w = 1..C`.
- 답: `K[n, C]`.
- 시간 / 공간: `O(nC)`.

**풀이 예시.** 물건 `(w, v) = (2, 12), (1, 10), (3, 20), (2, 15)`, 용량 `C = 5`.

| i \ w | 0 | 1 | 2 | 3 | 4 | 5 |
|-------|---|---|---|---|---|---|
| 0 | 0 | 0 | 0 | 0 | 0 | 0 |
| 1 (2, 12) | 0 | 0 | 12 | 12 | 12 | 12 |
| 2 (1, 10) | 0 | 10 | 12 | 22 | 22 | 22 |
| 3 (3, 20) | 0 | 10 | 12 | 22 | 30 | 32 |
| 4 (2, 15) | 0 | 10 | 15 | 25 | 30 | **37** |

`K[4, 5] = 37`. 역추적: 물건 4 담김(`K[4,5] > K[3,5]`이 물건 4 때문), 차감; `K[3, 3] = 22`는 `K[2, 3]`에서 옴(물건 3 미담김); `K[2, 3] = 22`는 `K[1, 2] + 10`에서 옴(물건 2 담김); `K[1, 2] = 12`는 물건 1 담김. 물건 `{1, 2, 4}`, 무게 `2+1+2 = 5`, 가치 `12+10+15 = 37`. ✓

**왜 O(nC)는 "의사다항시간"인가.** *입력 크기*(용량 `C` 인코딩 비트 수)는 `O(log C)`. 알고리즘 실행 시간은 `O(nC) = O(n · 2^(log C))` — 입력 비트 길이에 대해 지수. 0/1 배낭은 고전적 복잡도 감각에서 **NP-hard**; DP는 `C`의 수치 값 관점에서만 다항.

#### 6.4.6 동전 거스름돈 (최소 동전)

**문제.** 화폐 `d₁ > d₂ > ... > dₖ = 1`. 금액 `n`을 최소 동전 수로 만들기.

- 부분 문제: `C[j]` = 금액 `j` 만드는 최소 동전 수.
- 점화식: `C[j] = min_{dᵢ ≤ j} { C[j − dᵢ] + 1 }`.
- 기저: `C[0] = 0`.
- 순서: `j = 1..n`.
- 답: `C[n]`.
- 시간: `O(nk)`. 공간: `O(n)`.

**추적 — `d = {16, 10, 5, 1}`, 금액 20.**

```
C[0] = 0
C[1] = C[0] + 1 = 1
C[2] = 2
C[3] = 3
C[4] = 4
C[5] = min(C[4]+1, C[0]+1) = 1      (5 사용)
C[6] = 2
C[7] = 3
C[8] = 4
C[9] = 5
C[10] = min(..., C[0]+1) = 1         (10 사용)
...
C[16] = min(..., C[0]+1) = 1         (16 사용)
...
C[20] = min(C[19]+1, C[15]+1, C[10]+1, C[4]+1) = min(4, 3, 2, 5) = 2    (10 두 개)
```

그리디는 `16 + 1 + 1 + 1 + 1 = 5` 선택. DP는 올바르게 `10 + 10 = 2` 발견.

#### 6.4.7 플로이드-워셜 (모든 쌍 최단 경로)

**문제.** 방향 가중 그래프. 모든 정점 쌍 사이 최단 거리 찾기.

- 부분 문제: `d[i, j, k]` = `{1..k}`의 중간 정점만 사용한 `i`에서 `j`로의 최단 거리.
- 점화식: `d[i, j, k] = min( d[i, j, k−1], d[i, k, k−1] + d[k, j, k−1] )`.
- 기저: 간선 있으면 `d[i, j, 0] = w(i, j)`, 없으면 `∞`; `d[i, i, 0] = 0`.
- 답: 모든 쌍에 대해 `d[i, j, n]`.
- 시간: `O(n³)`. 공간: `O(n²)` (`k` 차원 없이 구현 가능).

```
FloydWarshall(W, n):
    D = W
    for k = 1 to n:            // ← 가장 바깥, 양보 불가
        for i = 1 to n:
            for j = 1 to n:
                D[i, j] = min(D[i, j], D[i, k] + D[k, j])
    return D
```

**왜 `k`가 가장 바깥이어야 하는가.** `k` 반복은 현재 `D`가 중간 정점 `{1..k−1}`을 사용한 최단 거리를 담고 있다고 가정. 그 반복이 완료된 후에만 `D`가 중간 정점 `{1..k}`에 대해 유효. 루프 순서를 바꾸면 불변식이 파괴되어 잘못된 답.

**해석.** "`i`에서 `j`로 정점 `k` 경유가 현재 최선보다 짧은가?" 각 반복이 묻는 질문.

### 6.5 DP 패턴과 등장 상황

| 패턴 | 예시 | 점화식 형태 |
|------|------|-------------|
| 1D DP | 피보나치, 동전 거스름돈 | `DP[i]`는 `DP[i−1], DP[i−2], ...`에 의존 |
| 문자열 2D DP | LCS, 편집 거리 | `DP[i, j]`는 `DP[i−1, j], DP[i, j−1], DP[i−1, j−1]`에 의존 |
| (물건, 용량) 2D DP | 0/1 배낭 | `DP[i, w]`는 `DP[i−1, w], DP[i−1, w−wᵢ]`에 의존 |
| 격자 2D DP | 행렬 경로 | `DP[i, j]`는 `DP[i−1, j], DP[i, j−1]`에 의존 |
| (소스, 목적지, 중간 집합) 3D DP | 플로이드-워셜 | `DP[i, j, k]`는 `DP[i, j, k−1], DP[i, k, k−1], DP[k, j, k−1]`에 의존 |

패턴을 인식하면 점화식이 저절로 써진다.

## 7. 패러다임 비교

### 7.1 D&C vs 그리디 vs DP — 나란히

| 측면 | 분할 정복 | 그리디 | 동적 프로그래밍 |
|------|-----------|--------|-----------------|
| 접근 | 분할, 재귀, 결합 | 각 단계 지역 최적 선택 | 모든 부분 문제 풀고 결합 |
| 부분 문제 | **독립** (트리) | N/A (부분 문제 재귀 없음) | **중복** (DAG) |
| 정확성 | 문제 의존 | 그리디 선택 속성 있을 때만 | 최적 부분 구조 있으면 보장 |
| 전형 시간 | `O(n log n)` | `O(n log n)` 또는 `O(n)` | `O(n²)` 또는 `O(n³)` |
| 공간 | `O(log n)` 재귀 | `O(1)` ~ `O(n)` | `O(n)` ~ `O(n²)` 테이블 |
| 전형 용도 | 정렬, 검색, 기하 문제 | 스케줄링, MST, 최단 경로 | 시퀀스 정렬, 배낭, 체인 최적화 |

### 7.2 결정 흐름 — 이 문제에 어느 패러다임?

```
문제가 최적(min / max / count)을 요구하는가?
├── 아니오 → D&C 또는 브루트 포스
└── 예
    ├── 안전한 그리디 선택(그리디 선택 속성)을 증명할 수 있나?
    │    ├── 예 → 그리디 (그리고 정확성 증명!)
    │    └── 아니오 → 다음
    ├── 부분 문제가 겹치는가(같은 부분 문제가 여러 번 재발)?
    │    ├── 예 → DP
    │    └── 아니오 → D&C
```

**실무 시험 전략.** 먼저 그리디 시도. 증명 못 하거나 반례가 있으면 DP로. 부분 문제가 독립적이고 문제가 자연스러운 재귀 구조를 가지면 D&C.

### 7.3 같은 문제, 다른 패러다임

| 문제 | 그리디 | DP | D&C |
|------|:------:|:--:|:---:|
| 동전 거스름돈 (표준) | ✓ | ✓ (과잉) | — |
| 동전 거스름돈 (임의) | ✗ | ✓ | — |
| 분할 가능 배낭 | ✓ | ✓ (과잉) | — |
| 0/1 배낭 | ✗ | ✓ | — |
| 최단 경로 (음 아닌 가중치) | ✓ (다익스트라) | ✓ (벨만-포드) | — |
| 최단 경로 (임의 가중치, 음수 사이클 없음) | — | ✓ (BF, 플로이드-워셜) | — |
| 모든 쌍 최단 경로 | — | ✓ (플로이드-워셜) | — |
| 정렬 | — | — | ✓ (병합/퀵) |
| i번째 작은 원소 | — | — | ✓ (선택) |
| MST | ✓ (크러스컬/프림) | — | — |
| 피보나치 | — | ✓ | ✗ (지수) |
| LCS | ✗ | ✓ | — |
| 편집 거리 | ✗ | ✓ | — |

### 7.4 통합 관점

D&C, 그리디, DP는 모두 **최적 부분 구조** — 문제 해가 부분 문제 해로부터 구성될 수 있다는 속성 — 를 활용하는 방법이다.

- **D&C**는 부분 문제가 독립(겹치지 않음)일 때 작동.
- **그리디**는 추가로 단일 지역 최적 선택을 미리보기 없이 안전하게 확정할 수 있을 때 작동.
- **DP**는 부분 문제가 겹치지만 최적 부분 구조가 여전히 성립할 때 작동.

최적 부분 구조 없으면 이 패러다임들 모두 적용 안 됨 — 브루트 포스, 백트래킹, 또는 근사.

## 8. 공식 치트시트

**점근적 표기 단축.** 다항식 `f(n) = aₖnᵏ + ... + a₀`:
- `f(n) = Θ(nᵏ)` — 최고차항 지배.
- `O(nᵏ)` 증명: `c = |aₖ| + ... + |a₀|`, `n₀ = 1`.
- `Ω(nᵏ)` 증명: 충분히 큰 `n`에 대해 `c = |aₖ|/2`.

**마스터 정리 — 빠른 참조.** `T(n) = aT(n/b) + f(n)`, `h(n) = n^(log_b a)`:

| `f(n)` 조건 | Case | 결과 |
|-------------|------|------|
| 어떤 `ε > 0`에 대해 `O(h(n) / n^ε)` | 1 | `Θ(h(n))` |
| `Θ(h(n))` | 2 | `Θ(h(n) log n)` |
| `Ω(h(n) · n^ε)` + 정규성 | 3 | `Θ(f(n))` |

**반드시 외울 점화식:**

| 점화식 | 해 | 예시 |
|--------|-----|------|
| `T(n) = T(n/2) + Θ(1)` | `Θ(log n)` | 이진 탐색 |
| `T(n) = T(n−1) + Θ(1)` | `Θ(n)` | 팩토리얼 |
| `T(n) = 2T(n/2) + Θ(1)` | `Θ(n)` | — |
| `T(n) = 2T(n/2) + Θ(n)` | `Θ(n log n)` | 병합 정렬 |
| `T(n) = 2T(n/2) + Θ(n log n)` | `Θ(n log² n)` | 최근접 쌍 (기본) |
| `T(n) = T(n−1) + Θ(n)` | `Θ(n²)` | 삽입 정렬, 퀵 정렬 최악 |
| `T(n) = 2T(n−1) + Θ(1)` | `Θ(2ⁿ)` | 하노이 탑 |
| `T(n) = T(n−1) + T(n−2) + Θ(1)` | `Θ(φⁿ)` ≈ O(1.618ⁿ) | 순진한 피보나치 |
| `T(n) = T(n/5) + T(7n/10) + Θ(n)` | `Θ(n)` | 중앙값의 중앙값 |

**반드시 외울 DP 점화식:**
- **LCS:** 일치면 `c[i,j] = c[i−1, j−1] + 1`; 아니면 `max(c[i−1, j], c[i, j−1])`.
- **편집 거리:** 일치면 `E[i,j] = E[i−1, j−1]`; 아니면 `1 + min(E[i−1, j], E[i, j−1], E[i−1, j−1])`.
- **0/1 배낭:** `wᵢ ≤ w`면 `K[i, w] = max(K[i−1, w], vᵢ + K[i−1, w − wᵢ])`.
- **행렬 경로 (최대):** `c[i, j] = m[i, j] + max(c[i−1, j], c[i, j−1])`.
- **동전 거스름돈 (최소):** `C[j] = min_{dᵢ ≤ j} { C[j − dᵢ] + 1 }`.
- **플로이드-워셜:** `D[i, j] = min(D[i, j], D[i, k] + D[k, j])`, `k` 가장 바깥.

**알고리즘 복잡도 요약:**

| 알고리즘 | 시간 | 공간 | 주의 |
|----------|------|------|------|
| 이진 탐색 | O(log n) | O(1) | 정렬 입력 필요 |
| 병합 정렬 | O(n log n) | O(n) | 안정 |
| 퀵 정렬 (평균) | O(n log n) | O(log n) | 실무에서 가장 빠름 |
| 힙 정렬 | O(n log n) | O(1) | 제자리, 보장 |
| 계수 정렬 | O(n + k) | O(k) | `k = O(n)` 필요 |
| 기수 정렬 | O(d(n + k)) | O(n + k) | LSD → MSD |
| 선택 (랜덤) | O(n) 평균 | O(log n) | 퀵-선택 |
| 선택 (MoM) | O(n) 최악 | O(n) | 결정적 선형 |
| 최근접 쌍 | O(n log² n) | O(n) | 스트립 D&C |
| 크러스컬 | O(m log m) | O(n + m) | 유니온-파인드 |
| 프림 (힙) | O(m log n) | O(n + m) | 우선순위 큐 |
| 다익스트라 (힙) | O(m log n) | O(n + m) | 음 아닌 가중치 |
| 플로이드-워셜 | O(n³) | O(n²) | 모든 쌍 |
| LCS, 편집 거리 | O(mn) | O(mn) | 문자열 |
| 0/1 배낭 | O(nC) | O(nC) | 의사다항 |
| 허프만 | O(n log n) | O(n) | 최적 접두 코드 |

## 9. 연습 문제 (시험 스타일)

### 문제 1 — 마스터 정리

**Q.** `T(n) = 4T(n/2) + n`의 복잡도.

**풀이.** `a = 4, b = 2, f(n) = n`. `n^(log_b a) = n^(log₂ 4) = n²`. `f(n) = n`과 `n²` 비교: `n = O(n^(2−1))` (`ε = 1`). **Case 1**. 답: `T(n) = Θ(n²)`.

### 문제 2 — Big-O 증명

**Q.** `5n³ + 2n² + 7 = O(n³)` 증명.

**풀이.** `c = 5 + 2 + 7 = 14, n₀ = 1`. 모든 `n ≥ 1`:
`5n³ + 2n² + 7 ≤ 5n³ + 2n³ + 7n³ = 14n³ = c · n³`. ∎

### 문제 3 — 그리디 vs DP (동전 거스름돈)

**Q.** 동전 `{1, 3, 4}`, 금액 6. 그리디와 DP 비교.

**그리디:** `4 + 1 + 1 = 3`개.

**DP:**

| j    | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|------|---|---|---|---|---|---|---|
| C[j] | 0 | 1 | 2 | 1 | 1 | 2 | **2** |

`C[6] = min(C[5]+1, C[3]+1, C[2]+1) = min(3, 2, 3) = 2`. 최적: `3 + 3 = 2`개.

**논의.** `{1, 3, 4}`에서 그리디 선택 속성 실패: 4를 확정하면 3 두 개라는 더 나은 선택지가 막힘. 고전적 시험 함정.

### 문제 4 — LCS

**Q.** `X = "ABCB"`와 `Y = "BDCAB"`의 LCS.

|     |   | B | D | C | A | B |
|-----|---|---|---|---|---|---|
|     | 0 | 0 | 0 | 0 | 0 | 0 |
| A   | 0 | 0 | 0 | 0 | **1** | 1 |
| B   | 0 | **1** | 1 | 1 | 1 | **2** |
| C   | 0 | 1 | 1 | **2** | 2 | 2 |
| B   | 0 | 1 | 1 | 2 | 2 | **3** |

**답.** 길이 3, LCS = `"BCB"` (여러 개 중 하나). `c[4, 5]`에서 문자 일치 시 대각선으로 추적.

### 문제 5 — 0/1 배낭

**Q.** 물건 `(w, v) = (2, 12), (1, 10), (3, 20), (2, 15)`, 용량 5. `K[4, 5]`와 선택된 물건.

| i \ w | 0 | 1 | 2 | 3 | 4 | 5 |
|-------|---|---|---|---|---|---|
| 0 | 0 | 0 | 0 | 0 | 0 | 0 |
| 1 (2, 12) | 0 | 0 | 12 | 12 | 12 | 12 |
| 2 (1, 10) | 0 | 10 | 12 | 22 | 22 | 22 |
| 3 (3, 20) | 0 | 10 | 12 | 22 | 30 | 32 |
| 4 (2, 15) | 0 | 10 | 15 | 25 | 30 | **37** |

**답.** `K[4, 5] = 37`. 역추적: 물건 1, 2, 4 총 무게 5, 총 가치 37.

### 문제 6 — 퀵 정렬 최악

**Q.** 퀵 정렬(마지막 원소 피벗)이 `Θ(n²)`가 되는 입력은? 랜덤 피벗 선택이 어떻게 해결하나?

**답.** 이미 정렬된 (또는 역정렬된) 배열. 모든 파티션 호출이 `0 : (n−1)` 분할 생성. 점화식은 `T(n) = T(n−1) + Θ(n) = Θ(n²)`.

랜덤 피벗: `[p..r]`에서 균일 랜덤 인덱스 `r'` 선택, `A[r'] ↔ A[r]` 교환 후 표준 파티션. 반복적 최악 분할 확률은 미미; 기대 시간은 `Θ(n log n)`. 보장은 알고리즘의 난수성 위에 있고 입력 위가 아님.

### 문제 7 — 음수 간선 있는 다익스트라

**Q.** 다익스트라가 잘못된 최단 거리를 반환하는 작은 그래프.

**답.** 방향 간선 `s → a = 4, s → b = 2, a → b = −5`.

- `s` 확정. 완화: `D[a] = 4, D[b] = 2`.
- `b` 선택 (최소 = 2). 확정. 나가는 간선 없음.
- `a` 선택 (D = 4). 확정. `a → b` 완화 시도: `D[b] = −1`이 되어야 하지만, `b`가 이미 확정 — 갱신 무시.

다익스트라 `D[b] = 2` 반환. 실제 최단 거리: `s → a → b = 4 − 5 = −1`. 그리디 불변식 "한번 확정되면 거리가 최적"은 음수 간선이 이미 확정된 정점을 개선할 수 있는 순간 깨진다.

### 문제 8 — 활동 선택

**Q.** 활동 `(s, f)`: `(1, 3), (2, 5), (4, 7), (1, 8), (5, 9), (8, 10)`. 가장 빠른 종료 시간 적용.

종료 시간 정렬: `(1,3), (2,5), (4,7), (1,8), (5,9), (8,10)`.

- `(1,3)` 선택: `last = 3`.
- `(2,5)`: `시작 2 < 3` → 스킵.
- `(4,7)`: `4 ≥ 3` → 선택. `last = 7`.
- `(1,8)`: `1 < 7` → 스킵.
- `(5,9)`: `5 < 7` → 스킵.
- `(8,10)`: `8 ≥ 7` → 선택. `last = 10`.

**답.** 3개 활동: `{(1, 3), (4, 7), (8, 10)}`.

### 문제 9 — 반복 대입으로 병합 정렬 점화식

**Q.** 마스터 정리 없이 `T(n) = 2T(n/2) + n`, `T(1) = 1` 풀기.

**답.** `n = 2ᵏ` 가정. 전개:
```
T(n) = 2T(n/2) + n
     = 4T(n/4) + 2n
     = 2ᵏ T(n/2ᵏ) + kn
```
`2ᵏ = n` (`k = log₂ n`): `T(n) = n · T(1) + n log₂ n = n + n log n = Θ(n log n)`. ∎

### 문제 10 — 허프만 코딩

**Q.** `A:5, B:9, C:12, D:13, E:16, F:45`의 허프만 코드 구성. 문자당 평균 비트 계산.

**구성.**
```
Q = [A:5, B:9, C:12, D:13, E:16, F:45]

A+B 병합 → N1:14
Q = [C:12, D:13, N1:14, E:16, F:45]

C+D 병합 → N2:25
Q = [N1:14, E:16, N2:25, F:45]

N1+E 병합 → N3:30
Q = [N2:25, N3:30, F:45]

N2+N3 병합 → N4:55
Q = [F:45, N4:55]

F+N4 병합 → Root:100
```

유효한 한 코드 트리 (왼쪽=0, 오른쪽=1):
- `F = 0` (길이 1)
- `C = 100` (길이 3)
- `D = 101` (길이 3)
- `A = 1100` (길이 4)
- `B = 1101` (길이 4)
- `E = 111` (길이 3)

총 비트 = 5·4 + 9·4 + 12·3 + 13·3 + 16·3 + 45·1 = 20 + 36 + 36 + 39 + 48 + 45 = **224**. 평균 = 224 / 100 = **2.24 비트/문자**.

### 문제 11 — 크러스컬 MST

**Q.** 간선 `(A,B,4), (A,C,1), (B,C,2), (B,D,5), (C,D,3)`에 크러스컬 적용. MST 계산.

가중치로 간선 정렬: `(A,C,1), (B,C,2), (C,D,3), (A,B,4), (B,D,5)`.

- `(A,C,1)`: 추가. 성분: `{A,C}, {B}, {D}`.
- `(B,C,2)`: 추가. 성분: `{A,B,C}, {D}`.
- `(C,D,3)`: 추가. 성분: `{A,B,C,D}`. **완료** (n − 1 = 3 간선).

MST 가중치: `1 + 2 + 3 = 6`. 스킵된 간선: `(A,B,4)`와 `(B,D,5)`는 사이클 생성.

### 문제 12 — 점화식 귀납 증명

**Q.** `T(n) = T(n/2) + 1, T(1) = 1`이 `T(n) = O(log n)`을 만족함을 귀납으로 증명.

**추측:** `T(n) ≤ c log₂(n) + d`, 상수 `c, d`는 나중에 고정.

*기저:* `T(1) = 1 ≤ c · 0 + d = d`, `d ≥ 1`이면 작동.

*귀납 단계:* `T(n/2) ≤ c log(n/2) + d = c log n − c + d` 가정. 그러면

```
T(n) = T(n/2) + 1
     ≤ c log n − c + d + 1
     ≤ c log n + d          (c ≥ 1일 때)
```

`c = 1, d = 1` 선택. ∎

## 10. 자주 하는 실수

### 10.1 표기와 증명

| 함정 | 예방 |
|------|------|
| `O`, `Ω`, `Θ` 혼동 | `O` = 상한, `Ω` = 하한, `Θ` = 양쪽 타이트. "`Θ`가 `O`보다 낫다" 같은 말 금지. 서로 다른 경계를 기술. |
| 느슨한 `O` 경계 | 둘 다 성립하면 `O(n log n)`이 `O(n²)`보다 선호; 느슨함은 정보 손실. |
| `c, n₀` 명시 없이 `O` 증명 | 항상 "`c = ..., n₀ = ...` 취하면"을 쓸 것. 채점자가 증인을 보고 싶어함. |
| "귀납: `cn + 1 ≤ cn`?" | **아니다, 증명 아님.** 추측을 `cn − d`로 강화. |
| `f(n)`이 다항식 아닌데 마스터 정리 | 예: `T(n) = 2T(n/2) + n log n`은 확장 MT 또는 대입 필요. |
| Case 3 정규성 망각 | 대부분 시험은 Case 1, 2만 물어봄; Case 3 우회 가능. |

### 10.2 정렬

| 함정 | 예방 |
|------|------|
| "퀵 정렬은 `O(n log n)`" (단서 없이) | 명시: 평균 `O(n log n)`, 최악 `O(n²)`. |
| "병합 정렬은 안정하고 `O(1)` 공간" | 아니, 병합 정렬은 `O(n)` 보조 공간 필요. |
| "계수 정렬은 임의 정수 입력에 작동" | `k = O(n)`일 때만. `k = n²`면 비교 정렬보다 느림. |
| "기수 정렬은 MSD부터 가능" | LSD부터가 단순 고전. MSD부터는 다름 (재귀). |
| "힙 정렬은 안정" | 아니, 힙 정렬은 불안정. |
| 최소 힙과 최대 힙 혼동 | 최소 힙은 최소 추출; 힙 정렬에서 내림차순 결과. 최대 힙은 오름차순. |
| `buildHeap`이 `O(n log n)` | 아니, 타이트 분석으로 `O(n)`. 깊이 `h`의 heapify는 `O(h)`, 깊은 노드는 얕은 노드보다 훨씬 적음. |
| 비교 기반 정렬 하한 `Ω(n²)` | 아니, `Ω(n log n)`, 결정 트리로 증명. |

### 10.3 그리디

| 함정 | 예방 |
|------|------|
| 증명 없이 그리디 최적 주장 | 항상 먼저 반례 확인, 그 다음 그리디 선택 속성 증명. |
| 분할 가능 vs 0/1 배낭 혼동 | 분할 가능: 가치/무게 정렬, 그리디. 0/1: DP. |
| 작업 스케줄링 vs 활동 선택 혼동 | 여러 기계 → EST. 단일 기계 → EFT. |
| 음수 간선에 다익스트라 | 벨만-포드 사용. 다익스트라 그리디 불변식은 음 아닌 가중치 필요. |
| 유니온-파인드 없는 크러스컬 | 사이클 검출이 간선당 `O(n)` — 복잡도 파괴. |
| 희소 그래프에 프림이 최선 | 실제로 희소에는 보통 크러스컬; 프림은 밀집에 빛남. |
| 모든 쌍이 필요한데 다익스트라 | 플로이드-워셜 사용 또는 `n`회 다익스트라 실행 (복잡도 주의). |

### 10.4 DP

| 함정 | 예방 |
|------|------|
| 잘못된 기저 | `n = 0, 1, 2`에서 점화식 손 추적. |
| DP 테이블 off-by-one | 0-인덱스인지 1-인덱스인지 명시. ε를 위한 헤더 행/열 포함. |
| 플로이드-워셜 루프 순서 오류 | `k`가 **반드시** 가장 바깥. 나머지 둘은 임의 순서 가능. |
| `O(min(m,n))`이면 되는데 `O(mn)` 공간 | LCS 길이만이면 두 행만 있으면 됨. 큰 입력에서 메모리 절약. |
| "중복 부분 문제"와 "입력 공유" 혼동 | 중복은 순진한 재귀에서 *같은* 부분 문제가 여러 번 계산됨을 의미. |
| "0/1 배낭은 다항" | *의사*-다항 — 용량 값에 대해 다항, 입력 비트 길이에 대해서는 아님. 고전적 의미에서 NP-hard. |

### 10.5 D&C

| 함정 | 예방 |
|------|------|
| 부분 문제가 겹치는데 D&C | 대신 DP. |
| 의사코드에 기저 망각 | 항상 "if `n ≤ 어떤 임계값`, 직접 풀이" 명시. |
| 최근접 쌍 스트립 검사에서 모든 쌍 비교 | y좌표로 다음 7개 이웃만 비교. |
| 3개 그룹의 중앙값의 중앙값 | 선형 시간 안 됨. 5개 그룹 사용. |
| "랜덤 선택은 항상 `O(n)`" | **기대** `O(n)`. 최악은 여전히 `O(n²)`. |

## 11. 최종 체크리스트

시험장에 들어가기 전에, 이 모든 것에 "예"로 답할 수 있어야 합니다.

**기초 (W01):**
- [ ] 알고리즘의 네 가지 성질을 말할 수 있다.
- [ ] 유클리드 GCD를 손으로 추적한다.
- [ ] 명시적 배열에 이진 탐색을 적용한다.
- [ ] 정점 차수에 대한 오일러 회로/경로 조건을 말한다.
- [ ] 가짜 동전과 독 든 포도주 문제가 왜 `log₂(n)`을 사용하는지 설명한다.

**복잡도 (W02):**
- [ ] 주어진 의사코드의 기본 연산 수를 센다.
- [ ] 명시적 `c, n₀`을 찾아 `f(n) = O(g(n))` 증명한다.
- [ ] 일치하는 `Ω(g(n))` 경계를 증명해 `Θ`를 얻는다.
- [ ] `T(n) = 2T(n/2) + n`을 대입으로 푼다.
- [ ] 마스터 정리 Case 1과 2를 자유롭게 적용한다.
- [ ] 마스터 정리가 다루지 않는 점화식을 식별한다.
- [ ] `cn − d` 강화 트릭으로 귀납 증명을 완성한다.

**정렬 (W03):**
- [ ] 정렬 완전 비교표를 재현한다.
- [ ] 작은 배열에 파티션을 추적한다.
- [ ] 정렬된 입력에서 퀵 정렬이 왜 `Θ(n²)`이고 랜덤화가 어떻게 해결하는지 설명한다.
- [ ] 비교 기반 정렬이 왜 `Ω(n log n)`인지 설명한다.
- [ ] `buildHeap`이 왜 `O(n log n)`이 아니라 `O(n)`인지 설명한다.
- [ ] 계수/기수 정렬이 언제 적용 가능한지 말한다.

**D&C (W04):**
- [ ] 3단계(분할/정복/결합)를 말한다.
- [ ] D&C와 DP의 차이(독립 vs 중복 부분 문제)를 설명한다.
- [ ] 작은 예제에서 최근접 쌍 스트립 검사를 추적한다.
- [ ] 중앙값의 중앙값이 왜 정확히 5개 그룹을 사용하는지 설명한다.

**그리디 (W05):**
- [ ] 그리디 선택 속성과 최적 부분 구조를 말한다.
- [ ] 그리디 동전 거스름돈이 실패하는 반례를 제시한다.
- [ ] 작업 스케줄링(EST)과 활동 선택(EFT)을 구별한다.
- [ ] 작은 그래프에 크러스컬과 프림을 실행한다.
- [ ] 음 아닌 가중치의 작은 그래프에 다익스트라를 실행한다.
- [ ] 주어진 빈도의 허프만 트리를 구축한다.
- [ ] 다익스트라가 실패하는 음수 간선 반례를 제시한다.

**DP (W06):**
- [ ] 두 필수 속성(최적 부분 구조, 중복 부분 문제)을 말한다.
